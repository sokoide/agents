# X68000 ゲームアーキテクチャ

C言語による実践的なゲーム制作パターン。スプライト管理、当たり判定、背景制御の設計指針。

---

## ゲームループ構造

```
初期化 → データロード → メインループ → 終了・復元

メインループ:
  入力処理 → 論理演算(移動・AI・当たり判定) → 描画データ準備 → VSYNC待ち → ハードウェア反映
```

### 処理落ち

1フレーム内に処理が収まらない場合、次のVSYNCまで待ちになり、実質フレームスキップが発生する。X68000/XVI/030で**同じ速度**で動くが、処理落ち限界点が異なる。速度調整はVSYNC同期で行い、CPU速度に依存しない設計にすること。

## 68000 アラインメント制限

ワード (16bit) / ロングワード (32bit) アクセス時は**偶数アドレス整列が義務**。奇数番地からワード以上のアクセスを行うと Address Error が発生し、システムがクラッシュする。

### 構造体パディングによる対策

構造体メンバの型サイズと配置順序に注意し、アラインメントのズレが発生しないようにパディングを手動で挿入する。

```c
typedef struct {
    short x;              /* 2バイト (偶数境界) */
    short y;              /* 2バイト (偶数境界) */
    unsigned short code;  /* 2バイト (偶数境界) */
    unsigned short attr;  /* 2バイト (偶数境界) */
} SPRITE_OBJ; /* 合計 8バイト (偶数サイズ) */
```

コンパイラ最適化時に意図しないパディングが挿入されてハードウェアレジスタの物理レイアウトとズレるのを防ぐため、必要に応じて `__attribute__((packed))` を付与するか、メンバサイズを 2/4 バイトの倍数に固定する。

## スプライト管理 (2層アーキテクチャ)

### 下層: ハードウェアSAT

`disp_sp[128]` — ハードウェアスクロールレジスタ ($EB0000) の仮想コピー。VSYNC割り込み内で `$EB0000` に一括転送するダブルバッファリング。

```c
static SP_CTRL disp_sp[128];  /* 仮想SAT */
```

`$EB0808` の bit9 (DISP/CPUスイッチ) 制御:
- 書き換え前: `*(short*)0xeb0808 &= 0xfdfc;` (CPUに開放)
- 書き換え後: `*(short*)0xeb0808 |= 0x0200;` (表示ON)

### 上層: 論理スプライト

```c
/* スプライト1個の管理データ */
typedef struct {
    SP_CTRL *spr;          /* 下層SATへのポインタ */
    short   char_no;       /* スクロールレジスタ番号 (0-127) */
    short   pcg_defined;   /* PCG定義済みフラグ */
    short   anime;         /* アニメーションパターン数 */
    short   pcg_no[8];     /* アニメ用PCG番号 */
    PCG     *pcg_data[8];  /* アニメ用PCGデータポインタ */
} SPRITE;

/* ゲームキャラクタの管理データ */
typedef struct SP_DATA {
    enum sp_type type;     /* DOT16/DOT32/DOT48 */
    short life;            /* 生存時間 */
    short registance;      /* 耐久力/ダメージ */
    short work0, work1, work2, work3;  /* 汎用ワーク */
    sprite body[3][3];     /* マルチサイズ対応 (最大3x3 SPRITE) */
    move_def def_move;     /* 移動定義(配列 or 関数ポインタ) */
    short  *anime_def;     /* アニメーション定義 */
} *Sprite;
```

### アロケーション管理

```c
static char this_no_is_use[128];  /* スクロールレジスタ使用状況 */
static char pcg_is_use[128];      /* PCG番号参照カウント */
```

- `alloc_sprite()`: 空きスクロールレジスタを検索し、SPRITEをmalloc
- `free_sprite()`: 参照カウントをデクリメントし、0になったPCGを解放
- `dup_sprite()`: 既存Spriteを複製 (PCGは共有、参照カウントをインクリメント)

### マルチサイズ対応

```c
enum sp_type {
    DOT16,           /* 16x16: body[0][0] のみ使用 */
    DOT32_1_2,       /* 16x32: body[0][0], body[0][1] */
    DOT32_2_1,       /* 32x16: body[0][0], body[1][0] */
    DOT32_2_2,       /* 32x32: body[0-1][0-1] */
    DOT48_1_3,       /* 16x48: body[0][0-2] */
    DOT48_2_3,       /* 32x48: body[0-1][0-2] */
    DOT48_3_3,       /* 48x48: body[0-2][0-2] */
    DOT48_3_1,       /* 48x16: body[0-2][0] */
    DOT48_3_2,       /* 48x32: body[0-2][0-1] */
};
```

## 動的オブジェクト管理 (friend_loop)

弾や敵キャラクターなど、数がリアルタイムに変遷するオブジェクトは、固定長配列ではなくポインタを用いたリンクリスト構造で管理する。

- `malloc` により必要な瞬間にオブジェクト用メモリを動的に確保
- リンクリストのポインタを付け替えるだけで、高速かつメモリ効率の高いオブジェクト消去が可能
- 実装コードの具体例は [point.c](file:///Users/scott/repo/sokoide/agents/skills/x68k-master/resources/samples/point.c) を参照

```c
typedef struct friend_loop {
  char *name;
  char *address;
  struct friend_loop *next;
} FRIEND;

void add_friend(FRIEND **root, char *name) {
    FRIEND *new_p = (FRIEND *)malloc(sizeof(FRIEND));
    new_p->name = name;
    new_p->next = *root;
    *root = new_p;
}
```

## SP_CTRL 構造体リファレンス

スプライトスクロールレジスタ (4ワード = 8バイト) のC言語定義。

```c
typedef struct {
    short int sp_x;              /* 表示座標X (仮想画面1024x1024上) */
    short int sp_y;              /* 表示座標Y */
    union {
        short int dummy;
        struct {
            unsigned v_invert : 1;  /* 垂直方向反転 */
            unsigned h_invert : 1;  /* 水平方向反転 */
            unsigned dummy    : 2;  /* 未使用 */
            unsigned color    : 4;  /* スプライトパレット (0-15) */
            unsigned sp_code  : 8;  /* PCGコード (0-255) */
        } sp;
    } sp_ctrl;
    union {
        short int dummy;
        struct {
            unsigned dummy : 13;   /* 未使用 */
            unsigned ext   : 1;    /* 拡張用,未使用 */
            unsigned pwr   : 2;    /* BG画面とのプライオリティ (0=非表示) */
        } sp;
    } sp_pwr;
} SP_CTRL;
```

- 仮想画面サイズ: 1024×1024。256×256モードの表示範囲は (16,16)〜(271,271)
- `pwr`: 0でスプライト非表示。1-3でBG画面との前後関係を制御
- レジスタ番号0が最優先。128個のスクロールレジスタが存在

## PCG定義 (DEF_SPマクロ)

PCGパターンを高速に書き込むマクロ。`$EB8000 + 0x80 * pat_no` にint* (ロングワード) で転送。

```c
#define DEF_SP(pat_no, sp_dat)                           \
{                                                         \
    int *sp_reg = (int *)(0xeb8000 + 0x80 * (pat_no));   \
    int *dat = (int *)(sp_dat);                           \
    /* 左半分 16ロングワード */                              \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    /* 右半分 16ロングワード */                              \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
    *sp_reg++ = *dat++; *sp_reg++ = *dat++;               \
}
```

1パターン = 128バイト = 32ロングワード (左16 + 右16)。VSYNC割り込み内で定義すること。

## VSYNC割り込みハンドラ (vsync_disp)

```c
static CRTC_REG *const crtc = (CRTC_REG *) 0xe80018;

void vsync_disp(void) {
    *(short *)0xe82500 = 0x09e4;          /* H-SYNCカウンタリセット */

    if (sp_is_ready || request_def) {
        /* SPスクロールレジスタをCPUに開放 */
        *(short *)0xeb0808 &= 0xfdfc;

        /* PCG定義 (1フレームに最大4パターン) */
        if (request_def) {
            /* 空きPCG番号を検索 → DEF_SPマクロで書き込み */
        }

        /* スプライトSAT一括転送 */
        if (sp_is_ready) {
            SP_CTRL *sp_scr_reg = (SP_CTRL *)0xeb0000;
            for (int i = 0; i < 128; i++)
                if (this_no_is_use[i])
                    *sp_scr_reg++ = disp_sp[i];
        }

        /* SP表示ON */
        *(short *)0xeb0808 |= 0x0200;
    }

    /* CRTCスクロールレジスタ反映 */
    *crtc = scroll_data;

    vsync_counter++;
}
```

## 当たり判定

### 矩形判定 (バウンディングボックス)

座標の絶対値差が閾値未満なら衝突と判定。閾値はキャラクタサイズに応じて調整。

```c
#define 絶対値(X,Y) ((X) - (Y) >= 0 ? (X) - (Y) : -((X) - (Y)))

/* 弾と敵の当たり判定 */
if (絶対値(弾_x, 敵_x) < 10 && 絶対値(弾_y, 敵_y) < 10) {
    /* 衝突 */
}
```

### 一括消去バッファ

イテレーション中にスプライトを個別freeするとリストが破壊される。消去を遅延させ、VSYNC経過後に一括実行する。

```c
static Sprite 消去バッファ[128];
static int 消去カウンタ;

void 消去記録(Sprite 対象, int 番号) {
    display_sprite(対象, 非表示);         /* まず非表示にする */
    消去バッファ[消去カウンタ++] = 対象;  /* バッファに記録 */
}

void 消去実行(void) {
    for (int i = 0; i < 消去カウンタ; i++)
        delete_sprite(消去バッファ[i]);
    消去カウンタ = 0;
}
```

メインループでは `当たり判定()` → `消去実行()` の順で呼び出す。

## 背景アーキテクチャ

### グラフィック画面スクロール

CRTCスクロールレジスタ ($E80018) を構造体で一括設定。

```c
typedef struct {
    short sc0_x_reg, sc0_y_reg;   /* グラフィック画面0 */
    short sc1_x_reg, sc1_y_reg;   /* グラフィック画面1 */
    short sc2_x_reg, sc2_y_reg;   /* グラフィック画面2 */
    short sc3_x_reg, sc3_y_reg;   /* グラフィック画面3 */
} CRTC_REG;

CRTC_REG scroll_data;
/* VSYNC割り込み内で: */
*(CRTC_REG *)0xe80018 = scroll_data;
```

### BG画面 (スコア/HUD)

BG画面はグラフィック画面とは独立してスクロール可能。スコア表示等に利用。

```c
/* IOCS呼び出し */
_iocs_bgtextcl(0);       /* BGテキストクリア */
_iocs_bgtextst(0, addr); /* BGテキスト設定 */
_iocs_bgscrlst(0, x, y); /* BGスクロール設定 */
_iocs_bgctrlst(0, ...);  /* BG制御設定 */
```

### ラスタースクロール

H-SYNC割り込みでCRTCスクロールレジスタを行単位で変更し、波形効果を演出。事前計算したsinカーブテーブルを使用。

```c
/* sinカーブテーブル生成 */
for (int i = 0; i < 256; i++)
    sin_table[i] = amplitude * sin(2 * PI * i / period);
```

## 広域マップ (512×512)

画面より大きいマップを効率的に表示するパターン。

### マップデータ形式

- **ワード配列**: 512×512 = 256KB。単純だがメモリ消費大
- **バイト配列 + 変換テーブル**: 256種類以下のタイルなら1バイト/タイルで64KBに圧縮
- **ランレングス圧縮**: さらに圧縮可能

### 差分更新

毎フレーム全タイルを書き直さず、新しく可視範囲に入ったタイルのみ描画。

```
スクロール → 可視範囲の端を計算 → 新規露出タイルのみGVRAMに書き込み
```

### ビューポート座標計算

```
スクロール位置(scroll_x, scroll_y)
  → タイルインデックス = (scroll_x / 16, scroll_y / 16)
  → GVRAM書き込みアドレス = タイルパターンを256色モードで展開
```

256色モードのGVRAMは1ドット=1バイト、ストライド=512バイト。

## 例外ハンドリング (TRAP #14)

スーパーバイザモードでの直接ハードウェア制御は Human68k の保護対象外。バスエラーやアドレスエラーを独自にトラップし、デバッグ情報をログ出力して安全に Human68k へ復帰する仕組みが不可欠。

### TRAP #14 ベクタのフック

`trap #14`（ベクタ `$2e`）を自前のハンドラにフックすることで、システムをフリーズさせることなくクラッシュ原因を特定できる。

- 実装の全体構造は [trap14.c](file:///Users/scott/repo/sokoide/agents/skills/x68k-master/resources/samples/trap14.c) を参照
- テストプログラムは [bustrap.c](file:///Users/scott/repo/sokoide/agents/skills/x68k-master/resources/samples/bustrap.c) を参照

```c
#include <interrupt.h>

static void (*trap_14)();

static void trap14_handler(void) {
    PRAMREG(code, d7);
    PRAMREG(add,  a6);
    SET_FRAME(a5);
    /* デバッグログ出力とシステム復旧処理 */
}

void init_trap14(void) {
    trap_14 = (void *)INTVCG(0x2e);
    INTVCS(0x2e, trap14_handler);
}
```

ハンドラ登録時は、終了時に必ず旧ベクタを復元すること。
