# X68000 BG（背景画面）

X68000のBGは、スプライト機能の一部であり、スプライトとの前後関係を指定して、主にゲームの背景表示に利用する。

## BGテキスト

背景を設定するには、まず「BGテキスト」と呼ぶ64×64マスの方眼に、スプライトパターンと同様なパターンコードを書き込む。

BGテキスト面は2枚持つことができ、これを「BG画面」という表示用のシステムに紐付けることで画面に表示される。
BG画面は、表示モードにより2枚または1枚利用でき、独立してスクロールも可能。

なおBGテキストの実体は、BG0が$ebc000-$ebdfff、BG1が$ebe000-$ebffffの、それぞれ64×64ワード（4096ワード、8KB）にある。
ここはパターン番号128-191および192-255の領域でもあるため、BGテキストを使うと使用可能なスプライトパターン数が減るというジレンマがある。

## パターンコード

BGテキストに書き込むパターンコードは、スプライト表示と同じ構成。

- bit 0-7: パターン番号 (0-255)
- bit 8-11: パレット番号 (0-15)
- bit 14: 左右反転 (0 or 1)
- bit 15: 上下反転 (0 or 1)

ただしパターン番号が示す絵柄は、表示モードによって以下のように変わる。

| 表示画面サイズ | 絵柄 | 仮想画面サイズ | 使用可能なBG画面 |
| --- | --- | --- | --- |
| 256x256 | 8×8dot | 512x512 | BG画面0 または 1 |
| 512x512 | 16×16dot | 1024x1024 | BG画面0 |
| 384x256 | 16×16dot | 1024x1024 | BG画面0 |

CRTCを操作して作る384x256ドットモードでは16×16ドットのマップチップ扱いになる。256x256モードで作っていたゲームを途中で384x256に作り変える場合はチップサイズの違いに注意。

## IOCSコール一覧

| IOCS番号 | 名前 | 機能 |
| --- | --- | --- |
| $C8 | BGSCRLST | BGスクロール座標設定 |
| $CA | BGCTRLST | BG画面の表示設定（テキスト割当・表示ON/OFF） |
| $CC | BGTEXTCL | BGテキストの一括クリア |
| $CD | BGTEXTST | BGテキストの個別配置 |

## 設定手順（IOCSコール）

BGはスプライト機能の一部なので、最初にスプライトを初期化し有効化する。
スプライトパターン（PCG）の登録も同様。引数の指定により8×8ドット単位での登録も可能。
以降、BGテキストの書き換え → BG画面への割当 → スクロール座標設定と続く。

```asm
.include    iocscall.mac
* _SP_INIT    equ    $C0        * sp_init()
* _SP_ON    equ    $C1        * sp_disp(1)
* _SP_OFF    equ    $C2        * sp_disp(0)
* _BGSCRLST    equ    $C8        * bg_scroll()
* _BGCTRLST    equ    $CA        * bg_set()
* _BGTEXTCL    equ    $CC        * bg_fill()
* _BGTEXTST    equ    $CD        * bg_put()

    * BGテキストをクリアする
    moveq    #0,d1    * BG画面(0 or 1→256x256モードのみ可)
    clr    d2    * パターンコード
    IOCS    _BGTEXTCL

    * BGテキストを配置する
    moveq    #0,d1    * BG画面(0 or 1→256x256モードのみ可)
    moveq    #10,d2    * X座標(0〜63)
    moveq    #20,d3    * Y座標(0〜63)
    move    #$101,d4    * パターンコード
    IOCS    _BGTEXTST

    * スクロール座標を設定する
    moveq.l    #0,d1    * BG画面(0 or 1→256x256モードのみ可)
    *bset    #31,d1    * vsyncなしならbit31を1にする
    move    scrollX(a6),d2    * スクロールX座標
    andi    #1023,d2
    move    scrollY(a6),d3    * スクロールY座標
    andi    #1023,d3
    IOCS    _BGSCRLST

    * BG面を表示する
    moveq.l    #0,d1    * BG画面(0 or 1→256x256モードのみ可)
    moveq    #1,d2    * BGテキスト(0 or 1)
    moveq    #1,d3    * 表示するなら1、消すなら0
    IOCS    _BGCTRLST
```

## 設定手順（直叩き）

Supervisor モードで直接メモリを書き換える方法。IOCSコールを使わずに同等の操作を行う。

```asm
    * BGテキスト0 のクリア（$ebc000、64×64ワード）
    movea.l #$ebc000,a0    * BGテキスト0
    *movea.l #$ebe000,a0    * BGテキスト1
    move    #64*64-1,d0
    clr    d2
    @@:
        move    d2,(a0)+
        dbra    d0,@b

    * BGテキスト0 へパターンコードを配置（X=10, Y=20）
    moveq    #10,d2
    moveq    #20,d3
    andi    #63,d2    * X:0-63
    andi    #63,d3    * Y:0-63
    lsl    #6,d3    * 6bit左シフト=64倍
    add    d2,d3    * d3= X+Y*64
    add    d3,d3    * ワード単位なので2倍する
    movea.l #$ebc000,a0    * BGテキスト0
    *movea.l #$ebe000,a0    * BGテキスト1
    move    #$101,d4
    move    d4,(a0,d3.w)

    * BG画面0 のスクロール座標設定（$eb0800/$eb0802）
    move    d2,$eb0800
    move    d3,$eb0802
    *move    d2,$eb0804    * BG画面1
    *move    d3,$eb0806

    * 表示制御レジスタ（$eb0808）
    move    #%1_000_000_011,$eb0808
    * bit 15-10: 未使用。00000でよい
    * bit 9: 1=スプライト表示 0=CPUアクセス
    * bit 8-6: 未使用。000でよい
    * bit 5-4: BG画面1へのBGテキスト割当。00か01
    * bit 3: BG画面1を使うなら1
    * bit 2-1: BG画面0へのBGテキスト割当。00か01
    * bit 0: BG画面0を使うなら1
```

## タイミング上の注意

BGテキストは、スプライトレジスタと違って毎フレームごとに書き換える必要はない。
ただしスプライト表示中はメモリアクセスが遅くなるため、垂直帰線期間中に表示をオフにして書き換えるのが推奨される。
