# X68000 入力デバイス — ハードウェアリファレンス

キーボード制御は MFP（MC68901）の USART、マウス制御は SCC（Z8530）、ジョイスティックは I/O コントローラが担当。

---

## システムポート

| ポート | アドレス | 主要ビット |
| --- | --- | --- |
| システムポート#2 | `$E8E003` | Bit1: Mouse L-Button / Bit0: Mouse R-Button / デバイス接続監視 |
| システムポート#4 | `$E8E007` | キーボード制御信号（LED、リピート設定） |

`$E8E003` はハードウェア接続状況と物理入力の有無を判定する最優先監視対象。

---

## キーボード（MFP USART 経由）

### 関連レジスタ

| レジスタ | アドレス | 役割 |
| --- | --- | --- |
| RSR | `$E8802B` | Bit7 (BF: Buffer Full): 受信データ準備完了で '1' |
| UDR | `$E8802F` | キーボードからのスキャンコードが格納される |

- 通信仕様: 4800bps / 非同期 / 8bit / パリティなし（Timer B 生成クロック依存）
- 擬似キー入力: 物理入力をマスクした上で UDR にスキャンコードを書き込むことで可能。

### 同期タイミング

GPIP（`$E88001`）経由:

- **GPIP 7 (H-SYNC)**: 水平同期
- **GPIP 4 (V-DISP)**: 垂直表示期間

V-DISP 監視でフレーム単位の入力精度を確保。

---

## マウス（SCC 経由）

トラックマウスは SCC Ch.B（`$E98001`/`$E98003`）で制御。相対座標パケットの連続処理。

- 割り込みレベル: **5**（SCC）
- マウスカーソル制御: SCC バッファに AI 生成の座標変化パケットを周期的に供給することで、物理マウスの移動をシミュレート可能。

---

## ジョイスティック

2 系統（ポート#1/#2）のジョイスティック端子。アタリ規格準拠 D-Sub 9 ピン。初代〜XVI まで完全互換。

### データレジスタ

| ポート | アドレス |
| --- | --- |
| ポート#1 データ | `$E9A001` |
| ポート#2 データ | `$E9A003` |

**負論理**: スイッチ ON で '0'。

| ビット | 入力 |
| --- | --- |
| Bit 0 | 上 (UP) |
| Bit 1 | 下 (DOWN) |
| Bit 2 | 左 (LEFT) |
| Bit 3 | 右 (RIGHT) |
| Bit 4 | 未使用 |
| Bit 5 | ボタンA (TRIGGER 1) |
| Bit 6 | ボタンB (TRIGGER 2) |

### コントロールレジスタ（`$E9A005`）

ポートのピン出力を制御。コントロールワード送出で信号線の役割を動的に変更するマルチプレクス機能を提供。多ボタンパッドやトラックマウス等の拡張デバイスは、このコントロールワードによるサンプリングが必要。

### タイミング制約

コントロールワード書き込み直後は**信号が電気的に安定していない**。数マイクロ秒のウェイト（NOP 命令等）が必要。このウェイトを無視したコードは XVI 等の高速機で深刻な誤動作を引き起こす。

### 実装例

```c
volatile unsigned char *joy1 = (unsigned char *)0xE9A001;
unsigned char raw = *joy1;
unsigned char active = ~raw;  // 負論理を正論理に反転

if (active & 0x20) { /* ボタンA */ }
if (active & 0x40) { /* ボタンB */ }
if (active & 0x01) { /* 上 */ }
if (active & 0x02) { /* 下 */ }
```

### チャタリング対策

16MHz CPU はスイッチの物理的接点ノイズ（チャタリング）を数回〜数十回サンプリングする。前回値との比較で一定期間の安定を確認すること。

上/下の同時入力等の異常入力に対しても、「上優先」等の論理規定をプログラム側で設けること。

---

## 割り込み優先順位

| レベル | デバイス | 担当 |
| --- | --- | --- |
| **6** | MFP | キーボード、タイマ |
| **5** | SCC | マウス、RS-232C |

レベル 6（キーボード）の処理に時間を取られると、レベル 5（マウス）のパケットをドロップする可能性がある。両レベルを統合監視する設計が必要。

---

## 物理入力のマスク（割り込み抑制）

物理入力と AI 生成入力を分離するため、MFP レジスタで物理デバイスからの割り込みを一時マスク:

- **IERA/IERB**（`$E88007`/`$E88009`）: 割り込み発生の許可/禁止
- **IMRA/IMRB**（`$E88013`/`$E88015`）: CPU への通知抑止

マスク中に独自ハンドラを例外ベクタテーブル（MFP: `$40-$4F`, SCC: `$50-$5F`）に登録することで、OS やアプリに変更を加えず自動操作が可能。
