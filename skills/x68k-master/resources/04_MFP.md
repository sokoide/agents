# X68000 MFP (MC68901) 

X68000のポテンシャルを最大限に引き出すためには、ハードウェアと密接に連携する「低レイヤ」の制御が不可欠である。その中核を担うのが、MC68901、通称**MFP (Multi-Function Peripheral)**である。本資料では、システムアーキテクトの視点から、MFPの内部構造、レジスタ配置、およびプログラミングにおける勘所を網羅的に解説する。

--------------------------------------------------------------------------------

## 1. MFP (MC68901) の概要とシステム上の役割

MFPは、X68000システムにおいて「周辺機器の統合管理」と「非同期イベントの交通整理」を担う戦略的デバイスである。タイマ、シリアル通信、汎用I/O、および割り込みコントローラの4機能を1チップに集約しており、限られたCPUリソースを効率的に運用するための鍵となる。

### 内部ブロック構成と接続関係

MFPは主に以下の4つのブロックで構成され、X68000内の主要デバイスと密接にリンクしている。

- **タイマブロック（Timer A-D）**: システムクロックの分周やイベント計測を行う。タイマAにはCRTCのV-DISP信号が入力され、タイマBはキーボード通信のボーレート生成に直結している。
- **USARTブロック**: 非同期シリアル通信をサポート。X68000ではキーボードとの通信に特化して運用される。
- **GPIP（汎用I/Oポート）**: 8ビットの信号入力を管理。CRTC、FM音源、RTC、電源スイッチ等のステータス監視を行う。
- **割り込み制御ブロック**: 上記各ブロックおよび外部信号（H-SYNC等）からの割り込み要求を集約し、CPUへ通知する。

### システム設計における重要性

MFPはシステム全体の「時間軸」と「外部イベント」を統括している。MFPの制御が不適切であれば、キーボード入力の取りこぼしや、CRTCとの同期不全（画面の乱れ）に直結する。周辺機器との確実な同期と即応性を維持するためには、MFPの精密なレジスタ制御が最優先事項となる。

次に、これらの機能を操作するための物理的な窓口である、メモリアドレス配置を確認する。

--------------------------------------------------------------------------------

## 2. メモリアドレス配置とレジスタ・インターフェース

X68000のメモリマップにおいて、MFPはシステムI/O領域である `$E88001` から `$E8802F` に配置されている。68000 CPUには専用のI/O空間が存在しないため、メモリマップドI/Oとして読み書きを行う。

### MFPレジスタマップ

全レジスタは8ビット長であり、X68000の設計上、**奇数番地（データバスの下位8ビット D0-D7）のみ**に配置されている。

|アドレス|略称|名称|R/W|

|$E88007|IERA|割り込みイネーブルレジスタ A|R/W|
|$E88009|IERB|割り込みイネーブルレジスタ B|R/W|

### 実装上の制約と警告

MFPへのアクセスは常に**8ビット単位（バイトアクセス）**で行わなければならない。X68000のデータバス構造上、偶数番地を含むワード（16ビット）アクセスを行った場合、上位バイトには不定のデータが乗り、意図しない動作や誤診を招く。また、レジスタマップ上で定義されていない未使用ビットをリードした場合は常に '0' が返されるが、ライト時には将来的な互換性維持のため常に '0' を書き込むべきである。

この物理的な窓口を通じて、最初に制御すべきなのがシステムの状態を監視するGPIPである。

--------------------------------------------------------------------------------

## 3. GPIP (汎用I/Oポート) の仕様と信号定義

GPIPは、X68000というハードウェアの状態をリアルタイムで監視するための「感覚器官」である。電源スイッチの押下や映像信号の同期タイミングといった重要なステータスが、この8ビットに集約されている。

### GPIP信号定義一覧

X68000では、GPIP5以外のすべてのビットが入力ポートとして定義されている。

|   |   |   |   |
|---|---|---|---|
|ビット|信号名|内容|論理|
|**GPIP 7**|H-SYNC|CRTC 水平同期信号|1:同期期間|
|**GPIP 6**|CIRQ|CRTC 割り込み（ラスタ）要求|0:要求あり|
|**GPIP 5**|-|未使用（外部でH固定）|常に1|
|**GPIP 4**|V-DISP|CRTC 垂直表示期間信号|1:表示期間, 0:帰線期間|
|**GPIP 3**|FMIRQ|FM音源 割り込み要求|0:要求あり|
|**GPIP 2**|POW SW|本体前面電源スイッチ|0:ON（通常時1）|
|**GPIP 1**|EXPON|拡張スロット/リモート電源ON信号|0:ON|
|**GPIP 0**|ALARM|RTC アラーム信号|0:アラーム発生|

### 制御レジスタの役割

- **AER (Active Edge Register)**: 各ビットの割り込みを「0→1」の変化で発生させるか（1設定）、「1→0」の変化で発生させるか（0設定）を決定する。
- **DDR (Data Direction Register)**: ポートの入出力を設定する。X68000では全ビットを '0' に設定し、入力専用として運用する。

### システム設計の合理性

GPIPが出力として利用されないのは、主要な制御線がすでに専用LSI（CRTCやI/Oコントローラ等）によって確立されているためである。MFPのGPIPは他LSIの「実行結果」を受け取る受動的な監視役に徹することで、メインCPUが周辺回路の状態を最小限の負荷で把握できる、堅牢な設計となっている。

これら信号の「変化」を契機として起動するのが、割り込み制御メカニズムである。

--------------------------------------------------------------------------------

## 4. 割り込み制御メカニズムとベクタ設定

MFPは16種類の割り込み要因を管理し、CPUに対して整然とリクエストを送る「調停役」を果たす。優先順位は固定されており、GPIP7が最優先、GPIP0が最低優先度となる。

### 割り込み制御フローとレジスタ

1. **IERA/B (Enable)**: 割り込み発生の許可/禁止。
2. **IPRA/B (Pending)**: 割り込み要求が発生し、CPUが未受付の状態（1で保留）。
3. **ISRA/B (In-Service)**: CPUが割り込みを受付、現在処理中の状態。
4. **IMRA/B (Mask)**: CPUへの出力を抑止。IERと異なりIPRへのフラグ立ちは阻止しない。

### ベクタ生成ルールとオフセット

MFPがCPUに渡す8ビットのベクタ番号は、VRレジスタの上位4ビットと、要因ごとに決まる下位4ビット（オフセット）の合成で決定される。

|   |   |   |   |
|---|---|---|---|
|割り込み要因|下位4bit (Hex)|割り込み要因|下位4bit (Hex)|
|**GPIP 0 (ALARM)**|0000 ($0)|**タイマ B**|1000 ($8)|
|**GPIP 1 (EXPON)**|0001 ($1)|**送信エラー**|1001 ($9)|
|**GPIP 2 (POW SW)**|0010 ($2)|**送信バッファ空**|1010 ($A)|
|**GPIP 3 (FMIRQ)**|0011 ($3)|**受信エラー**|1011 ($B)|
|**タイマ D**|0100 ($4)|**受信バッファフル**|1100 ($C)|
|**タイマ C**|0101 ($5)|**タイマ A**|1101 ($D)|
|**GPIP 4 (V-DISP)**|0110 ($6)|**GPIP 6 (CIRQ)**|1110 ($E)|
|**GPIP 5 (未使用)**|0111 ($7)|**GPIP 7 (H-SYNC)**|1111 ($F)|

例えば VR=`$40` の場合、タイマAのベクタは `$4D` となる。

### 実装上のガイドライン

VRのビット3（Sビット）で終了通知（EOI）モードを選択する。

- **ソフトウェアEOI (S=1)**: ハンドラ内で手動によりISRをクリアする。多重割り込みの制御を厳密に行う必要があるシステムに適している。
- **自動EOI (S=0)**: CPUにベクタを渡した瞬間にISRが自動クリアされる。処理は簡略化されるが、同一要因による割り込みの再入に注意を要する。

## 5. マルチファンクション・タイマ (Timer A-D) の徹底解剖

MFPは4つの独立したタイマ（A-D）を備えている。これらは4MHzのXTALクロックをソースとし、システムの定時処理やボーレート生成を担う。

### タイマ機能の比較

|   |   |   |
|---|---|---|
|特徴|Timer A / B|Timer C / D|
|**外部入力端子**|あり (TAI, TBI)|なし|
|**外部出力端子**|あり (TAO, TBO)|あり (TCO, TDO)|
|**動作モード**|全モード（ディレイ、パルス幅、イベント）|ディレイモードのみ|
|**主な用途**|V-DISP計測(A) / USARTクロック(B)|カーソル点滅(C) / OSシステム用(D)|

### 動作モードと制御

各タイマは、入力クロックをプリスケーラ（1/4, 1/10, 1/16, 1/50, 1/64, 1/100, 1/200）で分周し、データレジスタ（1〜255）でカウントダウンする。

- **リセット機能**: TACR/TBCRのビット4を '1' にすることで、タイマ出力を強制的に 'L' レベルに設定できる。これは初期化時に出力状態を確定させる際に有効である。

### システム制約の分析

タイマBはキーボード通信速度に直結しており、その設定変更は入力系統の遮断を意味する。一方で、タイマAは外部入力(TAI)にV-DISP信号が接続されているため、ラスタスクロールや垂直帰線期間に同期した処理のトリガーとして極めて汎用性が高い。タイマC/Dは出力端子を持つものの、X68000では主に内部ディレイ用としてOSに管理されている。

--------------------------------------------------------------------------------

## 6. USART (シリアルポート) とキーボード通信

MFP内蔵のUSARTは、X68000では「キーボード専用通信路」として固定されている。

### ボーレート算出の理論と実測

X68000の標準通信仕様は **2400bps または 4800bps / 非同期 / 8bit / パリティなし** である。この速度はタイマBの出力周波数に完全に依存する。

- **Human 68K 設定値**: `TBCR = $01` (1/4分周), `TBDR = $0D` (13)
- **計算式**: 4,000,000 \text{Hz} / 4 / 13 = 76,923 \text{Hz}
- **ボーレート**: USARTを `1/16モード` (UCR bit7=1) で運用した場合、 76,923 / 16 \approx 4,807 \text{bps} (実質 4800bps)

ソース上の記述には2400bpsとの混在も見られるが、レジスタの初期設定値 `$0D` に基づけば4800bps運用が実態である。USART制御時にはこの「タイマBとの密結合」を常に意識しなければならない。

### エラー検知と回復プロセス

RSR（レシーバステータス）にてオーバーラン(OE)やフレーミングエラー(FE)を監視する。エラー発生時は、RSRをリードすることでフラグをクリアし、必要に応じてレシーバを再イネーブル（REビットの操作）することで通信を復旧させる。

## 7. Human 68Kにおける初期設定値と制御の要諦

標準OSであるHuman 68Kが設定するMFPの状態は、アプリケーションがハードウェアを「横取り」する際のベースラインとなる。

### Human 68K 初期設定値一覧

|   |   |   |
|---|---|---|
|レジスタ|設定値|意味|
|**AER**|`$06`|GPIP1, 2の立ち上がりエッジ設定|
|**DDR**|`$00`|全GPIPを入力に設定|
|**VR**|`$40`|ベクタ番号を 40-4F に配置（ソフトウェアEOI）|
|**TACR**|`$08`|Timer A：イベントカウントモード（V-DISP同期）|
|**TBCR**|`$01`|Timer B：1/4プリスケーラ（USARTクロック供給）|
|**TCDCR**|`$77`|Timer C/D：1/200プリスケーラ|
|**TBDR**|`$0D`|Timer B：分周比13 (Baud Rate 4800相当)|

### ハードウェア直接制御のチェックリスト

1. **割り込みの保護**: 設定変更中は `IMR` をクリアし、不要な割り込みを確実にマスクすること。
2. **フラグの完全クリア**: 新たな処理を開始する前に、`IPR` および `ISR` の該当ビットに '0' を書き込み、保留中の要求を消去すること。
3. **バイトアクセスの厳守**: 常に奇数番地をターゲットとし、`move.b` 命令を使用すること。
4. **タイマ書き込みのタイミング**: `TACR/TBCR` の変更はタイマがアクティブでない状態（または停止中）に行うのが原則である。

### 既存システムとの共存戦略

Human 68K環境下で独自の定期処理を実装する場合、タイマAのイベントカウント（V-DISP同期）を利用するのが最も安全である。OSが管理するタイマC/Dを操作する場合は、元の設定値と割り込みベクタを保存し、終了時に必ず復元（ハンドラ・チェイン）することで、システム全体の崩壊を回避できる。

本資料に示した仕様を理解することは、X68000という「生きているハードウェア」を自在に操るための第一歩である。
