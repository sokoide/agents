# X68000 MFP (MC68901)

MFP（マルチファンクションペリフェラル MC68901）は、カウンタ/タイマ、シリアルポート、汎用I/Oポートなどを1つのLSIの中に組み込んだデバイスである。X68000ではCRTCからの割り込みや電源ONの要因判別、キーボードとのインタフェースなどに使用している。

---

## 1. MFPの各機能の割り付け

### 内部ブロック図

MFP内部は4つのタイマ、4チャンネルのシリアルポート、8ビット分の汎用I/Oポートを持っている。X68000での接続状態の概略を以下に示す。

```
                 4MHz
                  |
     MC68901      |
     +------------+---------------+
     | タイマA   | --- TAO             | ----> CRTC V-DISP信号
     | TAI <---- | ---- (未使用)       |
     | タイマB   | --- TBO             |
     | TBI <---- | ---- (未使用)       |
     | タイマC   | --- TCO             | ----> (AER)
     | タイマD   | --- TDO             | ----> (未使用)
     |           |                     |
     | USART     | RC (受信クロック)   |
     | シリアル  | TC (送信クロック)   |
     | 入力      | (送信レディ:未使用) |
     | 出力      | キーボード          |
     |           |                     |
     | GPIP0 <-- | RTC ALARM           |
     | GPIP1 <-- | 拡張EXPON           |
     | GPIP2 <-- | POWER SW            |
     | GPIP3 <-- | FM音源IRQ           |
     | GPIP4 <-- | VCC                 |
     | GPIP5 <-- | VCC ($FF)           |
     | GPIP6 <-- | CRTC IRQ            |
     | GPIP7 <-- | CRTC H-SYNC         |
     +----------------------------+
```

### 各タイマの用途

4つのタイマのうち、**タイマBはキーボードとの通信を行うシリアルポートの伝送速度を決めるクロック**として使用されている。設定や動作モードを変更するとキーボードが使えなくなってしまう。その他のタイマはハード的には用途は指定されていない。Human68kではタイマCをカーソルの点滅やFDDのモータ停止タイミングの作成などに、タイマDはVersion 2.0以降で疑似マルチタスク動作用として使用している。

タイマAの制御線であるTAI入力にはCRTCが出力するV-DISP信号が入っている。V-DISP信号の変化した回数をカウントして、一定回数ごとにCPUに割り込みをかけることや、V-DISP信号の周期測定も可能である。

MFPのシリアルポートはいくつもの動作モードを持っているが、X68000では接続する相手がキーボードに限定されているため、キーボードの通信モードに合わせた設定で使うことになる。

GPIP0〜GPIP7の8本のI/Oポートのうち、未使用となっているGPIP5以外はすべて入力ポートとして使われている。GPIP5は外部でHレベルに固定されているため、リードすると$FFが読み出される。

## 2. MFPのレジスタ一覧

MFPのレジスタは$E88001〜$E8802F番地に配置されている。レジスタはすべて8ビット長であるため、奇数番地（ワードアクセス時の下位バイト）のみとなる。MFPのレジスタのうち、GPIPの制御に使われるのが$E88001〜$E88005、割り込み制御に使われるのが$E88007〜$E88017、タイマ制御用が$E88019〜$E88025、USART（シリアルポート）制御用が$E88027〜$E8802Fとなっている。

MFPのレジスタは、ステータス入力や一部の特殊な機能を持たせたもの以外は基本的にすべてライト/リードとも可能となっている。未使用ビットはリードすると"0"が読み出される。ライト時は1、0のいずれでもかまわないが、他のLSIなどと同様"0"にしておくのが普通である。

### レジスタマップ

| 種別         | アドレス | 略称  | レジスタ名                       | R/W  |
| :--- | :--- | :--- | :--- | :--- |
| **GPIP**     | $E88001  | GPIP  | 汎用I/Oレジスタ                  | R/W  |
|              | $E88003  | AER   | アクティブエッジレジスタ         | R/W  |
|              | $E88005  | DDR   | データ方向レジスタ               | R/W  |
| **割り込み** | $E88007  | IERA  | 割り込みイネーブルレジスタA      | R/W  |
|              | $E88009  | IERB  | 割り込みイネーブルレジスタB      | R/W  |
|              | $E8800B  | IPRA  | 割り込みペンディングレジスタA    | R/W* |
|              | $E8800D  | IPRB  | 割り込みペンディングレジスタB    | R/W* |
|              | $E8800F  | ISRA  | 割り込みインサービスレジスタA    | R/W* |
|              | $E88011  | ISRB  | 割り込みインサービスレジスタB    | R/W* |
|              | $E88013  | IMRA  | 割り込みマスクレジスタA          | R/W  |

|              | $E8802F  | UDR   | USARTデータレジスタ              | R/W  |

IPR, ISRなどのステータスレジスタは、'0'を書き込むことでフラグをクリアする。

### 実装上の制約と警告

MFPへのアクセスは常に**8ビット単位（バイトアクセス）**で行わなければならない。X68000のデータバス構造上、偶数番地を含むワード（16ビット）アクセスを行った場合、上位バイトには不定のデータが乗る。また、常に奇数番地をターゲットとし、`move.b`命令を使用すること。

## 3. GPIP（汎用I/Oポート）

GPIPの制御に関係するレジスタのビット配置を以下に示す。GPIP、AER、DDRはどれも同じビット配置である。

```
bit 7     6     5     4     3     2     1     0
+-----+-----+-----+-----+-----+-----+-----+-----+
| H-SYNC | CIRQ  | V-DISP | FMIRQ | POWSW | EXPON | ALARM |
| GPIP7  | GPIP6 | GPIP5  | GPIP4 | GPIP3 | GPIP2 | GPIP1 | GPIP0 |
+-----+-----+-----+-----+-----+-----+-----+-----+
```

### 信号定義一覧

| ビット | 信号       | 説明                                                                  |
| :--- | :--- | :--- |
| GPIP0  | RTC ALARM  | RTCのALARM信号、1: "H"、0: "L"（アラーム発生）                        |
| GPIP1  | EXPON      | 拡張スロットのEXPON信号、1: "H"（通常）、0: "L"（電源ON状態）         |
| GPIP2  | POWER SW   | 本体前面の電源スイッチ、1: 電源スイッチOFF、0: 電源スイッチON（通常） |
| GPIP3  | FMIRQ      | FM音源ICの割り込み要求信号、1: 要求なし、0: 要求中                    |
| GPIP4  | V-DISP     | CRTCのV-DISP信号、1: "H"（垂直表示期間）、0: "L"（帰線期間）          |
| GPIP5  | （未使用） | 外部でHレベルに固定、常に"1"が読み出される                            |
| GPIP6  | CIRQ       | CRTCの割り込み（ラスタ割り込み）要求、1: 要求なし、0: 要求中          |
| GPIP7  | H-SYNC     | CRTCのH-SYNC信号、1: "H"（水平同期期間）、0: "L"                      |

### 電源ON要因の判別

X68000は、本体上面の電源スイッチによる通常のON/OFFのほか、拡張スロットのリモート電源ON信号や本体背面のリモート端子、RTCのALARM信号などで電源を入れることができる。このため、ソフトウェアで電源がONとなった要因を判定できるようになっている。

- **GPIP2**が"0" → 本体正面の電源スイッチがON
- **GPIP1**が"0" → 拡張スロットやリモート端子が"H"（電源ON状態）
- **GPIP0**が"0" → RTCのALARM信号がALARM状態

### 制御レジスタの役割

- **GPIPレジスタ**：各信号の状態がそのまま読み出せる。X68000ではすべて入力として使うためリードのみ。
- **AERレジスタ**：各信号ごとに割り込みを発生させる変化方向を設定する。"1"で"0"→"1"の変化で、"0"で"1"→"0"の変化で割り込みを発生する。
- **DDRレジスタ**：各信号ごとに入力/出力のいずれで使うかを設定する。"0"で入力、"1"で出力。X68000では全ビット"0"を設定。

> **注意**: AERのGPIP3とGPIP4のビットは、タイマの制御信号の割り込みのエッジ設定と兼用になっている。詳細はタイマのセクションを参照。

## 4. 割り込み制御

MFPは16種類の割り込み要因を持っており、8ビット×2本のレジスタに分類されている。割り込みの優先順位は固定で、GPIP7（H-SYNC）がもっとも高く、GPIP0（ALARM）がもっとも低い。

### IERA, IPRA, ISRA, IMRA（$E88007, $E8800B, $E8800F, $E88013）

```
bit 7     6     5     4     3     2     1     0
+-----+-----+-----+-----+-----+-----+-----+-----+
| H-SYNC | CIRQ  | Tmr A | Tmr B | MPSC  | MPSC | MPSC  | MPSC |
| GPIP7  | GPIP6 |       |       | TXErr | TXE  | RXErr | RXF  |
+-----+-----+-----+-----+-----+-----+-----+-----+
```

| ビット | 割り込み要因                         |
| :--- | :--- |
| bit7   | CRTC H-SYNC（水平同期信号）          |
| bit6   | CRTCの割り込み（ラスタ割り込み）要求 |
| bit5   | タイマAからの割り込み要求            |
| bit4   | タイマBからの割り込み要求            |
| bit3   | MPSC送信エラー割り込み               |
| bit2   | MPSC送信バッファエンプティ割り込み   |
| bit1   | MPSC受信エラー割り込み               |
| bit0   | MPSC受信バッファフル割り込み         |

### IERB, IPRB, ISRB, IMRB（$E88009, $E8800D, $E88011, $E88015）

```
bit 7     6     5     4     3     2     1     0
+-----+-----+-----+-----+-----+-----+-----+-----+
| V-DISP | GPIP4 | Tmr C | Tmr D | FMIRQ | POWSW | EXPON | ALARM |
| GPIP5  | GPIP4 |       |       | GPIP3 | GPIP2 | GPIP1 | GPIP0 |
+-----+-----+-----+-----+-----+-----+-----+-----+
```

| ビット | 割り込み要因                    |
| :--- | :--- |
| bit7   | CRTC V-DISP（垂直表示期間）信号 |
| bit6   | GPIP4                           |
| bit5   | タイマC                         |
| bit4   | タイマD                         |
| bit3   | FM音源ICからの割り込み要求      |
| bit2   | 本体前面の電源スイッチ          |
| bit1   | EXPON信号                       |
| bit0   | RTCのALARM信号                  |

### VR（ベクタレジスタ）$E88017

```
bit 7   6   5   4   3   2   1   0
+---+---+---+---+---+---+---+---+
| V7 | V6 | V5 | V4 |  |  | S |  |
+---+---+---+---+---+---+---+---+
  割り込みベクタの上位4bit     S
  (下位4bitはMFP内の割り込み    |
   要因によって決まる)          |
                        S=1: ソフトウェアEOIモード
                           (インサービスレジスタ有効)
                        S=0: オートEOIモード
                           (インサービスレジスタ無効)
```

### 各レジスタの機能

#### IERA/IERB（割り込みイネーブルレジスタ）

割り込み発生の許可/禁止を制御する。"1"で許可、"0"で禁止。**IERが"0"の間の割り込み要求は完全に無視される。**

#### IPRA/IPRB（割り込みペンディングレジスタ）

割り込み要求が保留されていることを示す。MFPが割り込み要求を認識すると"1"となり、CPUに該当する割り込みベクタが渡されたときに"0"に復帰する。IERによって割り込みが禁止されても"0"になる。CPUが該当ビットに"0"を書き込むことでもクリアできる。

#### ISRA/ISRB（インサービスレジスタ）

該当する割り込みが処理中であることを示す。MFPからCPUにベクタが渡されると該当ビットが"1"になり、CPUが該当ビットに"0"を書き込む（EOI: End Of Interrupt）と"0"になる。

自動EOIモード（S=0）では、CPUにベクタを渡した時点で即座に"0"に復帰する。**ISRA/ISRBが"0"でIPRA/IPRBが"1"になると、MFPは再度割り込み要求を行う**ため、自動EOIモードでは同一要因の連続割り込みが可能となる。

#### IMRA/IMRB（割り込みマスクレジスタ）

割り込みのマスク制御を行う。"1"で割り込み発生可。IERとよく似ているが、**両者の違いは重大である**：

- **IER = "0"**: 割り込み要求の発生元を抑え、要求自体を完全に無視する
- **IMR = "0"**: MFPからのCPUへの割り込み要求出力だけを抑える。IERが"1"であれば、MFPは要求を受け取りIPRにフラグを立てる。その後IMRが"1"になった時点でCPUに割り込みを発生する

IERは要求の発生元を抑えるもの、IMRは出力だけを抑えるものと考えるとわかりやすい。

### ベクタ生成ルール

MFPがCPUに渡す8ビットのベクタ番号は、VRレジスタの上位4ビットと、要因ごとに決まる下位4ビット（オフセット）の合成で決定される。

| 割り込み要因    | 下位4bit  | 割り込み要因     | 下位4bit  |
| :--- | :--- | :--- | :--- |
| GPIP 0 (ALARM)  | 0000 ($0) | タイマ B         | 1000 ($8) |
| GPIP 1 (EXPON)  | 0001 ($1) | 送信エラー       | 1001 ($9) |
| GPIP 2 (POW SW) | 0010 ($2) | 送信バッファ空   | 1010 ($A) |
| GPIP 3 (FMIRQ)  | 0011 ($3) | 受信エラー       | 1011 ($B) |
| タイマ D        | 0100 ($4) | 受信バッファフル | 1100 ($C) |
| タイマ C        | 0101 ($5) | タイマ A         | 1101 ($D) |
| GPIP 4 (V-DISP) | 0110 ($6) | GPIP 6 (CIRQ)    | 1110 ($E) |
| GPIP 5 (未使用) | 0111 ($7) | GPIP 7 (H-SYNC)  | 1111 ($F) |

例えば VR=`$40` の場合、タイマAのベクタは `$4D` となる。

## 5. タイマ

MFPはタイマAからタイマDまでの4つのタイマを持っている。タイマCとDはディレイモード動作しかできないが、タイマAとBはパルス幅測定モードやイベントカウントモードも利用可能である。

### タイマ機能の比較

| 特徴         | Timer A / B                              | Timer C / D                       |
| :--- | :--- | :--- |
| 外部入力端子 | あり (TAI, TBI)                          | なし                              |
| 外部出力端子 | あり (TAO, TBO)                          | あり (TCO, TDO)                   |
| 動作モード   | 全モード（ディレイ、パルス幅、イベント） | ディレイモードのみ                |
| 主な用途     | V-DISP計測(A) / USARTクロック(B)         | カーソル点滅(C) / OSシステム用(D) |

### 動作モードの概略

```
【ディレイモード】タイマA/B/C/D
XTAL(4MHz) -> プリスケーラ -> 8bitカウンタ -> +2 -> TAO/TBO/TCO/TDO
                |  |
             選択可能                  割り込み発生
    (+4,+10,+16,+50,+64,+100,+200)

【パルス幅測定モード】タイマA/B
TAI/TBI -> スタート/ストップ制御
XTAL(4MHz) -> プリスケーラ -> 8bitカウンタ -> +2 -> TAO/TBO
                |  |
             選択可能                  割り込み発生

【イベントカウントモード】タイマA/B
TAI/TBI -> 8bitカウンタ -> TAO/TBO
              |
          割り込み発生
```

### ディレイモード

ディレイモードは任意の周波数を得たり、一定周期で割り込みを発生する用途に使用される。カウンタにプリスケーラの出力を接続し、カウンタが$01になると次のクロックパルスでタイマ割り込みを発生させ、タイマ出力端子の状態を反転させる。カウンタにはタイマデータレジスタの値が自動的に再ロードされる。

プリスケーラの分周比は 1/4, 1/10, 1/16, 1/50, 1/64, 1/100, 1/200 から選択できる。最終的な分周比はプリスケーラとタイマデータレジスタの積になる。

> **計算例**: プリスケーラ 1/100、データレジスタに200をセット → カウンタ出力 = 4MHz / (200×100) = 200Hz。出力端子はこの周期で反転するため、出力周波数はさらに半分の100Hzとなる。

### パルス幅測定モード

パルス幅測定モードは、TAI/TBI入力が指定されたレベルである期間だけタイマが動くようにすることで、入力信号のパルス幅の測定を行える。**タイマAとBだけで利用可能**。X68000ではTBI端子はHレベルに固定されているため、実際に利用できるのはタイマAだけである。

測定完了時にCPUに割り込みをかけることができる。"H"と"L"のいずれのレベルでスタートするかはAERのGPIP4（タイマA）/ GPIP3（タイマB）の設定で決まる。**発生する割り込みはタイマAがGPIP4、タイマBがGPIP3の割り込みになる**。つまり、タイマAをパルス幅測定モードにするとGPIP4の変化による割り込みが行えなくなる。

> **AERの反転に注意**: 通常、AERが"1"だと"0→1"の変化で割り込みだが、パルス幅測定モードでは"1"にすると"H→L"の変化で割り込みとなる。

再測定時は、タイマデータレジスタに値を再書き込みする際、制御入力がアクティブでないことを確認すること。アクティブ時に書き込むと正しい値がロードされない場合がある。

### イベントカウントモード

イベントカウントモードは、TAI/TBIの入力をクロックとして8ビットカウンタを動作させる（プリスケーラは使用不可）。入力のどちら方向の変化でカウントするかはAERのGPIP4/GPIP3で行う。

X68000では**TAI入力にV-DISP信号が接続されており、Human68kはタイマAをイベントカウントモードで使用している**。

| 1110   | パルス幅測定モード     | +100         |
| 1111   | パルス幅測定モード     | +200         |

**Rビット（ビット4）**: タイマ出力（TAO/TBO）リセット。"1"にすると出力ピンを強制的に"L"に設定する。クリア後、次のカウントアップパルスが来れば通常動作どおり出力は反転される。タイマ出力が"L"の状態から動作開始させたいときに使用する。

### TCDCR（タイマC&Dコントロールレジスタ）$E8801D

```
bit 7   6   5   4   3   2   1   0
+---+---+---+---+---+---+---+---+
|  | CC1 | CC0 |  |  | DC2 | DC1 | DC0 |
+---+---+---+---+---+---+---+---+
|  | <--タイマC--> |  | <--タイマD--> |
|  | 動作モード    |  | 動作モード    |
+---+---+---+---+---+---+---+---+
```

| 設定値 | モード         | プリスケーラ |
| :--- | :--- | :--- |
| 000    | タイマストップ | -            |
| 001    | ディレイモード | +4           |
| 010    | ディレイモード | +10          |
| 011    | ディレイモード | +16          |
| 100    | ディレイモード | +50          |
| 101    | ディレイモード | +64          |
| 110    | ディレイモード | +100         |
| 111    | ディレイモード | +200         |

### タイマデータレジスタ

それぞれのタイマごとに1本ずつ、値のリード/ライトを行うタイマデータレジスタが用意されている。タイマはカウントパルスが入るたびに減少し、$01になると次のパルスでタイマデータレジスタに設定した値が自動的に再ロードされる。

## 7. USART（シリアルポート）

MFP内蔵のUSARTは、全二重の同期/非同期通信をサポートする汎用シリアルインタフェースである。X68000ではキーボードと接続するため、キーボードの伝送モード（**非同期、2400bps、スタートビット1ビット、データ8ビット、パリティなし、ストップビット1ビット**）に合わせる。また、USARTの伝送クロックはタイマBから得るようにしており、クロックモードも1/16以外は選択できない。

### ボーレート算出

X68000のキーボード通信速度は2400bpsである。この速度はタイマBの出力周波数に依存する。

- **Human68K設定値**: `TBCR = $01`（プリスケーラ +4）、`TBDR = $0D`（13）
- **タイマB出力周波数**: 4MHz / 4 / 13 = 76,923 Hz（オーバーフロー率）
- **TBO出力周波数**: 76,923 / 2 = 38,462 Hz（トグルで÷2）
- **ボーレート**: 38,462 / 16 = 2,404 bps ≈ 2400 bps

UCRのCLKビットを"1"（1/16モード）に設定し、USARTはスタートビットを自動検出してデータビットの中心をサンプリングする。

### SCR（SYNCキャラクタレジスタ）$E88027

同期転送モード時、SCRに設定されたデータが受信されるまで待ち続ける。送信時にアンダーラン状態になると自動的にSCRのキャラクタが送信される。**X68000では非同期モードで使用するため、SCRは無視してかまわない。**

### UCR（USARTコントロールレジスタ）$E88029

```
bit 7   6   5   4   3   2   1   0
+---+---+---+---+---+---+---+---+
| CLK | WL1 | WL0 | ST1 | ST0 | PE | E/O |
+---+---+---+---+---+---+---+---+
```

| ビット  | 機能                  | 設定                                                                         |
| :--- | :--- | :--- |
| CLK     | 送受信速度            | 1: 1/16、0: 同一。X68000では"1"固定                                          |
| WL1/WL0 | データ長              | 00: 8bit、01: 7bit、10: 6bit、11: 5bit。X68000では"00"                       |
| ST1/ST0 | モード/ストップビット | 00: 同期、01: 非同期/1/1、10: 非同期/1/1.5\*、11: 非同期/1/2。X68000では"01" |
| PE      | パリティ有効/無効     | 1: 有効、0: 無効。X68000では"0"                                              |
| E/O     | 偶数/奇数パリティ     | 1: 偶数、0: 奇数                                                             |

\* CLK="1"のときのみ設定可

### RSR（レシーバステータスレジスタ）$E8802B

```
bit 7   6   5   4   3   2   1   0
+---+---+---+---+---+---+---+---+
| BF | OE | PE | FE | F/S | M/C | SS | RE |
|    |    |    |    | orB | IP  |    |    |
+---+---+---+---+---+---+---+---+
```

| ビット   | 名称                            | 機能                                                                                          |
| :--- | :--- | :--- |
| BF       | バッファフル                    | 1: 受信バッファにデータあり。UDRを読み出すと"0"になる                                         |
| OE       | オーバーランエラー              | 1: データを引き取らないまま次のデータが入った。新データは捨てられる。RSRを読み出すと"0"になる |
| PE       | パリティエラー                  | 1: パリティ不一致。正常データ受信で"0"に復帰                                                  |
| FE       | フレーミングエラー              | 1: ストップビットが見つからない（非同期のみ）。正常データ受信で"0"に復帰                      |
| F/S or B | ファウンド/サーチまたはブレーク | 同期: ワードサーチ、非同期: ブレーク検出（"0"が続く状態）                                     |
| M/CIP    | マッチ/文字処理中               | 同期: SYNC一致、非同期: スタートビット検出時に"1"                                             |
| SS       | シンクロナスストップ            | SYNCキャラクタ受信の可否（"0"でSYNCデータはバッファに入らない）                               |
| RE       | レシーバイネーブル              | 1: イネーブル、0: ディセーブル。"0"にすると各ステータスビットも"0"になる                      |

### TSR（トランスミッタステータスレジスタ）$E8802D

```
bit 7   6   5   4   3   2   1   0
+---+---+---+---+---+---+---+---+
| BE | UE | AT | END | B | H | L | TE |
+---+---+---+---+---+---+---+---+
```

| ビット | 名称                     | 機能                                                                                |
| :--- | :--- | :--- |
| BE     | バッファエンプティ       | 1: 送信バッファ空。UDRに書き込むと"0"になる                                         |
| UE     | アンダーランエラー       | 1: データ未書き込みのまま最後のデータ送信完了。TE操作またはTSR読み出しでクリア      |
| AT     | オートターンアラウンド   | 1: 送信完了後自動的にレシーバをイネーブル。送信完了で自動的に"0"になる              |
| END    | エンディング             | 送信中またはディセーブル時に"1"                                                     |
| B      | ブレーク                 | 1: 送信データラインをブレーク状態に（非同期のみ）。"1"の間はBEが"1"になることはない |
| H/L    | High/Low                 | トランスミッタディセーブル時のライン状態                                            |
| TE     | トランスミッタイネーブル | 1: イネーブル                                                                       |

H/Lビットの動作：

| H/L | 動作                                                                 |
| :--- | :--- |
| 00  | ハイインピーダンス                                                   |
| 01  | Lレベル                                                              |
| 10  | Hレベル                                                              |
| 11  | ループバックモード（自己診断。送信データが内部で受信に折り返される） |

通常、H/Lには"10"を設定しておく。

### UDR（USARTデータレジスタ）$E8802F

データの受け渡しを行うレジスタ。書き込んだデータは送信ラインから送出され、受信データはこのレジスタを通してCPUに渡される。

### 初期値の読み方

| レジスタ  | 設定値 | 意味                                                        |
| :--- | :--- | :--- |
| **AER**   | $06    | bit1,2を"1"に設定（GPIP1,2の立ち上がりエッジで割り込み）    |
| **DDR**   | $00    | 全GPIPを入力に設定                                          |
| **VR**    | $40    | ベクタ番号を$40-$4Fに配置。Sビット(bit3)は不定              |
| **TACR**  | $08    | Timer A：イベントカウントモード（V-DISP同期）               |
| **TBCR**  | $01    | Timer B：ディレイモード +4プリスケーラ（USARTクロック供給） |
| **TCDCR** | $77    | Timer C/D：ともにディレイモード +200プリスケーラ            |
| **TBDR**  | $0D    | Timer B：分周比13（Baud Rate ≈ 2400bps相当）                |
| **UCR**   | $88    | CLK=1(1/16)、WL=00(8bit)、ST=01(非同期1/1)、PE=0、E/O=0     |

### ハードウェア直接制御のチェックリスト

1. **割り込みの保護**: 設定変更中は`IMR`をクリアし、不要な割り込みを確実にマスクする。
2. **フラグの完全クリア**: 新たな処理を開始する前に`IPR`および`ISR`の該当ビットに"0"を書き込み、保留中の要求を消去する。
3. **バイトアクセスの厳守**: 常に奇数番地をターゲットとし、`move.b`命令を使用する。
4. **タイマ書き込みのタイミング**: TACR/TBCRの変更はタイマがアクティブでない状態で行うのが原則。

### 既存システムとの共存戦略

Human68K環境下で独自の定期処理を実装する場合、タイマAのイベントカウント（V-DISP同期）を利用するのが最も安全である。OSが管理するタイマC/Dを操作する場合は、元の設定値と割り込みベクタを保存し、終了時に必ず復元（ハンドラ・チェイン）することで、システム全体の崩壊を回避できる。
