# 数値演算プロセッサ（MC68881）

X68000 では MC68881 はコプロセッサではなく「周辺 I/O デバイス」としてマッピングされている。68000 は F-Line 命令を実行できないため、CIR（コプロセッサ・インタフェース・レジスタ）を通じて手動でバスサイクルを管理する必要がある。

## 配置とアクセス

- **XVI 以前**: 拡張ボード（CZ-6BP1 等）経由。**XVI 以降**: 本体基板専用ソケットに直接実装。
- **ベースアドレス**: `$E9E000`（2 枚目は `$E9E080`）。システム I/O エリアのため Supervisor モード必須。

## 内部レジスタ

- **FP0〜FP7**: 80 ビット長のデータレジスタ x8。内部演算は常に 80 ビット拡張精度。
- **FPCR**: 丸め精度（32/64/80bit）と丸めモード（近似/ゼロ方向/無限大方向）。
- **FPSR**: コンディションコード（N/Z/I/NAN）、商バイト、例外ステータス/アクルード例外。
  - 5 つの IEEE 例外ビット: IOP（無効操作）, OVFL（溢れ）, UNFL（不足）, DZ（零除算）, INEX（不正確）。
  - アクルード例外は演算結果の累積（OR）を保持。一連の計算後のバルクチェックに適する。
- **FPIAR**: 最後に実行された命令のメモリアドレス。

## データフォーマット

| 型 | 略称 | サイズ | 備考 |
| --- | --- | --- | --- |
| バイト整数 | B | 8bit | 転送時に拡張精度へ変換 |
| ワード整数 | W | 16bit | 転送時に拡張精度へ変換 |
| ロングワード整数 | L | 32bit | 転送時に拡張精度へ変換 |
| 単精度実数 | S | 32bit | 指数8bit, 小数23bit |
| 倍精度実数 | D | 64bit | 指数11bit, 小数52bit |
| 拡張精度実数 | X | 96bit | 内部80bit。メモリ上は上位16bitがゼロ |
| パック型10進数 | P | 96bit | 17桁仮数の BCD 形式 |

入力に関わらず内部演算は常に 80 ビット拡張精度。中間計算の精度落ちを極限排除。

## CIR メモリマップ（ベース: `$E9E000` / `$E9E080`）

| オフセット | レジスタ | 属性 | 機能 |
| --- | --- | --- | --- |
| $00 | 応答 (Response) | R | FPU の状態・要求（プリミティブ） |
| $02 | コントロール (Control) | W | アボート（実行中断） |
| $04 | セーブ (Save) | R | 内部状態読み出し（コンテキストスイッチ用） |
| $06 | リストア (Restore) | R/W | 内部状態復旧 |
| $0A | コマンド (Command) | W | 演算命令（オペレーションワード） |
| $0E | コンディション (Condition) | W | 条件判定要求 |
| $10 | オペランド (Operand) | R/W | データ転送ポート |
| $14 | レジスタ選択 | R | FMOVEM 時の対象レジスタリスト |
| $18 | 命令アドレス | W | ホストの PC を FPU に通知 |
| $1C | オペランドアドレス | W | 転送先アドレス（通常未使用） |

## 通信プロトコル

### CA（Come Again）ビット

応答 CIR のビット 15。CA=1 の間、ホストは「Master Sync」ループを維持し FPU の要求に応じ続ける。

### 例外ベクタ

| ベクタ | エラー内容 |
| --- | --- |
| $0B | F-Line（未定義命令） |
| $31 | Inexact（丸め不正確） |
| $32 | Zero Div（ゼロ除算） |
| $33 | Underflow（指数部不足） |
| $34 | Operand Error（無効な演算対象） |
| $35 | Overflow（指数部溢れ） |
| $36 | Signaling NAN（NAN を演算対象にした） |

## 命令実行シーケンス（OPクラス別）

### 1. 内部レジスタ間演算（OPクラス 000）

- コマンド CIR に命令書き込み → 応答 CIR が `$0802`（アイドル）になるまで待機。

### 2. 外部データ → レジスタ（OPクラス 010）

- コマンド書き込み → 応答 CIR がデータ転送要求（`$950x` 等）を示すまで待機 → オペランド CIR にデータ書き込み。

### 3. レジスタ → 外部（OPクラス 011）

- コマンド書き込み → 応答 CIR が `$8900`（再読込）の間待機 → データ読み出し要求（`$B10x` 等）でオペランド CIR から読み出し。
- **Dynamic K-Factor 時**: 変換前にメインレジスタから K ファクタを読み出し、オペランド CIR に書き込んでから変換。

### 4. FMOVEM（複数レジスタ転送）

- コマンド書き込み → レジスタ選択 CIR（$14）読み出し → 応答 `$810C`（転送要求）→ 対象レジスタ数分、オペランド CIR で連続転送。

## コンテキストスイッチ

`FSAVE`（セーブ CIR 読み出し）で取得するフレームサイズ:

| 状態 | フレームID | サイズ |
| --- | --- | --- |
| 初期 | $0018 | 24 bytes |
| IDLE | $XX18 | 24〜36 bytes |
| BUSY | $XXB4 | 180 bytes |

命令前例外（Pre-Instruction Exception）発生時は、コントロール CIR に値を書き込んでアボート → システム例外ハンドラへ遷移。

## 実装上の注意

- CIR へのアクセスは必ず Supervisor モード。
- 応答 CIR 監視ループ: `while(!(response_cir & 0x8000))` で CA ビットをチェック。
- CIR ポインタには必ず `volatile` を付与。
- デバッガ `db.x` の `me` コマンドはワードアクセスするため FPU の STR 操作には使えない。
