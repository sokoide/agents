# X68000 数値演算プロセッサ（MC68881）

68881は、CPUの苦手とする浮動小数点演算処理を専門に行うLSIである。X68000では、数値演算プロセッサとして68000ファミリーのMC68881をオプションで搭載できる。実装形態は、XVI以前の機種では拡張ボード（CZ-6BP1）、XVI以降は本体内部の専用ソケットに挿入するが、ソフトウェアから見た場合にはまったく同じである。

68881はもともと68020と直結しコプロセッサとして使うのが本来の姿だが、68000などの他のCPUと接続することもできる。X68000では68881を**周辺I/Oデバイスとしてアクセス**するようにしている。68020の場合はCPUがコプロセッサ専用命令を自動的に処理するが、X68000ではこの操作をすべてソフトウェアで行う必要がある。

---

## 1. 68881の内部レジスタ

68881の内部レジスタへのアクセスは、演算命令やデータ転送命令を利用して行われるものであり、CPUから見てあるアドレスに直接配置されるものではない。

```
bit 79                                                          bit 0
+---+-----------------------------------------------------------+
|   |                                                           | FP0
+---+-----------------------------------------------------------+
|   |                                                           | FP1
+---+-----------------------------------------------------------+
|   |                                                           | FP2
+---+-----------------------------------------------------------+
|   |                                                           | FP3
+---+-----------------------------------------------------------+
|   |                                                           | FP4
+---+-----------------------------------------------------------+
|   |                                                           | FP5
+---+-----------------------------------------------------------+
|   |                                                           | FP6
+---+-----------------------------------------------------------+
|   |                                                           | FP7  80bit浮動小数点データレジスタ
+---+-----------------------------------------------------------+
|                       32bit                                   | FPCR コントロールレジスタ
+---------------------------------------------------------------+
|                       32bit                                   | FPSR ステータスレジスタ
+---------------------------------------------------------------+
|                       32bit                                   | FPIAR 命令アドレスレジスタ
+---------------------------------------------------------------+
```

### FPn（浮動小数点データレジスタ）

FP0からFP7の8本の80ビット長のレジスタ。68881の演算処理はこれらのレジスタを使用して行う。8本のレジスタはまったく同等であり、あるレジスタだけが特殊なものとして扱われることはない。CPUのデータレジスタ（D0-D7）に相当する。

### FPCR（コントロールレジスタ）

```
bit 31                          16 15         8 7         0
+-------------------------------+-----------+---+-------+---+
|           予約                |例外イネーブル|モードコントロール|
+-------------------------------+-----------+---+-------+---+

bit 15                                            bit 8
+---+---+---+---+---+---+---+---+
|BSUN|SNAN|OPERR|OVFL|UNFL| DZ|INEX2|INEX1|
+---+---+---+---+---+---+---+---+
 BSUN : シグナリングNAN
 SNAN : シグナリングNAN（代替）
 OPERR: オペランドエラー
 OVFL : オーバフロー
 UNFL : アンダフロー
 DZ   : ゼロによる割り算
 INEX2: 不正確なオペレーション
 INEX1: 不正確な10進入力

bit 7         4 3         0
+---+---+---+---+---+---+---+---+
|PREC  | RND  |予約|PREC|
+---+---+---+---+---+---+---+---+

PREC: 00=拡張精度, 01=単精度, 10=倍精度
RND:  00=近似値に丸め, 01=ゼロ方向
      10=マイナス無限大方向, 11=プラス無限大方向
```

FPCRのPRECビットによる丸め精度の変更は、拡張精度で演算できない他の計算機との互換性維持用であり、演算速度も低下するため、通常は拡張精度以外を指定する必要はない。

### FPSR（ステータスレジスタ）

```
bit31     24 23     16 15      8 7       0
+----------+----------+----------+----------+
|条件コード|  商      |例外ステータス|累積例外|
+----------+----------+----------+----------+

bit31   28 27   24
+---+---+---+---+---+---+---+---+
| N | Z | I | NAN|
+---+---+---+---+---+---+---+---+
  負  ゼロ 無限大 数字でない/アンオーダ

bit23         16
+---+---+---+---+---+---+---+---+
| S |     商 (下位7bit)   |
+---+---+---+---+---+---+---+---+

bit15                         8
+---+---+---+---+---+---+---+---+
|BSUN|SNAN|OPERR|OVFL|UNFL|DZ|INEX2|INEX1|
+---+---+---+---+---+---+---+---+

bit7       3 2     0
+---+---+---+---+---+---+---+---+
|IOP|OVFL|UNFL|DZ|INEX|
+---+---+---+---+---+---+---+---+
```

条件コードバイトは演算命令の終わりでセットされる。商バイトはFMOD命令やFREM命令でセットされる。累積例外はこれまでの演算結果のORを保持する。

## 2. データフォーマット

68881の内部演算は常に拡張精度で行われており、外部とのやりとりでは自動的に型変換が行われる。外部から転送されたデータは必ず内部で拡張精度に変換され、外部に転送するときには指定された型に変換される。

### サポートデータ型

| 型 | 略称 | ビット長 | 説明 |
|----|------|----------|------|
| バイト整数 | B | 8 | 符号付き整数 |
| ワード整数 | W | 16 | 符号付き整数 |
| ロングワード整数 | L | 32 | 符号付き整数 |
| 単精度実数 | S | 32 | IEEE単精度浮動小数点 |
| 倍精度実数 | D | 64 | IEEE倍精度浮動小数点 |
| 拡張精度実数 | X | 96(80) | IEEE拡張精度（内部では80ビット） |
| パック形式10進 | P | 96 | 17桁BCD実数 |

略称はアセンブラで使用する（例：`FMOVE.S`で単精度実数の転送）。

### 実数フォーマットの比較

| 項目 | 単精度 | 倍精度 | 拡張精度 |
|------|--------|--------|----------|
| S（符号） | 1 | 1 | 1 |
| e（指数） | 8 | 11 | 15 |
| バイアス | +127 | +1023 | +16383 |
| f（小数部） | 23 | 52 | 63 |
| 合計ビット | 32 | 64 | 96(80) |
| 正規化数表現 | (-1)^S×1.f×2^(e-bias) | (-1)^S×1.f×2^(e-bias) | (-1)^S×1.f×2^(e-bias) |
| 正規化数最大 | 3.4×10^38 | 1.8×10^308 | 6×10^4931 |
| 正規化数最小 | 1.2×10^-38 | 2.2×10^-308 | 8×10^-4933 |
| 非正規化数最小 | 1.4×10^-45 | 4.9×10^-324 | 9×10^-4951 |

単精度・倍精度では仮数の整数部分（常に1）を省略し小数部分だけを格納する。拡張精度では64ビット仮数部の最上位ビットが整数部分。指数部はバイアス値をオフセットとして格納する（例：単精度で2^0は$7F、2^1は$80）。

### 特殊な実数データ

| 指数 | 仮数 | 種類 |
|------|------|------|
| 最小値 < 指数 < 最大値 | 任意のビットパターン | 正規化数（通常） |
| 0 | All 0以外の任意のビットパターン | 非正規化数（アンダフロー限界付近） |
| 0 | 0 | ゼロ |
| 最大値 | 0 | 無限大 |
| 最大値 | All 0以外の任意のビットパターン | NAN（Not A Number） |

NANは無限大-無限大など数学的に意味を持たない演算の結果として返される。

### 68881内部の演算フォーマット

精度落ちを防ぐため、仮数部分は拡張精度の64ビットに対して67ビットとなっている。

```
指数(17bit) | 整数bit | 小数部分(63bit) | G | R | S
                                ガード  丸め  スティッキー
```

## 3. CIR（コプロセッサインタフェースレジスタ）

68881とX68000のコミュニケーションをとるためのレジスタ。68020が規定したCIR規定に基づく。シャープ純正ボードCZ-6BP1では、標準設定**$E9E000〜$E9E01F**、ピン設定変更で**$E9E080〜$E9E09F**（Human68kドライバは標準設定のみサポート）。

### CIRメモリマップ

| オフセット | レジスタ | アクセス |
|------------|----------|----------|
| +$00 | 応答CIR (Response) | R |
| +$02 | コントロールCIR (Control) | W |
| +$04 | セーブCIR (Save) | R |
| +$06 | リストアCIR (Restore) | R/W |
| +$08 | オペレーションワードCIR | R |
| +$0A | コマンドCIR (Command) | W |
| +$0C | （予約） | |
| +$0E | コンディションCIR (Condition) | W |
| +$10 | オペランドCIR (Operand) | R/W |
| +$14 | レジスタ選択CIR (Register Select) | R |
| +$18 | 命令アドレスCIR (Instruction Address) | W |
| +$1C | オペランドアドレスCIR | W |

### 各CIRの機能

**応答CIR**：68881の動作状態やホストCPUへのサービス要求（プリミティブ）を示す。いつでも読み出し可能。ホストCPUはこの値をチェックしながら68881と同期をとる。

**コントロールCIR**：68881へのアボート指示に使用する。書き込みが行われると68881は実行中の処理をただちに中止し、ペンディングされている例外をすべてクリアした後アイドル状態に復帰する。

**セーブCIR**：68881の内部状態を読み出すためのレジスタ。FSAVE命令用。マルチタスクOSでタスク切替時に内部状態をメモリにセーブする用途。

**リストアCIR**：FRESTORE命令用。セーブCIRで最初に返されたデータ（ステートフレーム）を書き込む。

**コマンドCIR**：ホストCPUが68881に命令を書き込むために使用する。各種演算命令やデータ転送命令はすべてこのレジスタへの書き込みで開始される。

**コンディションCIR**：条件チェック（等しい、大きい、小さい等）を行うために使用する。

**オペランドCIR**：ホストCPUと68881の間のデータ転送に使用する。

**レジスタ選択CIR**：FMOVEM命令時に68881からホストCPUにレジスタマスクを渡すために使用する。ホストCPUは渡されたデータの"1"の数をカウントすることで転送するレジスタ数を知る。

**命令アドレスCIR**：応答CIRのPCビットがセットされているときにPCの値を渡す。

**オペランドアドレスCIR**：68881はこのCIRを使用しない。アクセスはすべて無視される。

### 一般フォーマット


## 5. ホストCPUとのコミュニケーションシーケンス

### 5.1 内部レジスタ間演算/データ転送

1. CPUが命令をコマンドCIRに書き込む
2. 68881は応答CIRに$0900/$4900（内部処理実行中）をセット
3. CPUが応答CIRを読み出すと68881は演算・丸め処理を実行
4. 命令終了後、応答CIRを$0802（アイドル状態）として次の要求を待つ

### 5.2 外部データからレジスタへの転送・演算

1. コマンドCIRに命令を書き込む
2. 応答CIRに実効アドレス評価/データ転送プリミティブがセットされる
3. オペランドCIR経由でデータを転送
4. 応答CIRを$0900とし、データ型変換・演算・丸め処理を実行

### 5.3 レジスタから外部へのデータ転送

1. コマンドCIRにコマンドを書き込む
2. 応答CIRとして$8900/$C900（再読み出し要求）が返る
3. 68881内部でデータ変換を開始
4. 変換終了後、応答CIRはデータ転送プリミティブに変わる
5. オペランドCIRからデータを読み出す

### 5.4 コントロールレジスタの転送

基本的に浮動小数点データ転送と同様だが、データ変換が不要な分簡単。

### 5.5 複数レジスタの転送（FMOVEM）

転送するレジスタの指定を命令中に含める場合（スタティックレジスタリスト）とパラメータとして与える場合（ダイナミックレジスタリスト）の2通りがある。

### 5.6 条件付き命令

コマンドCIRではなくコンディションCIRを使用する。応答CIRとして返るのはヌルプリミティブで、条件成立時は$0800、不成立時は$0801。

### 5.7 FSAVE/FRESTORE

セーブCIRの読み出しから開始。返ってくる値：

- **$0018**（NULLステート）：初期状態
- **$0118**（カムアゲイン）
- **$XX18**（アイドルステート）：24バイト
- **$XXB4**（ビジーステート）：180バイト

### 5.8 例外処理

命令前例外、BSUN例外、Fラインエミュレータ例外等の処理手順がある。いずれの場合も、ホストCPUはコントロールCIRへの書き込みを行い、68881をアイドル状態に復帰させる。

## 7. サンプルプログラム

### 7.1 ROM内データの読み出し（円周率）

```c
#include "stdio.h"

union DAT {
    unsigned char cdat;
    unsigned short sdat;
    unsigned int idat;
    float fdat;
    double ddat;
} dat;

struct CIR {
    unsigned short response;
    unsigned short control;
    unsigned short save;
    unsigned short restore;
    unsigned short operation_word;
    unsigned short command;
    unsigned short reservel;
    unsigned short condition;
    unsigned int operand;
    unsigned short register_select;
    unsigned short reserve2;
    unsigned int instruction_address;
    unsigned int operand_address;
};

volatile struct CIR *cir = (struct CIR *)0xe9e000;

void main()
{
    SUPER(0);

    wait_copro(0x0802);   /* FMOVECR #0, FP0 */
    cir->command = 0x5c00;
    wait_copro(0x0802);
    cir->command = 0x6400;   /* FMOVE.S FP0, xxx */
    wait_copro(0xb104);
    dat.idat = cir->operand;
    wait_copro(0x0802);

    printf("PAI = %f\n", dat.fdat);
}

void wait_copro(unsigned short response)
{
    unsigned int i, ack;
    for (i = 0; i < 0x20; i++) {
        ack = cir->response;
        printf("%04x\n", ack);
        if ((ack & 0xbfff) == response)
            break;
    }
    printf("----\n");
}
```

実行結果：`PAI = 3.141993`

## 8. 実装上のガイドライン

1. **スーパーバイザモード**: CIRへのアクセスは必ずSUPER(0)で行う。
2. **CAビット監視**: 応答CIRのbit15（CA）が"1"の間はFPUの要求に応じ続ける。
3. **volatileの強制**: CIRアドレスポインタには必ず`volatile`を付与し、コンパイラ最適化による読み飛ばしを防ぐ。
4. **バスサイクル管理**: 68000はFPU専用命令を直接実行できないため、68020のマイクロコードが行うコプロセッサプロトコルをソフトウェアでシミュレートする。
