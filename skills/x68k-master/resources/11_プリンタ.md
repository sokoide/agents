# X68000 プリンタインタフェース

## 1. X68000におけるプリンタ制御の戦略的役割

X68000のアーキテクチャにおいて、プリンタインタフェース（セントロニクス準拠）は単なる外部出力端子以上の価値を有しています。本機の設計思想である「パーソナルワークステーション」を支える強力なI/Oサブシステムにおいて、このインタフェースはCPU（68000）のメモリアドレス空間に直接マッピングされた、極めてレスポンスの良い8ビット並列出力ポートとして機能します。

特に、シャープ独自のI/Oコントローラ（IOC）ASICを介した制御は、現代のAIによるレトロハードウェア制御において決定的な意味を持ちます。OSの抽象化レイヤー（IOCS等）をバイパスし、I/Oレジスタを直接叩くことで、ナノ秒単位のタイミング制御を伴う「物理層のハンドシェイク」が可能になります。これは、プリンタのみならず、外部マイクロコントローラや自作ハードウェアとの高速データ通信路として本機を再定義できることを意味します。本マニュアルでは、この「生」のハードウェア制御を完遂するための論理的仕様を詳解します。

## 2. 信号ラインと制御タイミングの物理的解析

プリンタポートの制御は、STB（ストローブ）、BUSY（ビジー）、ACK（アック）という3つの信号線の相互作用によって成立します。X68000ではこれらがIOC ASIC内部のステートマシンおよびレジスタに直結されています。

### 主要信号の役割と論理特性

|   |   |   |   |
|---|---|---|---|
|信号名|方向|役割|制御上の重要性|
|**STB**|出力|データ確定通知|**負論理（Active Low）**。Lパルスを送ることでプリンタ側にデータバスの値が有効であることを伝える。AIによるソフトウェア制御の場合、パルス幅の管理が肝となる。|
|**BUSY**|入力|受信不可通知|**正論理**。プリンタが処理中、またはエラー時にHighとなる。この信号がLow（Ready）であることを確認せずに送信を開始するとデータ消失を招く。|
|**ACK**|入力|受信完了通知|**負論理**。データ受理後の「立ち下がりエッジ」が割り込みトリガーとなる。ハンドシェイクの完了を物理的に担保する信号。|

**エンジニアの視点：** X68000のSTB信号は完全にソフトウェア生成であるため、AIは転送速度（ボーレート）を自在に可変できます。ただし、セントロニクス規格準拠のためには、最低でも0.5µs（500ns）のパルス幅を確保しなければなりません。

## 3. プリンタ関連レジスタの構成とアドレスマップ

X68000の周辺機器レジスタは、16ビットバスの下位8ビット側に配置されており、すべて**奇数アドレス**へのバイトアクセス（`move.b`命令等）が必須となります。

### 3.1 プリンタデータポート ($E8802D)

- **役割:** 送信データのラッチ。
- **仕様:** 8ビットの書き込み専用ポート。ここにデータを書き込むと、物理的なDATA 0-7ラインに値が保持されます。ただし、これだけでは出力は確定せず、次のストローブ操作を待機します。

### 3.2 プリンタ制御／ステータスポート ($E8802F)

このレジスタはビットごとに役割が厳格に分離されています。

- **Bit 0 (STB出力):** 書き込み用。`1`を書き込むと物理信号がLow（Active）に、`0`でHighに戻ります。負論理の実装に注意してください。
- **Bit 1 (ACKステータス):** 読み出し用。現在のACK信号の論理状態をリアルタイムで監視します。
- **Bit 2 (割り込みマスク):** 書き込み用。`1`でACK信号による割り込みを許可、`0`で禁止します。

### 3.3 割り込みベクタレジスタ ($E9C003)

- **役割:** IOC ASICが発生させる割り込みのベースベクタを設定します。
- **重要警告:** このレジスタは**FDDおよびHDDコントローラと共有**されています。プリンタのベクタを変更すると、ストレージデバイスのベクタも同時にシフトするため、システム全体の整合性を保つ細心の注意が必要です。

## 4. プログラマブルなデータ転送プロトコルの実装

AIがドライバレベルで実装すべき、標準的なポーリング（ステータスチェック）方式のデータ転送フローを以下に定義します。

1. **BUSY状態の監視:** `$E8802F`を読み出し、BUSY信号がLow（受信可能）になるまでループ待機。
2. **データのアサート:** 送信データをデータポート `$E8802D` に `move.b` で書き込む。
3. **ストローブ信号の発火（Lパルス開始）:** `$E8802F` の `bit 0` に `1` をセットし、STBをLowに落とす。
4. **物理タイミングの確保（Wait挿入）:** 規格上の最低パルス幅（500ns）を確保するため待機。10MHz機であれば最低5クロック、16MHz機であれば最低8クロック相当のディレイが必要です。
    - _実装例:_ 複数の `NOP` 命令、または無関係なレジスタのダミー読み出しを数回実行。
5. **ストローブ信号の解除（Lパルス終了）:** `$E8802F` の `bit 0` に `0` を書き込み、STBをHighに復帰させる。
6. **転送完了の最終確認:** ACK信号がLowに変化したことを確認し、次のデータの処理へ移行する。

## 5. 割り込み機構を利用した高度な非同期制御

効率的なバックグラウンド転送を実現するには、IOC ASICの割り込み要因判定を利用します。ACK信号の立ち下がりエッジにより、IOCはCPUへ割り込みを要求します。

### 割り込みベクタ構造と要因オフセット

IOC（$E9C003）に設定したベースベクタに対し、以下の要因オフセットが加算されてCPUに渡されます。

```text
[IOC 割り込み要因構造]
設定されたベースベクタ番号 + 要因コード
---------------------------------------
要因コード (Binary):
00 : FDC (フロッピーディスクコントローラ)
01 : FDD (フロッピーディスクドライブ本体)
10 : HD  (ハードディスク)
11 : PRN (プリンタ - ACK信号検出)
```

プリンタ（要因コード `11`）はオフセット3に相当します。例えば、ベースに `$60` を設定した場合、プリンタ割り込みはベクタ番号 `$63` で発生します。

**高度な制御戦略:** 割り込みハンドラ内で次のデータの書き込みとSTBパルスの生成を行うことで、CPUのメイン処理を止めることなく、オープンコレクタ駆動のハンドシェイクサイクルを完遂できます。

## 6. 結論：AIによるハードウェア制御の最適化

X68000のプリンタポートを「汎用高速パラレルI/O」として活用することは、本機のポテンシャルを現代に蘇らせる最良の手段です。AIがこのポートを制御するロジックを構築する際は、以下の技術的留意点を厳守してください。

- **奇数番地アクセスの徹底:** すべてのレジスタ操作はバイトサイズ（`.b`）で行い、16ビットバスのバスエラーを回避すること。
- **500nsの壁:** STBパルス幅が不足すると、プリンタ側LSIのサンプリングレートを下回り、文字化けや通信不能の原因となる。クロック周波数に応じた適切な `NOP` 数を算定すること。
- **共有レジスタの排他制御:** $E9C003（ベクタ設定）および $E8802F（マスク設定）を操作する際は、FDD等の他のI/O状態を破壊しないよう、読み出し・変更・書き出し（Read-Modify-Write）を原則とすること。
- **負論理の反転思考:** ソフトウェア上の `1` 書き込みが物理的な電位の `Low` を意味する「Active Low」の論理構造を前提に論理回路を構築すること。

これらの低レイヤー・プロトコルを完全に掌握することで、X68000は現代のAI演算結果をリアルタイムに出力する、極めて精緻な物理インタフェースへと進化します。
