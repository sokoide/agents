# X68000 プリンタインタフェース

## 1. プリンタインタフェースの概要

X68000では、プリンタとしてセントロニクスインタフェース準拠のものが接続できるようになっています。

PA0〜PA7は8ビットのデータライン、STROBEはプリンタに対してデータの引き取りを要求するもの、BUSYはプリンタが次のデータを受け付ける準備ができているかどうかを示す信号です。X68000ではBUSY信号をI/Oコントローラに入力し、ビジー状態からレディ状態への変化時（次のデータ転送が行えるようになったとき）に割り込みを発生することもできるようになっています。

セントロニクスインタフェースを採用しているプリンタは、ほとんどがプリンタ上のSELECTスイッチや紙切れ状態、データ引き取り完了パルスなどを個別の信号線で出力しているのですが、X68000ではこれらの信号は使用していません。X68000側ではBUSY信号によってビジー（データ引き取り不可）か、レディ（データ引き取り可）かをチェックし、一定時間たってもビジーのままであれば、タイムアウトエラーとして処理することになります。

### プリンタインタフェースブロック図

```
                    LS273
$E8C001 ──────→ CLRO
 アクセス
 PA7 ─┐
 PA6 ─┤
 PA5 ─┤
 PA4 ─┤────→ プリンタコネクタ
 PA3 ─┤
 PA2 ─┤
 PA1 ─┤
 PA0 ─┘

$E8C003 ビット0 ──→ STROBE

$E9C001 ビット5 ──→ BUSY (ALS244経由入力)
```

## 2. プリンタ制御タイミング

プリンタ制御のタイミング例を以下に示します。STROBE信号は'1'、プリンタからのBUSY信号は'1'になっているものとします。なお、ここでの'1'や'0'はポートに書き込んだり、ポートから読み出されるデータを指します。

まず、BUSY信号が'1'になっている（プリンタがレディ状態である）ことを確認して、PA0〜PA7にプリンタに送りたいデータをセットします。次にSTROBE信号を'0'にすると、プリンタがデータを引き取るため、BUSYが'0'になりますので、これを見てSTROBEを'1'に復帰させます。プリンタ側でデータが引き取られ、次のデータ引き取りの準備ができると、BUSYが'1'に復帰します（必要ならば、この時点で割り込みを発生させることもできます）。X68000側はBUSYが'1'に復帰したのを見て、次のデータの送出を行うわけです。

BUSY信号が'0'になっている期間はプリンタ側の都合で決まるものであり、どの程度になるかはわかりません。また、実際はBUSY信号をチェックせずSTROBEを0、1に連続して変化させて、BUSYが'1'になるのを待つという方法がよくとられているようです。

### プリンタ制御タイミング例

```

STROBE    "1" |    |                          |    |
              |    |                          |    |
        "0"   └────┘                          └────┘
                                                         "High"
BUSY          |                                    |
              |                                    |
        "0"   └────────────────────────────────────┘
                                                         "High"
PA0-PA7                                   [データ]

$E8C001: データをセット
$E9C001: I/Oコントローラのポートで読み出される状態

```

## 3. プリンタ関連ポート

プリンタインタフェースに関連するポートの一覧を示します。プリンタに送るデータは$E8C001にセットし、STROBE信号を$E8C003で制御します。

| アドレス | アクセス | ポート名 |
| :--- | :--- | :--- |
| $E8C001 | W | プリンタデータポート |
| $E8C003 | W | プリンタストロープポート |
| $E9C001 | R/W | 割り込み信号ステータス |
| $E9C003 | R/W | 割り込みベクタレジスタ |

プリンタからのBUSY信号がビジー状態からレディ状態へ変化したとき割り込みを発生させることができるほか、$E9C001の上位ビットでステータスとして読み出すこともできるようになっています。

### プリンタデータポート（$E8C001）

プリンタに送出するデータをセットします。STROBEを操作する前に、このポートにデータをセットしておくようしてください。

### プリンタストロープポート（$E8C003）

プリンタストロープポートのビット配置を以下に示します。最下位ビットがSTROBE信号の制御ビットとなっており、'1'にするとSTROBE信号がHighレベル、'0'にするとLowレベルになります。

```

bit 7     6     5     4     3     2     1     bit 0
[  |  |  |  |  |  |  | STROBE ]

```

- 1：STROBE信号を'High'レベルにする
- 0：STROBE信号を'Low'レベルにする

## 4. 割り込み制御

### 割り込み信号ステータス（$E9C001）

割り込み信号ステータスポートのビット配置を示します。上位4ビットは各割り込み要因が発生しているか否かを示すビット、下位4ビットは割り込みマスクレジスタに書き込まれた値がそのまま読み出されます。

```

bit 7     6     5     4     3     2     1     bit 0
[ FDC  | FDD  | PRT  | HDD  | EOI  | PRTI | FDDI | FDCI | HDDI ]
  INT    INT    INT    INT    E.     EN     EN     EN     EN

```

| ビット | 内容 |
| :--- | :--- |
| 7 | FDC割り込み許可中 |
| 6 | FDD割り込み許可中 |
| 5 | プリンタBUSY信号（1: 'Low'=READY状態、0: 'High'=BUSY状態） |
| 4 | HDD割り込み許可中 |
| 3 | HDD割り込みが発生している |
| 2 | EOI |
| 1 | プリンタ割り込み許可中 |
| 0 | プリンタ割り込みが発生している |

このうち、ビット5がプリンタの割り込み要求状態、すなわちBUSY信号の状態を示し、ビット0がプリンタからの割り込みがマスクされているか否かを示しています。

プリンタの割り込みは、BUSY信号が'0'から'1'に変化したとき発生します。割り込みマスクレジスタによってプリンタ割り込みの発生が禁止されていても、ビット5はBUSY信号の状態が反映されますので、これを使って割り込みを使用せずプリンタの制御を行うこともできます。

### 割り込みマスク（$E9C001書き込み時）

I/Oコントローラが管理している各割り込み要因ごとに、CPUへの割り込み要求を行うかどうかを決めるレジスタです。

プリンタの割り込み制御はビット4で行います。このビットが'1'なっているとプリンタ割り込みの発生が許可、'0'なっていれば禁止になります。

```

bit 7     6     5     4     3     2     1     bit 0
[ HDDI | FDCI | FDDI | PRTI |  |  |  |  ]
   EN     EN     EN     EN

```

| ビット | 内容 |
| :--- | :--- |
| 7 | HDD割り込み許可（1: 許可、0: 禁止） |
| 6 | FDC割り込み許可（1: 許可、0: 禁止） |
| 5 | FDD割り込み許可（1: 許可、0: 禁止） |
| 4 | プリンタ割り込み許可（1: 許可、0: 禁止） |

### 割り込みベクタ（$E9C003）

```

bit 7     6     5     4     3     2     1     bit 0
[      任意設定可 (6ビット)      | VECTOR(2ビット) ]

```

| 下位2ビット | 内容 |
| :--- | :--- |
| 11 | プリンタ割り込み |
| 10 | ハードディスク割り込み |
| 01 | FDD割り込み |
| 00 | FDC割り込み |

I/Oコントローラの割り込みベクタ設定レジスタは、割り込み発生時にCPUに与えるベクタ番号を設定します。上位6ビットは任意設定可能で、下位2ビットは割り込み要因によって自動的に変化するため、CPUからの設定は無効になります。プリンタからの割り込みが発生したときは、下位2ビットが11になったベクタ番号がCPUに渡されます。

---

## 5. 実装上のガイドライン

1. **奇数アドレスアクセス**: すべてのレジスタは16ビットバスの下位8ビット側に配置されているため、奇数アドレスへのバイトアクセス（`move.b`等）で操作する。
2. **STROBE極性**: STROBEは'1'でHigh、'0'でLow。データをセット後、STROBEを'0'に落としてプリンタにデータを引き取らせる。
3. **BUSY監視**: BUSYが'1'（レディ）になるのを確認してから次のデータを送る。
4. **割り込み利用**: BUSYの'0'→'1'変化で割り込みを発生可能。$E9C001のビット4を'1'にすると許可。
