# X68000 システム初期化と終了

ゲーム等のアプリケーションは、起動時に Human68k の標準機能を抑制し、終了時にすべて復元するライフサイクルを徹底する。

---

## 初期化シーケンス

### Supervisor モード移行

I/O領域へのアクセスに必須。最初に実行。

```c
/* C言語 (elf2x68k / XC) */
_iocs_super(0);  /* または B_SUPER(0) */
```

```
; アセンブラ
DOS  _SUPER($ff20)
```

### Human68k 標準機能の抑制

ゲーム中にシステム標準機能が動作すると支障が出るため、以下を抑制する。

| IOCS/DOS | 番号 | 処理 | 引数 |
| --- | --- | --- | --- |
| CONCTRL | $FF23 | カーソル非表示 | sp: 18 |
| CONCTRL | $FF23 | ファンクションキー非表示 | sp: 14, sp: 2 |
| MS_INIT | $70 | マウスカーソル初期化(消去) | なし |
| SKEY_MOD | $7D | マウス+ソフトキーボード使用禁止 | d1: 0 |
| TGUSEMD | $0E | 電卓使用禁止 | d1: 1, d2: 2 |

```
; 初期化例
    move    #18,-(sp)        ; カーソル非表示
    DOS    _CONCTRL
    addq.l    #2,sp

    move    #2,-(sp)
    move    #14,-(sp)        ; ファンクションキー消去
    DOS    _CONCTRL
    addq.l    #4,sp

    IOCS    _MS_INIT         ; マウスカーソル消去

    moveq    #0,d1
    IOCS    _SKEY_MOD         ; マウス+ソフトキーボード禁止

    moveq    #1,d1
    moveq    #2,d2
    IOCS    _TGUSEMD          ; 電卓禁止
```

### 画面モード設定

```c
/* 推奨: モード10 (256x256, 256色, 高解像度) */
_iocs_crtmod(10);

/* グラフィック画面表示ON — 忘れるとGVRAM描画が見えない */
_iocs_g_clr_on();

/* パレット設定 */
_iocs_gpalet(0x80000000 | index, rgb555);
```

### 白帯処理 (COPYキー / NMI)

COPYキーやINTERRUPTスイッチが押されるとゲームが中断される。例外ベクタを無害な処理に差し替える。

**COPYキー (trap #12)**

```
; 無効化
    pea    COPY_rte(pc)
    move    #32+12,-(sp)    ; trap #12
    DOS    _INTVCS($ff25)
    addq.l    #6,sp
    lea    vctCOPY(pc),a0
    move.l    d0,(a0)        ; 旧ベクタを保存

COPY_rte:
    rte                    ; 何もせず復帰

; 復元（終了時）
    move.l    vctCOPY(pc),-(sp)
    move    #32+12,-(sp)
    DOS    _INTVCS
    addq.l    #6,sp
```

**NMI / INTERRUPTスイッチ (trap #31)**

```
; 無効化
    pea    NMI_rte(pc)
    move    #31,-(sp)        ; NMI
    DOS    _INTVCS
    addq.l    #6,sp
    lea    vctNMI(pc),a0
    move.l    d0,(a0)

NMI_rte:
    move.b    #4,$e8e007    ; NMI検出リセット
    rte
```

**COPYキーの活用**: SHIFT+COPY で終了フラグを設定可能。

```
COPY_rte:
    btst.b    #0,$80e         ; SHIFTキー状態 ($000800ワーク)
    beq    @f
        bsr    QUIT            ; quit_flag = -1
    @@:
    rte
```

## 終了・復元シーケンス

初期化と**逆順**で復元する。

```
; 1. ベクタ復元
    move.l    vctCOPY(pc),-(sp)
    move    #32+12,-(sp)
    DOS    _INTVCS
    addq.l    #6,sp

    move.l    vctNMI(pc),-(sp)
    move    #31,-(sp)
    DOS    _INTVCS
    addq.l    #6,sp

; 2. 電卓許可
    moveq    #1,d1
    moveq    #1,d2
    IOCS    _TGUSEMD

; 3. マウス+ソフトキーボード許可
    moveq    #-1,d1
    moveq    #0,d2
    IOCS    _SKEY_MOD

; 4. 画面モード復元
    moveq    #3,d1
    IOCS    _B_COLOR

    clr    -(sp)
    move    #16,-(sp)        ; 768x512, グラフィック非表示
    DOS    _CONCTRL

    move    #17,(sp)         ; カーソル点滅開始
    DOS    _CONCTRL

    move    #14,(sp)         ; ファンクションキー表示
    DOS    _CONCTRL
    addq.l    #4,sp

; 5. キーバッファクリア
    clr    -(sp)
    DOS    _KFLUSH($ff0c)
    addq.l    #2,sp
```

## VSYNCタイミング詳細

MFP GPIP bit4 ($E88001) で垂直帰線期間を監視。

| パラメータ | 値 |
| --- | --- |
| 水平表示期間 | ~22.09 µs |
| 水平帰線期間 | ~9.66 µs |
| 水平周期 | ~31.75 µs (31.5 kHz) |
| 垂直表示期間 | 512スキャンライン (~16.25 ms) |
| 垂直帰線期間 | 56スキャンライン (~1.78 ms) |
| 垂直周期 | ~18 ms (**55.46 Hz**) |
| GPIP bit4 | 1=表示期間, 0=垂直帰線期間 |

**重要**: 高解像度モードのVSYNC周波数は55.46Hz。60Hzより約10%遅い。タイミング設計でこの差を考慮すること。

### VSYNC待ち (ポーリング)

```
vsync:
    btst.b    #4,$e88001    ; 帰線期間から抜けることを確認
    beq    vsync
    @@:
    btst.b    #4,$e88001    ; 表示期間が終わるまで待つ
    bne    @b
    rts
```

## データファイル I/O

### DOS ファイル操作

| DOS | 番号 | 機能 | 引数 | 戻り値 |
| --- | --- | --- | --- | --- |
| OPEN | $FF3D | ファイルオープン | sp: ファイル名ポインタ, sp: モード(0=読) | d0: ファイルハンドル (負=エラー) |
| READ | $FF3F | ファイル読み込み | sp: ハンドル, sp: バッファ, sp: バイト数 | d0: 読み込んだバイト数 |
| WRITE | $FF40 | ファイル書き込み | sp: ハンドル, sp: バッファ, sp: バイト数 | d0: 書き込んだバイト数 |
| CLOSE | $FF3E | ファイルクローズ | sp: ハンドル | なし |
| CREATE | $FF3C | ファイル作成 | sp: ファイル名ポインタ, sp: 属性 | d0: ファイルハンドル |

### C言語でのファイル読み込み

```c
FILE *fp = fopen(filename, "rb");
if (fp) {
    fread(buffer, 1, size, fp);
    fclose(fp);
}
```

D0負値 = エラーコード。エラー処理を省略しないこと。
