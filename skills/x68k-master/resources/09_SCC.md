# X68000 SCC (Z8530)

シリアル通信制御の中枢。RS-232C、マウス制御を担当。

## 配置とチャンネル

ベースアドレス: `$E98000`。8 ビットデバイスのため**奇数番地のみ**アクセス。

| チャンネル | アドレス | 役割 |
| --- | --- | --- |
| **Ch.A** | `$E98005` / `$E98007` | RS-232C 外部通信、マウス |
| **Ch.B** | `$E98001` / `$E98003` | キーボード |

割り込みレベル: **5**（MFP のレベル 6 に次ぐ高優先度）。

## 通信モード

- **非同期（Async）**: RS-232C・マウスの標準。スタート/ストップビット。
- **SDLC/HDLC**: フレーム同期、ゼロインサーション、CRC 生成をハードウェア処理。
- **符号化**: NRZ, NRZI, マンチェスター。
- **DPLL**: 外部クロックなしで受信データの遷移エッジからクロックを自律抽出。`WR14` で PCLK（4.9152MHz）にロック。

## ボーレート計算

ベースクロック（PCLK）= **4.9152 MHz**

```text
TC = PCLK / (2 × Mode × BaudRate) - 2
```

Mode はクロック倍率（1, 16, 32, 64）。非同期では通常 16。

TC の 16 ビット値を `WR12`（下位）と `WR13`（上位）に設定。

### 診断機能

`WR14` のビット操作で**ローカルループバック**（内部折り返し）と**オートエコー**（受信即時返送）が可能。通信開始前に I/O パスの健全性診断に利用。

## 割り込み制御

### 内部優先順位（デイジーチェーン）

- **Ch.A（高） > Ch.B（低）**
- 要因別: **受信（最高） > 送信 > 外部/ステータス（最低）**

### ベクタ設定

- ベクタ設定ポート: `$E98003` または `$E98007`（WR2）。
- **VIS（Vector Includes Status）**: `WR9` bit1 = 1 で、割り込み要因に応じてベクタ番号の下位ビットを SCC が自動書き換え。要因判定の分岐コードを省略可能。

### EOI（割り込み終了）

処理の最後に必ず `WR0` に **"Reset Highest IUS" コマンド（`0x38`）** を発行。これを行わないとレベル 5 の例外処理が終了せず、低優先度割り込みを受け付けなくなる。

## レジスタアクセス（インダイレクトアクセス）

コントロールポート（`$E98001` / `$E98005`）を介した 2 ステップ:

1. **ポインタ初期化**: コントロールポートに `0x00` を書き込み、ポインタを WR0 にリセット。
2. **レジスタ指定**: WR0 にアクセスしたいレジスタ番号（0〜15）を書き込み。
3. **データ操作**: 同一アドレスに設定値を書き込み（または読み出し）。

### リカバリタイム（必須）

SCC は MC68000（特に 16MHz の XVI）に対して内部同期速度が遅い。**連続アクセス間に最低約 3.5 PCLK サイクル以上の間隔が必要**。アクセス命令間に `NOP` を挿入して書き込み失敗を防ぐ。

## 主要レジスタ

| 宛先 | 区分 | 機能 |
| --- | --- | --- |
| **WR0** | コマンド | ポインタ指定、割り込みリセット（`0x38`: Reset Highest IUS） |
| **WR1** | 許可 | 受信/送信/外部割り込みの有効化 |
| **WR2** | ベクタ | 割り込みベクタのベース番号 |
| **WR3** | 受信 | 受信許可、データ長（5-8bit）、CRC |
| **WR4** | モード | ストップビット、パリティ、クロック倍率（x16推奨） |
| **WR5** | 送信 | 送信許可、DTR/RTS、データ長 |
| **WR9** | マスター | **Master Reset（`0xC0`）**, VIS 有効化（bit1） |
| **WR12/13** | BRG | ボーレートタイムコンスタント 下位/上位 |
| **WR14** | BRG/DPLL | BRG 有効化（bit0）、DPLL ソース選択 |
| **RR0** | ステータス | 送信バッファエンプティ、受信キャラクタ有り、DCD |
| **RR8** | データ | 受信データ読み出し実効ポート |

## 推奨初期化シーケンス

1. `WR9 = 0xC0`: ハードウェアリセット。全割り込み即時停止。
2. `WR4`: 通信モードとパリティ。
3. `WR3` / `WR5`: データ長と制御信号。
4. `WR12` / `WR13`: ボーレート。
5. `WR14`: ボーレートジェネレータ起動。
6. `WR2`: 割り込みベクタ。
7. `WR1` / `WR9`: 割り込み許可。
