# X68000 SCC (Serial Communication Controller) 

## 1. SCCの概要とX68000における戦略的役割

X68000におけるシリアル通信制御の中枢は、Zilog社のZ8530 SCC（Serial Communication Controller）が担っている。本LSIは単なる非同期通信のインターフェースではなく、システムI/O領域に緻密に組み込まれた、リアルタイム通信・入力デバイス制御の要である。

### 内部I/O領域における配置とバス構造

SCCはシステムI/O領域の `$E98000` ブロックに配置されている。MC68000の16ビットデータバスに対し、SCCは8ビットデバイスとして接続されており、データバスの低位バイト（D0〜D7）のみを使用する。したがって、全てのレジスタアクセスは**奇数番地**（E98001, E98003, E98005, E98007）に対して行う必要がある。偶数番地へのアクセスはバスエラー、あるいは意図しない動作を招くため、AIによる制御時にはアドレス演算に厳密さが求められる。

### 接続系統とチャンネル割当

ソースコンテキストの割り込み系統図およびシステム仕様に基づき、2つのチャンネルは以下の役割に固定されている。

- **チャンネルA（****E98005** **/** **E98007****）：** RS-232C外部通信およびマウス制御を担当。
- **チャンネルB（****E98001** **/** **E98003****）：** キーボードインターフェースを担当。

### AIによる直接制御の戦略的意義（So What?）

OS（Human68k）のIOCSコールは汎用性を優先するため、高ボーレート時のバッファオーバーランや、非標準プロトコルの取り扱いに制約がある。AIがスーパバイザモード（Supervisor Mode）でレジスタを直接操作することにより、OSの介在によるオーバーヘッドを排除し、独自のプロトコルスタックや、極小レイテンシのマウスエミュレーション、高速なMIDI通信などを自律的に実装することが可能となる。

## 2. 通信モードとデータ符号化の技術的解釈

Z8530は、ビット同期、バイト同期、非同期の各モードをサポートする高度な通信プロセッサである。

### データ通信モードの選択

1. **非同期（Async）モード：** X68000のRS-232Cおよびキーボード制御における標準。スタート/ストップビットを用いたキャラクタ同期を行う。
2. **SDLC/HDLCモード：** フレーム同期、ゼロインサーション、CRC生成をハードウェアレベルで処理する。高速データリンク構築に最適である。

### 符号化方式とエラー耐性

データ符号化（NRZ, NRZI, マンチェスター符号）の選択は、通信路のクロック抽出効率に直結する。

- **NRZ/NRZI：** 高速転送に向くが、同一ビットが連続すると同期が外れやすい。
- **マンチェスター符号：** ビット中央での遷移を保証するため、自己クロック復元能力が極めて高い。

### DPLL（Digital Phase Locked Loop）の活用

SCC内部のDPLLは、外部からクロック線（RxC）を供給されない環境下で、受信データの遷移エッジからクロックを自律抽出する。これにより、最小限の配線で同期通信と同等の信頼性を確保できる。AIは `$WR14`を操作し、DPLLをソースクロック（PCLK：4.9152 MHz）にロックさせることで、不安定な通信路を動的に補正すべきである。

## 3. ボーレート生成と同期制御のメカニズム

シリアル通信において、タイミングの不一致は決定的なデータ破損を招く。AIはSCC内部のボーレートジェネレータ（BRG）を完全に把握しなければならない。

### ボーレート計算論理

X68000のSCCに供給されるベースクロック（PCLK）は **4.9152 MHz** である。ボーレートを決定するタイムコンスタント（TC）は以下の数式で算出される。

TC = \frac{PCLK}{2 \times Mode \times BaudRate} - 2 ※ Mode はクロック倍率（1, 16, 32, 64倍）。通常、非同期通信では16倍を使用する。

この計算結果の16ビット値を `$WR12`（下位）と `$WR13`（上位）に設定する。計算ミスはフレーミングエラーの直撃を意味するため、浮動小数点演算ユニット（MC68881）等を用いた正確な算出が推奨される。

### 診断機能の運用

`$WR14` のビット操作により、**ローカルループバック**（内部折り返し）および**オートエコー**（受信データの即時返送）が可能である。AIは実通信開始前に、この機能を用いて自律的にI/Oパスの健全性診断（Self-Diagnostic）を実行し、転送パラメータの妥当性を検証しなければならない。

## 4. SCC割り込み処理と優先順位制御

X68000の割り込み階層において、SCCは**レベル5**に位置する。これはレベル6（MFP）に次ぐ高い優先度であり、DMAC（レベル3）や拡張スロット（レベル2/4）を圧倒する。

### 割り込み優先順位とデイジーチェーン

SCC内部には独自の優先順位（デイジーチェーン）が存在する。

- **チャンネルA（高） > チャンネルB（低）**
- 要因別：**受信（最高） > 送信 > 外部/ステータス（最低）**

AIはこの優先順位を理解し、特にチャンネルAの受信処理を最優先でディスパッチする必要がある。

### 割り込みベクタの設定

割り込みベクタ設定ポートは `$E98003` または `$E98007`（WR2）である。

- **VIS（Vector Includes Status）ビットの操作：** `$WR9` のビット1を `1` に設定することで、割り込み要因（受信、送信等）に応じてベクタ番号の下位ビットをSCCが自動的に書き換える。これにより、AIは要因判定の分岐コードを省略し、高速な割り込みハンドリングが可能となる。

### 割り込みの終了（EOI）

割り込み処理の最後には、必ず `$WR0` に対して **"Reset Highest IUS" コマンド (**`**0x38**`**)** を発行しなければならない。これを行わない限り、レベル5の例外処理が終了せず、CPUは他の低優先度割り込みを受け付けられなくなる。

## 5. SCCレジスタマップとプログラミング・インターフェース

### インダイレクトアクセスとポインタ・リセット

SCCのレジスタは、コントロールポート（E98001/5）を介した「ポインタ指定」と「データ書き込み」の2ステップでアクセスする。

1. **ポインタ初期化（必須）：** 制御開始前、内部ポインタの状態が不定であるリスクを排除するため、コントロールポートに `0x00` を書き込み、ポインタをレジスタ0（WR0）にリセットする。
2. **レジスタ指定：** WR0にアクセスしたいレジスタ番号（0〜15）を書き込む。
3. **実データ操作：** 同一アドレスに対して、目標とする設定値を書き込む（または読み出す）。

### 実装上の制約：バスサイクル・リカバリタイム

SCC（Z8530）は高速なMC68000（特に16MHz動作のXVIモデル）に比べ、内部同期速度が遅い。**連続するレジスタアクセス間には、最低でも数クロック（約3.5 PCLKサイクル以上）の間隔を空ける必要がある。**AIはコード実行時、アクセス命令間に `NOP` 指令を挿入することで、LSIのリカバリタイムを確保し、書き込みの失敗を防がなければならない。

### 主要レジスタ群要約表

|   |   |   |
|---|---|---|
|宛先ポインタ|区分|機能・ビット定義の要諦|
|**WR0**|コマンド|ポインタ指定、割り込みリセット（`0x38`: Reset Highest IUS）|
|**WR1**|許可|受信/送信/外部割り込みの有効化|
|**WR2**|ベクタ|割り込みベクタのベース番号設定|
|**WR3**|受信設定|受信許可、データ長（5-8bit）、CRC有効化|
|**WR4**|モード|ストップビット、パリティ、クロック倍率（x16推奨）|
|**WR5**|送信設定|送信許可、DTR/RTS制御、データ長|
|**WR9**|マスター|**Master Reset (**`**0xC0**`**)**, VIS有効化（`bit 1`）|
|**WR12/13**|BRG|ボーレートタイムコンスタントの下位/上位バイト|
|**WR14**|BRG/DPLL|BRG有効化（`bit 0`）、DPLLソース選択|
|**RR0**|ステータス|送信バッファエンプティ、受信キャラクタ有り、DCD信号|
|**RR8**|データ|受信データの読み出し実効ポート|

### 推奨初期化シーケンス（ベストプラクティス）

AIが制御を確立する際、以下の順序を遵守することでシステムハングを回避できる。

1. **WR9 =** `**0xC0**` ：ハードウェアリセットの発行。全割り込みの即時停止。
2. **WR4** ：通信モード（非同期/同期）とパリティの設定。
3. **WR3 / WR5** ：データ長および制御信号の確定。
4. **WR12 / WR13** ：ボーレートの確定。
5. **WR14** ：ボーレートジェネレータの起動。
6. **WR2** ：割り込みベクタの書き込み。
7. **WR1 / WR9** ：割り込み許可ビットのセット。

### 最終総括

本ガイドに従い、AIが `$E98001〜$E98007` の奇数アドレス空間を精密に制御することで、X68000は真の「パーソナルワークステーション」としての潜在能力を解放する。リカバリタイムの遵守と、デイジーチェーンを考慮した割り込み管理こそが、バイナリレベルでのハードウェア支配を完遂する鍵である。
