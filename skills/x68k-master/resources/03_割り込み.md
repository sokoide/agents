# X68000 割り込み

X68000シリーズは、MC68000 CPUの洗練された例外処理機構を最大限に活用し、高いリアルタイム性とシステムの堅牢性を両立させた設計となっている。本稿では、レガシーコンピューティングの真髄とも言えるX68000の割り込みアーキテクチャについて、ハードウェアとソフトウェアの両面からその内部構造を詳解する。

---

## 1. X68000における割り込みアーキテクチャの戦略的意義

X68000を単なる「ホビーパソコン」から「パーソナルワークステーション」へと昇華させている要因の一つが、この割り込み制御の設計にある。

### 応答性とスループットのトレードオフ

入出力処理において、最も簡便な手法は「ステータスチェック（ポーリング）」である。しかし、これはデバイスの準備が整うまでCPUがループ待機を強いられるため、マルチタスク性や応答性能を著しく損なう。割り込みシステムは、イベント発生時のみCPUのコンテキストを強制的に切り替えることで、バックグラウンドでのディスクI/Oや通信処理をフォアグラウンドの処理と並行して実行可能にする。

### 68000 CPUにおける割り込みの本質

MC68000アーキテクチャにおいて、割り込みは「例外（Exception）」という大きな枠組みの中に定義されている。これは、CPUが命令実行の境界（エッジ）でのみ割り込みをサンプリングし、現在の実行状態（PCおよびSR）をスタックに退避させ、特定のベクタに基づいて自動的なコンテキストスイッチングを行う高度な機構である。この自動化されたシーケンスこそが、複雑な周辺LSIを統制する基盤となっている。

## 2. 割り込みレベルとプライオリティ・階層構造

X68000は、MC68000の仕様に準拠した7段階のハードウェア割り込みレベルを採用している。CPUである68000は、外部回路からの要求する割り込みレベルを`IPL0`、`IPL1`、`IPL2`の3本の信号線で受け取る。レベル0は割り込みがない状態を示すため、実質的な割り込みレベルは1〜7となる。もっとも優先順位の低いのがレベル1で、もっとも高いのがレベル7である。CPUのステータスレジスタには3ビットの割り込みマスクビットがあり、この値以下の割り込みはマスクされる。CPUが割り込みを受け付けると、その割り込みレベルが自動的にマスクビットに反映され、より優先順位の高い割り込みだけが入り込めるようになる。ただし、レベル7の割り込みだけは例外で、マスクビットによってマスクされない。このことから、レベル7はNMI（Non Maskable Interrupt）とも呼ばれる。

### 割り込み系統図

X68000の割り込み系統図を以下に示す。各周辺LSIがプライオリティエンコーダを経由してCPUに接続されている。

```
                      CPU
                       |
                  +----+----+
                  |  IPL2   |
                  |  IPL1   |
                  |  IPL0   |
                  +----+----+
                       |
             プライオリティ
             エンコーダ
           （レベル7〜1）
               |
    +----------++----------+
    |         MFP         |
    |     (MC68901)       |
    |  GPIP7 - CRTC H-SYNC|
    |  GPIP6 - CRTC IRQ   |
    |  GPIP5 - (無接続)   |
    |  GPIP4 - CRTC V-DISP|
    |  GPIP3 - FM音源     |
    |  GPIP2 - POWER SW   |
    |  GPIP1 - EXP.ION    |
    |  GPIP0 - RTC ALARM  |
    |  タイマ A-D         |
    |  キーボード         |
    +--------------------+
    |  SCC (Z8530)       |
    |  Ch.A - RS-232C    |
    |  Ch.B - マウス     |
    +--------------------+
    |  DMAC (HD63450)    |
    |  ADPCM, FD, HD     |
    +--------------------+
    |  FDC (uPD72065)    |
    |  FDD               |
    +--------------------+
    |  I/Oコントローラ   |
    |  FDD, HDD, プリンタ|
    +--------------------+
    |  拡張スロット      |
    +--------------------+
```

### デバイス割り当てと「非初期化ベクタ」の役割

各レベルの優先順位とデバイス割り当ては以下の通りである。

| レベル | デバイス / 信号 | 主な機能 |
| :--- | :--- | :--- |
| **レベル 7 (NMI)** | NMIスイッチ | 本体上のNMIスイッチ（マスク不可） |
| **レベル 6** | **MFP (MC68901)** | CRTC、FM音源、タイマ、キーボード |
| **レベル 5** | **SCC (Z8530)** | RS-232C、マウス |
| **レベル 4** | **拡張スロット** | ユーザー増設デバイス |
| **レベル 3** | **DMAC (HD63450)** | ADPCM、FDD、HDD (DMA転送終了) |
| **レベル 2** | **拡張スロット** | ユーザー増設デバイス |
| **レベル 1** | **I/O コントローラ** | FDC (uPD72065)、FDD、HD、プリンタ |

ここで注目すべきは、Human 68K環境下で多くの周辺LSI（特にDMACの各チャネルなど）の初期ベクタとして設定されている**ベクタ番号 $0F** である。これはモトローラによって「非初期化割り込みベクタ」として予約されたものであり、システムの初期化が完了していないデバイスが万一割り込みを発生させた際、予測不能なジャンプを防ぐためのセーフティネットとして機能している。

### 階層ロジックとステータスレジスタ

CPU内部のステータスレジスタ（SR）にある3ビットの割り込みマスク（I0-I2）は、現在実行中の処理が受け付けるべき最低レベルを定義する。高レベルの割り込みは低レベルをプリエンプト（中断）できるが、逆は許されない。レベル7（NMI）は、このSRのマスク設定を無視して常に受け付けられる唯一のレベルであり、システムのデバッグやフリーズからの脱出において「絶対的な優先権」を持つ。

## 3. 割り込み応答動作とベクタ取得のメカニズム

割り込み要求が発生してからISR（割り込みサービスルーチン）が開始されるまでには、厳密なハードウェア・ハンドシェイクが存在する。

### 応答サイクルとバスシーケンス

周辺デバイスが割り込み要求を発生すると、外部回路で優先順位のデコードを行い、`IPL0`〜`IPL2`の3本の信号線でそのレベルをCPUに通知する。CPUは割り込みを受け付けると、アドレスバスの下位3ビット（`A1`〜`A3`）に受け付けた割り込みレベルを出力し、同時にファンクションコード（`FC0`〜`FC2`）をすべて"1"レベルにして割り込み応答サイクルであることを示し、周辺デバイスから割り込みベクタを読み出しにいく。

周辺デバイスは、データバスの下位8ビット（`D0`〜`D7`）に割り込みベクタを出力し、`DTACK`信号でCPUに対して有効なベクタがデータバス上に乗っていることを示す。CPUはこのベクタを読み取り、割り込み処理ルーチンへの移行を始める。

周辺デバイスがオートベクタを指定する（`DTACK`信号のかわりに`VPA`信号をアクティブにする）と、CPUはベクタの読み出しを行わず、各レベルに応じたデフォルトのベクタである$19〜$1F（それぞれレベル1〜レベル7に対応する）を使用する。

Human68kは、レベル7のNMIがオートベクタの$1Fを使用するほかは、すべて周辺デバイスがベクタを出力するようにして使っている。

### 割り込みハンドシェイクのシーケンス

```
CPU                          周辺デバイス
 |                              |
 |<---- (1) INT --------       |
 |                              |
 |---- IPL0-2 (2) ---------->  |
 |                              |
 |---- A1-A3 (level) ------->  |
 |---- FC0-FC2 = High ----->   |
 |                              |
 |<---- D0-D7 (vector) (4) --- |
 |                              |
 |<---- DTACK (5) ------------ |
 |                              |
 |        [ベクタ割り込み]      |
 |                              |
 |        または                |
 |                              |
 |<---- VPA (6) -------------- |
 |                              |
 |        [オートベクタ]        |
 |   $19-$1F (レベル1-7)       |
 |                              |
```

## 5. 主要LSIにおける割り込み制御とベクタ設定の実装

各周辺LSIのレジスタ操作は、システムアーキテクトにとって最も精緻な調整が求められる領域である。

### MFP (MC68901) の制御とSビットの論理

MFPはIERA/B（許可）、IPRA/B（保留）、ISRA/B（サービス中）、IMRA/B（マスク）の各レジスタで多層的な制御を行う。特にベクタレジスタ（VR）の**Sビット（ビット3）**の設定は再入可能性に直結する。

- **S=0 (オートEOI)**: CPUがベクタを読み取った瞬間にISRビットが自動クリアされる。これにより同一ソースの割り込みが即座に許可されるが、処理が追いつかない場合はスタックオーバーフローのリスクを伴う。
- **S=1 (ソフトウェアEOI)**: ソフトウェアでISRレジスタを明示的にクリアするまで、同一要因の割り込みをブロックできる。これは多重割り込みの深さを厳密に制御する設計思想に適している。

### アーキテクトの警告：タイマBとキーボード制御

MFPのタイマB（TBCR）については細心の注意が必要である。**タイマBはキーボード通信用のボーレートジェネレータとして固定**されており、この設定を不用意に変更すると、キーボードからの入力が即座に不可能になる。この「触れてはいけない領域」を理解しておくことは、X68000の実践的な開発において必須の知識である。

### 割り込みベクタ設定ポート

周辺デバイスごとに割り込みベクタを設定するポートを以下にまとめる。

| デバイス | レジスタ | アドレス | ベクタ設定 |
|----------|----------|----------|------------|
| **MFP** | VBR | $E88017 | 割り込み要因で変化 |
| **SCC** | WR2 | $E98003/7 | 任意設定可 |
| **DMAC CH#1** | IVR | $E84025 | 任意設定可 |
| **DMAC CH#0** | EIV | $E84027 | 任意設定可 |
| **DMAC CH#1** | NIV | $E84065 | 任意設定可 |
| **DMAC CH#1** | EIV | $E84067 | 任意設定可 |
| **DMAC CH#2** | NIV | $E840A5 | 任意設定可 |
| **DMAC CH#2** | EIV | $E840A7 | 任意設定可 |
| **DMAC CH#3** | NIV | $E840E5 | 任意設定可 |
| **DMAC CH#3** | EIV | $E840E7 | 任意設定可 |
| **I/Oコントローラ** | | $E9C003 | 任意設定可 |

### MFP 割り込み要因とベクタの対応

MFPのベクタレジスタ（VR）の上位4ビットにベクタ番号の上位を設定し、下位3ビットは要因番号として使用される（ビット3はSビット）。各要因に対応するベクタ番号は VR の上位ビット + 要因番号で決定される。

| 要因番号 | 割り込み要因 |
|----------|-------------|
| 0000 | GPIP0 (RTC ALARM) |
| 0001 | GPIP1 (EXP.ION) |
| 0010 | GPIP2 (POWER SW) |
| 0011 | GPIP3 (FM音源) |
| 0100 | タイマD |
| 0101 | タイマC |
| 0110 | GPIP4 (CRTC V-DISP) |
| 0111 | GPIP5 (無接続) |
| 1000 | タイマB |
| 1001 | タイマA |
| 1010 | 送信バッファ空 |
| 1011 | 受信バッファフル |
| 1100 | GPIP6 (CRTC IRQ) |
| 1101 | GPIP7 (CRTC H-SYNC) |

## 6. 実践的な割り込み利用方法とプログラミング上の留意点

実際のプログラミングにおいては、ハードウェアの挙動を前提とした実装指針が不可欠である。

### スーパーバイザモード（Supervisor Mode）の必然性

MC68000において、ステータスレジスタ（SR）の操作や$00から始まる例外ベクタテーブルの書き換えは、特権命令および特権領域のアクセスを伴う。したがって、割り込みを制御するプログラムを実装する際は、Human 68Kの`SUPER`コールなどを使用して**スーパーバイザモード**へ移行することが絶対的な前提条件となる。

### ISR実装の鉄則

割り込みはメインプログラムのコンテキストを任意に中断するため、ISR内では使用するすべてのレジスタをスタックに退避（`MOVEM`命令等）し、終了時に復帰させなければならない。最後は必ず`RTE`命令で終了させ、例外スタックフレームの破棄とSRの復帰を行う。

### リスク管理とシステム統合

ソースコンテキストのサンプルプログラムでは安定性のためにステータスチェックが多用されているが、これはベクタ書き換えミスによる「妙な動作（暴走）」を避けるための教育的配慮である。実用的なアプリケーションでは、マルチタスク環境での親和性を高めるため、可能な限り割り込みベースの設計に移行すべきである。

X68000のポテンシャルを極限まで引き出すためには、これら複雑に絡み合うLSI群の割り込み機構を俯瞰し、論理的な優先順位に基づいて制御することが唯一の道である。本稿が、次代のシステムアーキテクトによる深い理解の一助となれば幸いである。
