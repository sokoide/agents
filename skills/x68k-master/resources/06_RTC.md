# X68000 リアルタイムクロック（RTC）

RP5C15 ベース。時刻保持、ALARM による自動起動、SRAM 設定保持を担当。

## 配置

- ベースアドレス: `$E8A001` 以降の奇数アドレス。
- **ALARM 信号**: MFP GPIP0（`$E88001` bit0）に直結。システムをスリープから復帰させるトリガーとして機能。
- 電源投入要因の事後判定はシステムポート#2（`$E89001`）で行う。`$E88xxx` は MFP レジスタと混同注意。

## 内部レジスタ

| オフセット | レジスタ | 機能 |
| --- | --- | --- |
| +01, +03... | 時刻カウンタ群 | 秒、分、時、日、月、年（各桁ごと） |
| +$1B | アジャストレジスタ | 水晶発振子の誤差補正 |
| +$1D | 12/24時間セレクタ | 表記モード切替 |
| +$1F | 閏年カウンタ | カレンダー整合性の HW レベル維持 |
| +$21 | MODE レジスタ | 動作モード切替（カウント停止、テストモード含む） |
| +$23 | テストレジスタ | LSI 健全性確認用。通常運用では触れない |
| +$25 | RESET コントローラ | 分周回路・カウンタの初期化 |

## 時刻読み出し（ティアリング回避）

RTC は CPU に対して極めて低速で、内部で非同期にカウントアップする。読み出し中に秒が繰り上がることで不整合（ティアリング）が発生するリスクがある。

### 回避策

1. **二度読み比較法**: 時刻データを 2 回連続読み出し、完全一致するまでループ。
2. **Busyフラグ監視**: ステータスレジスタの Busy ビットを確認し、更新されていない隙にアクセス。

```c
#define RTC_BASE 0xE8A001
typedef volatile unsigned char * ioreg;

void read_rtc_seconds() {
    ioreg sec_units = (ioreg)(RTC_BASE + 0x00);
    unsigned char s1, s2;
    do {
        s1 = *sec_units & 0x0F;
        s2 = *sec_units & 0x0F;
    } while (s1 != s2);
}
```

## 時刻書き込み

MODE レジスタで書き込み許可状態へ移行 → 「秒」から上位桁へ順に設定 → 最後にカウント再開。

## ポーリング推奨

RTC のような低速デバイスでは割り込みハンドラのオーバーヘッドが相対的に大きいため、ポーリング（ステータスチェック）方式が実用的。
