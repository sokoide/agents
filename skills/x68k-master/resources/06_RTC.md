# X68000 RTC（リアルタイムクロック）

X68000は、日付・時刻の保持やアラーム動作を行うRTC（リアルタイムクロック）として、リコー製の**RP5C15**を使用している。時計としての用途のほか、指定時刻での本体電源ON（タイマ機能）や、本体前面のTIMER-LEDの点滅制御に使用している。

基本クロックとして時計IC用で一般的な32.768KHzの水晶発振子を使用し、内部で分周して1秒単位のクロックを作成している。

---

## 1. RTC周辺ブロック図

```
         電源制御                    MFP
             |                       |
    RP5C15   |                       | GPIP0
  +----------+---------+             |
  | ALARM ---------+   |             |
  |          |     |   |             | GPIP5
  | CLKOUT --+-----+---+-------------+
  |          |     |   |
  | OSCIN ---+ 32.768KHz
  | OSCOUT --+
  | VCC/VCC2
  +------------------+
```

ALARM信号はMFPのGPIP0に接続され、電源ONの要因として使用される。CLKOUT端子は本体前面のTIMER-LEDの点滅に使用されている。

---

## 2. RTCのレジスタ

RTCのレジスタはMODEレジスタのBNKビットでBANK0とBANK1を切り替えて使用する。ベースアドレスは$E8A001〜$E8A01F（奇数番地のみ）。

| アドレス | BANK0 | BANK1 |
| :--- | :--- | :--- |
| $E8A001 | CLKOUTセレクト | CLKOUTセレクト |
| $E8A003 | 10秒/秒レジスタ | Adjustレジスタ |
| $E8A005 | 10分/分レジスタ | （予約） |
| $E8A007 | 10時間/時間レジスタ | アラーム10分/分レジスタ |
| $E8A009 | 10日/日レジスタ | アラーム10時間/時間レジスタ |
| $E8A00B | 10月/月レジスタ | アラーム10日/日レジスタ |
| $E8A00D | 10年/年レジスタ | アラーム10月/月レジスタ |
| $E8A00F | 10曜日/曜日レジスタ | アラーム10曜日/曜日レジスタ |
| $E8A011 | MODEレジスタ | MODEレジスタ |
| $E8A013 | テストレジスタ | テストレジスタ |
| $E8A015 | 閏年カウンタ | 閏年カウンタ |
| $E8A017 | （予約） | （予約） |
| $E8A019 | （予約） | （予約） |
| $E8A01B | （予約） | （予約） |
| $E8A01D | テストレジスタ | テストレジスタ |
| $E8A01F | RESETコントローラ | RESETコントローラ |

BANK0の時刻レジスタは各桁ごとに分かれている（例：秒は$E8A003の下位4ビットが1の位、上位4ビットが10の位）。

### CLKOUTセレクトレジスタ（$E8A001）

CLKOUT端子への出力を決定する。有効なのは下位3ビット。

| 設定値 | CLKOUT出力 |
| :--- | :--- |
| 000 | ハイインピーダンス |
| 001 | 32.768KHz |
| 010 | 8192Hz |
| 011 | 1024Hz |
| 100 | 64Hz |
| 101 | 1秒周期 |
| 110 | 1分周期 |
| 111 | 1時間周期 |

X68000ではCLKOUT端子をTIMER-LEDの点滅に使用している。CLKOUT端子がHレベルのときに点灯する。したがって、"111"で消灯、"000"で点灯（ハイインピーダンスはHレベルと同等）、"101"で1秒周期の点滅となる。

### Adjustレジスタ（BANK1, $E8A003）

```
bit 7               4 3             0
+---+---+---+---+---+---+---+---+
|                       |   |   |
+---+---+---+---+---+---+---+---+
                        |
                        +--- Adjustビット
                           1: アジャストON
                           0: アジャストOFF

秒カウンタが0〜29のときにアジャストすると、
たんに秒カウンタが0になる。
秒カウンタが30〜59のときにアジャストすると、
秒カウンタが0になり、分カウンタが1つ進む。
```

### 閏年カウンタ（$E8A015）

閏年から何年たっているかを設定するレジスタ。閏年カウンタに設定された年は閏年の扱いとなり、2月が29日までカウントされる。1年ごとに自動的にインクリメントされるため、閏年の例外（100で割り切れて400で割り切れない年は閏年としない）が発生しないかぎり自動的に処理される。

この例外措置が次に行われるのは西暦2100年である。西暦2000年は100で割り切れるが400でも割り切れるため閏年となる。このレジスタには**西暦の年数を4で割った余り**を書き込む。

### MODEレジスタ（$E8A011）

```
bit 7             4 3 2             0
+---+---+---+---+---+---+---+---+
|   |   |   |   |TMEN|ALEN|BNK|
+---+---+---+---+---+---+---+---+
                |   |   |   |
                |   |   +--- BANK選択
                |   |       1: BANK1, 0: BANK0
                |   +--- アラーム動作許可/禁止
                |       1: アラーム動作許可
                |       0: アラーム動作禁止
                +--- タイマ動作許可/禁止
                    1: タイマ動作許可
                    0: タイマ動作禁止
```

### テストレジスタ（$E8A013, $E8A01D）

チップメーカ（リコー）での出荷検査用。通常はすべて"0"を設定すること。

### RESETコントローラ（$E8A01F）

| ビット | 名称 | 説明 |
| :--- | :--- | :--- |
| 3 | 1Hz ON | 1Hz信号によるタイマスタート |
| 2 | 16Hz ON | 16Hz信号によるタイマスタート |
| 1 | タイマRESET | 秒以下のタイマ部リセット |
| 0 | アラームRESET | アラーム部リセット |

## 3. アラーム機能

RTCのアラーム機能は、設定した時刻になるとALARM端子から信号を出力する。X68000ではこのALARM信号をMFPのGPIP0に接続し、電源ONの要因として使用している。

アラームリセットを行うと、新たにアラーム時刻設定レジスタ側に書き込まないかぎり、強制的に一致した状態になる。このため、毎日18:00と22:00といった動作を行わせる場合、毎回日付・曜日などを設定し直す手間を省ける。アラームの時・分のレジスタだけを変更すればよい。

### アラーム設定手順

1. **アラームディセーブル** — MODEレジスタのALENビット（bit2）を"0"にする
2. **アラームリセット** — RESETコントローラレジスタのbit0を"1"にする
3. **アラーム時刻設定** — 各アラーム設定レジスタに時刻を書き込む
4. **アラームイネーブル** — MODEレジスタのALENビット（bit2）を"1"にする

## 4. RTCのアクセス

RTCの時刻はCPUとは関係なく動作しており、CPU側からは一度に1つのレジスタしかアクセスできないため、アクセスには注意が必要。

### 時刻の読み出し

読み出し中に桁上がりが発生すると不正な値になる。回避方法：

1. **1Hz信号に同期** — アラーム出力はCLKOUTと比べ位相が約180°ずれており、アラーム出力の立ち下がりから96μs後にRTCの時刻更新が行われる
2. **時計を止めてから読み出す**
3. **複数回読み出して一致した値を有効とする**

### 時計データの書き込み

書き込み中に桁上がりが起こると不正な設定になる。回避方法：

1. 時計を止めてから設定する
2. RESETコントローラのタイマリセットビットで秒より下の桁をクリアし、停止させてから書き込む
3. 1Hz信号に同期させて書き込む

### 曜日カウンタ

曜日カウンタは1日ごとに0〜6までの値を順にとっていく7進カウンタで、どの値を日曜日に対応させるかはユーザ任せ。Human68kでは**0を日曜日**として扱っている。

## 5. サンプルプログラム

```c
/* XCの場合には
 * #define volatile
 * の1行を入れてください
 */

#define TRUE  1
#define FALSE 0

char *tdayofweek[7] = { "SUN", "MON", "TUE", "WED", "THU", "FRI", "SAT" };

volatile unsigned char *rtc_base = (unsigned char *)0xe8a000;
volatile unsigned char *rtc_mode = (unsigned char *)0xe8a011;

void read_time(unsigned char *buf)
{
    volatile unsigned char *rtc;
    unsigned int i;

    rtc = rtc_base;
    for (i = 0; i < 3; i++, rtc += 4) {
        *buf++ = (*rtc & 0x0f) + (*(rtc+2) & 0x0f) * 10;
    }
    *buf++ = *rtc & 0x0f;
    rtc += 2;
    for (i = 0; i < 3; i++, rtc += 4) {
        *buf++ = (*rtc & 0x0f) + (*(rtc+2) & 0x0f) * 10;
    }
}

void write_time(unsigned char *buf)
{
    volatile unsigned char *rtc;
    unsigned int i;

    rtc = rtc_base;
    for (i = 0; i < 3; i++, rtc += 4) {
        *rtc = *buf % 10;
        *(rtc+2) = *buf / 10;
        buf++;
    }
    *rtc = *buf & 0x07;
    rtc += 2;
    for (i = 0; i < 3; i++, rtc += 4) {
        *rtc = *buf % 10;
        *(rtc+2) = *buf / 10;
        buf++;
    }
}

void main()
{
    unsigned char time[7];

    read_time(time);
    printf("20%02d/%02d/%02d(%s) %02d:%02d:%02d\n",
        time[0], time[1], time[2], tdayofweek[time[3]],
        time[4], time[5], time[6]);
}
```

## 6. 実装上のガイドライン

1. **奇数アドレス厳守**: RTCはデータバスの下位8ビット（D0-D7）に接続されているため、$E8A001以降の奇数アドレスにアクセスする。
2. **BANK切替**: MODEレジスタのBNKビットでBANK0/BANK1を切り替える。時刻読み出し時はBANK0、アラーム設定時はBANK1を使用する。
3. **ティアリング回避**: 読み出し中の桁上がりを防ぐため、1Hz同期・二度読み比較などの手法を用いる。
4. **volatile修飾子**: C言語で実装する場合、RTCレジスタへのポインタには必ず`volatile`を付与する。
