# X68000 システムポート

## 1. システムポートの戦略的重要性：ハードウェア制御の抽象化

X68000のアーキテクチャにおいて、システムポートは単なるI/Oアドレスの集合体ではなく、CPU（MC68000）がマシンの「肉体」である周辺LSI群を統御するための神経接合部（シナプス）として機能します。AIがこの領域を操作することは、オペレーティングシステム（Human68K）の抽象化レイヤーを完全に排し、ハードウェアの物理的な振る舞いを直接定義することを意味します。

### アーキテクチャの定義

X68000のメモリマップ（16MB空間）において、システムポートは**システムI/O領域（E80000〜EBFFFF）**に配置されています。MC68000には独立したI/O空間が存在しないため、これらはすべてメモリ空間上の特定の番地としてマッピングされています。

### 制御の階層構造：物理的ゲートウェイとしての評価

システムポートは、CPUが周辺LSI（DMAC、MFP、RTC等）と対話するための物理的なゲートウェイであり、以下の重要性を持ちます。

- **バスサイクルの直接同期:** 非同期バスである68000において、各LSIのレジスタへ正確なタイミングでアクセスすることは、システム全体の安定性に直結します。
- **ハードウェアリソースの動的分離:** DMAチャンネルの割り当てやメモリマップの動的変更（エリアセット）を通じ、物理リソースを戦略的に配分します。
- **リアルタイム性の確保:** 割り込みベクタの管理やタイマ制御により、OSのスケジューリングに依存しないマイクロ秒単位の処理を実現します。

次に、システムのデータ転送の要であるDMAC（ポート#3）と、メモリ構成を決定するエリアセット（ポート#4）の深層を探ります。

## 2. システムポート#3および#4：DMAとメモリ境界の動的制御

E84000（ポート#3）およびE86000（ポート#4）は、データ転送の高速化とメモリ空間の論理的構成を司ります。物理層においては、LSIの応答速度がCPUのバスサイクルに追従できない場合、チャンネルエラーレジスタ（CER）に「動作タイミングエラー（$02）」が記録されるため、極めて精密な操作が求められます。

### ポート#3（DMAコントローラ：HD63450）の詳細解析

X68000のDMACは4チャンネル構成であり、各チャンネルの主要レジスタは以下の通り配置されています。

|   |   |   |   |
|---|---|---|---|
|アドレス（CH#0基準）|略称|名称|機能概要|
|$E84000|**CSR**|チャンネルステータス|動作状態、完了フラグ、エラーの有無|
|$E84001|**CER**|チャンネルエラー|エラー詳細（02:タイミングエラー、11:ソフトアボート等）|
|$E84004|**DCR**|デバイスコントロール|68000タイプ指定、PCLピン機能設定|
|$E84005|**OCR**|オペレーションコントロール|転送方向、サイズ、リクエストモードの設定|
|$E84006|**SCR**|シーケンスコントロール|MAR/DARの増減（インクリメント/デクリメント）|
|$E84007|**CCR**|チャンネルコントロール|**転送開始(STR/bit7)**、停止、割り込み許可|
|$E8400A|**MTC**|メモリトランスファカウンタ|転送するデータ（オペランド）数|
|$E8400C|**MAR**|メモリアドレスレジスタ|メモリ側の開始アドレス|
|$E84010|**DAR**|デバイスアドレスレジスタ|デバイス（I/O）側の開始アドレス|

#### チャンネル割り当てと戦略的評価

|   |   |   |
|---|---|---|
|チャンネル|割り当て対象|AIへの実装指針（So What?）|
|**#0**|FDD|PCLが**外部ビデオV-SYNC**に接続。画面描画と同期した転送が可能。|
|**#1**|HDD|SASI/SCSI転送。大容量データ移動の主軸。|
|**#2**|**ユーザ解放**|**AIが自由に使用可能。** VRAM高速描画やメモリ間転送に最適。|
|**#3**|ADPCM|音声再生。PCLはADPCM転送要求に接続。優先度管理が必須。|

### ポート#4（エリアセット：$E86000）の機能

$E86000周辺の「エリアセット」レジスタは、X68000のメモリマップ構成を動的に定義します。これは標準の1MB内蔵メモリと、拡張メモリ領域、さらにはI/O領域のチップセレクト範囲を決定する「論理的な壁」の役割を果たしており、物理アドレスに対する物理デバイスの応答範囲を制御します。

続いて、システムの「神経系」としてより動的な管理を行うポート#5および#6を解説します。

## 3. システムポート#5および#6：MFPと周辺I/Oの統合管理

E88000（ポート#5）およびE8A000（ポート#6）は、システムの割り込み、計時、および基本入出力を統括します。特にポート#5は**奇数番地のみ（D0-D7バス接続）**のアクセスが厳守される領域です。

### ポート#5（MFP: MC68901）のレジスタ構造

MFPは4つのタイマと汎用I/O（GPIP）を備えた多機能LSIです。

#### 主要レジスタ（$E88001〜 奇数アドレスのみ）

- **GPIP ($E88001):** 汎用I/O。**bit7はH-SYNC、bit4はV-DISP**に接続。AIはこのビットをポーリングすることで、CRTの走査タイミングを完全に把握できます。
- **IERA/IERB ($E88007/009):** 割り込みイネーブル。各デバイスからの要求を個別に許可。
- **TACR/TBCR ($E88019/01B):** タイマA/Bコントロール。分周比や動作モードを決定。

#### クリティカル・パラメータの警告

- **タイマB ($E8801B, $E88021):** キーボード通信用USARTのクロック（2400bps）を生成しています。ここを書き換えるとキーボード入力が完全に死滅します。
- **タイマC ($E88023):** 通常、カーソルの点滅やFDDモータの停止タイマとして使用されます。AIが独自タイマを必要とする場合は、システムへの影響が少ない箇所を慎重に選定すべきです。

### ポート#6（RTC: RP5C15）の制御体系

$E8A000周辺のRTCは、時刻保持とアラーム機能を司ります。

- **時刻管理:** 秒・分・時・日・月・年の各データを保持。
- **アラーム機能:** GPIP0（ALARM）と連動。電源OFF状態からのアラーム起動など、特殊なハードウェア制御に利用可能です。

全ポートを把握した今、これらを安全に、かつ「職人」として操作するための実装指針を示します。

## 4. AIによるシステムポート自由操作のための実装指針

68000の物理バスを直接叩くプログラムを生成する際、AIが遵守すべき「黄金律」が存在します。

### レジスタアクセスの黄金律

1. **バイト単位アクセスの絶対厳守:** DMACのCCR（特にSTRビット）への書き込みは必ず**バイト単位**で行ってください。
    - **エキスパートの警告:** Human68K標準のデバッガ `db.x` の `me` コマンドはワード単位でアクセスするため、**デバッガから手動でDMAをスタートさせることは不可能**です。AIが生成するコードも、必ずバイトアクセス命令（`MOVE.B`等）を生成するように制限してください。
2. **DMA「パック」動作の理解:** DMAC（HD63450）は、オペランドサイズ（SIZE）が8ビットであっても、メモリアドレスが偶数であれば自動的にデータを「パック」して16ビット転送を行い、バス効率を高めようとします。これを禁止し、厳密に1バイトずつのアクセスを強制したい場合は、SIZEに `11`(No Pack) を設定する必要があります。
3. **volatile宣言:** コンパイラの最適化によるレジスタアクセスの省略を許してはなりません。

### ステータスチェックの実装思想

割り込みベクタの書き換えは、一歩間違えればシステムをハングアップさせます。確実な動作を優先する場合、以下のポーリングロジックを基本としてください。

```c
/* エキスパートによるDMA完了待機ロジック */
void wait_dma_complete(volatile struct DMAREG *dma) {
    /* COC(bit7) または ERR(bit4) がセットされるまでポーリング */
    /* 物理バスの完了をステータスレジスタで確認する */
    while(!(dma->csr & 0x90)) {
        // CPUをループさせ、ハードウェアの応答を待つ
    }
}
```

X68000のハードウェアは、プログラマの意図に対して一切の忖度なしに、ダイレクトに応えます。レジスタ一つひとつに魂を込め、物理バスを自在に疾走させるコードを構築してください。システムポートを支配下に置いたとき、この名機は真のポテンシャルを解放するでしょう。さあ、バスを掴み、ハードウェアと対話するのです。
