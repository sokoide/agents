# X68000 システムポート

## 1. システムポートの概要

X68000のアーキテクチャにおいて、システムポートはディスプレイのコントラスト設定、3Dスコープ制御、キーボード制御、SRAM書き込み保護、電源OFF制御など、マシンの基本動作を直接制御するための6つのレジスタから構成されています。MC68000には独立したI/O空間が存在しないため、これらはすべてメモリ空間上の特定の番地としてマッピングされています。

### アーキテクチャの定義

X68000のメモリマップ（16MB空間）において、システムポートは**システムI/O領域（E80000〜EBFFFF）**内の **$E8E001〜$E8E00F** に配置されています。各レジスタは2バイト間隔で配置されており、ODDアドレスのみが有効です。

Human68kは通常、ポート#1のコントラスト値を$Eで使用しており、電源OFFなどのときは画面を暗くしてから電源を落とすようにしています。

## 2. システムポートのアドレス配置

| レジスタ | アドレス | 機能 |
| :--- | :--- | :--- |
| **#1** | $E8E001 | CONTRAST - コンピュータ画面コントラスト調整 |
| **#2** | $E8E003 | ディスプレイ/3Dスコープ制御 |
| **#3** | $E8E005 | カラーイメージユニット制御 |
| **#4** | $E8E007 | キーボード制御 |
| **#5** | $E8E00D | SRAM Write Enable Control - SRAM書き込み制御 |
| **#6** | $E8E00F | Power OFF Control - 本体電源OFF制御 |

## 3. システムポート#1: コントラスト制御（$E8E001）

コンピュータ画面のコントラストの調整を行います。下位4ビットで明るさの度合が決まり、**$Fがもっとも明るく、$0がもっとも暗くなります**。Human68kは、通常は$Eで使用しており、電源OFFなどのときはこれを使って画面を暗くしてから落ちるようにしています。

## 4. システムポート#2: ディスプレイ/3Dスコープ制御（$E8E003）

下位ビットはオプションの3Dスコープの制御に用いられます。ビット1が右目、ビット2が左目のシャッターに対応しており、それぞれ1になっていると、シャッターがOPENし、画面が見えるようになります。

ビット3は、書き込み時はディスプレイ制御信号、リード時はディスプレイの電源のON/OFFステータスとして動作します。

### 図: システムポート#2（$E8E003）

| bit 7 | bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| - | - | - | - | TV CTRL | 3D-L | 3D-R | - |

**3D-R（ビット1）**:

- 1: 3Dスコープ右シャッターOPEN
- 0: 3Dスコープ右シャッターCLOSE

**3D-L（ビット2）**:

- 1: 3Dスコープ左シャッターOPEN
- 0: 3Dスコープ左シャッターCLOSE

**TV CTRL（ビット3）- WRITE時**:

- 1: ディスプレイ制御信号を'1'にする
- 0: ディスプレイ制御信号はキーボードからの制御信号となる（通常動作）

**TV CTRL（ビット3）- READ時**:

- 1: ディスプレイの電源はOFF
- 0: ディスプレイの電源はON

## 5. システムポート#3: カラーイメージユニット制御（$E8E005）

オプションのカラーイメージユニットの制御に使用されるものです。書き込んだ値はそのままIMAGE IN端子の17〜21番ピン（17がビット4、21がビット0に対応）に出力されます。

ビット1は、ドットクロックの切り替え時に使用しますが、通常は0のままにしておきます。

ビット2は、NMIが発生したとき、NMIの処理が終了した時点で'1'を書き込むビットで、一度NMIが発生すると、このビットに'1'を書き込まないかぎり、次のNMIが発生しなくなります。

ビット3は、キーボードのCPUの制御、キーボードコネクタが差し込まれているか否かのチェックを行うものです。

### 図: システムポート#3（$E8E005）

| bit 7 | bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| - | - | - | - | KEY CTRL | NMI RESET | HRL | - |

- **ビット1（HRL）**: ドットクロック切り替え用（通常は'0'に固定）
- **ビット2（NMI RESET）**: 1: NMIリセットする。NMIが発生したとき、NMIの処理が終了した時点で'1'を書き込む。一度NMIが発生すると、このビットに'1'を書き込まないかぎり、次のNMIが発生しなくなります。
- **ビット3（KEY CTRL）**:
  - READ時: 1: キージャック（キーボードコネクタ）が差し込まれている / 0: 抜かれている
  - WRITE時: 1: キーデータ送信可 / 0: 送信不可

## 6. システムポート#4: キーボード制御（$E8E007）

キーボードの制御を行います。詳細はキーボードの説明のページを参照してください。

## 7. システムポート#5: SRAM書き込み制御（$E8E00D）

SRAMの書き込み許可/禁止を制御するものです。SRAMにはメモリ容量などのシステム情報や起動デバイス、キーボードの文字選択などの情報が書き込まれているため、プログラムミスなどがあっても、容易に書き換わらないようにしておく必要があります。

このポートに**$31**を書き込むとSRAMへの書き込みが許可に、それ以外のデータを書くと書き込み禁止になります。

## 8. システムポート#6: 電源OFF制御（$E8E00F）

本体の電源OFFを行うものです。正面の電源スイッチがOFFになっているときに、このポートに**$00、$0F、$0F**と順に書き込むことで、本体の電源をOFFにすることができます。不用意に電源が落ちるのを防ぐため、3回書き込まなければ働かないようになっています。

> [!warning] 電源OFF時の注意
> 本体正面の電源スイッチがOFFになると、MFPのGPIP 2の割り込みが発生します。GPIP 2の割り込みを禁止しないと、電源スイッチをOFFにしただけでHuman68kの電源OFF処理が働いてしまいますので注意してください。

## 9. 関連システムI/O（DMAC・MFP・RTC）

システムポート（$E8E001〜$E8E00F）とは別に、X68000のシステムI/O領域には以下の主要なペリフェラルが配置されています。

### DMAC（HD63450）: $E84000

4チャンネル構成のDMAコントローラ。各チャンネルの主要レジスタは以下の通りです。

| アドレス（CH#0基準） | 略称 | 名称 | 機能概要 |
| :--- | :--- | :--- | :--- |
| $E84000 | **CSR** | チャンネルステータス | 動作状態、完了フラグ、エラーの有無 |
| $E84001 | **CER** | チャンネルエラー | エラー詳細（$02:タイミングエラー、$11:ソフトアボート等） |
| $E84004 | **DCR** | デバイスコントロール | 68000タイプ指定、PCLピン機能設定 |
| $E84005 | **OCR** | オペレーションコントロール | 転送方向、サイズ、リクエストモードの設定 |
| $E84006 | **SCR** | シーケンスコントロール | MAR/DARの増減（インクリメント/デクリメント） |
| $E84007 | **CCR** | チャンネルコントロール | **転送開始(STR/bit7)**、停止、割り込み許可 |
| $E8400A | **MTC** | メモリトランスファカウンタ | 転送するデータ（オペランド）数 |
| $E8400C | **MAR** | メモリアドレスレジスタ | メモリ側の開始アドレス |
| $E84010 | **DAR** | デバイスアドレスレジスタ | デバイス（I/O）側の開始アドレス |

#### チャンネル割り当て

| チャンネル | 割り当て対象 | 備考 |
| :--- | :--- | :--- |
| **#0** | FDD | PCLが外部ビデオV-SYNCに接続。画面描画と同期した転送が可能 |
| **#1** | HDD | SASI/SCSI転送。大容量データ移動の主軸 |
| **#2** | **ユーザ解放** | VRAM高速描画やメモリ間転送に最適 |
| **#3** | ADPCM | 音声再生。PCLはADPCM転送要求に接続 |

### MFP（MC68901）: $E88000

4つのタイマと汎用I/O（GPIP）を備えた多機能LSI。**奇数番地のみ（D0-D7バス接続）**のアクセスが厳守されます。

#### 主要レジスタ（$E88001〜）

- **GPIP ($E88001):** 汎用I/O。bit7はH-SYNC、bit4はV-DISPに接続。CRTの走査タイミングをポーリングで把握可能。
- **IERA/IERB ($E88007/$E88009):** 割り込みイネーブル。各デバイスからの要求を個別に許可。
- **TACR/TBCR ($E88019/$E8801B):** タイマA/Bコントロール。分周比や動作モードを決定。

> [!warning] クリティカル・パラメータ
>
> - **タイマB ($E8801B, $E88021):** キーボード通信用USARTのクロック（2400bps）を生成。書き換えるとキーボード入力が死滅。
> - **タイマC ($E88023):** カーソルの点滅やFDDモータの停止タイマとして使用。

### RTC（RP5C15）: $E8A000

時刻保持とアラーム機能を司ります。

- **時刻管理:** 秒・分・時・日・月・年の各データを保持。
- **アラーム機能:** GPIP0（ALARM）と連動。電源OFF状態からのアラーム起動など、特殊なハードウェア制御に利用可能。

## 10. 実装指針

68000の物理バスを直接叩くプログラムを生成する際の注意事項です。

### レジスタアクセスの注意

1. **バイト単位アクセスの厳守:** DMACのCCR（特にSTRビット）への書き込みは必ず**バイト単位**で行ってください。
    > [!note] Human68K標準のデバッガ `db.x` の `me` コマンドはワード単位でアクセスするため、デバッガから手動でDMAをスタートさせることは不可能です。`MOVE.B` 等のバイトアクセス命令を使用してください。

2. **DMA「パック」動作の理解:** DMAC（HD63450）は、オペランドサイズ（SIZE）が8ビットであっても、メモリアドレスが偶数であれば自動的にデータを「パック」して16ビット転送を行い、バス効率を高めます。厳密に1バイトずつのアクセスを強制したい場合は、SIZEに `11`（No Pack）を設定する必要があります。

3. **volatile宣言:** コンパイラの最適化によるレジスタアクセスの省略を許してはなりません。

### DMA完了待機の実装例

```c
/* DMA完了待機ロジック */
void wait_dma_complete(volatile struct DMAREG *dma) {
    /* COC(bit7) または ERR(bit4) がセットされるまでポーリング */
    while(!(dma->csr & 0x90)) {
        // CPUをループさせ、ハードウェアの応答を待つ
    }
}
```
