# X68000 システムポート

`$E80000`〜`$EBFFFF` のシステム I/O 領域を統括するレジスタマップ。

## ポート統合リファレンス

| ポート | 基点 | 対応LSI | 主要機能 | アクセス制約 |
| --- | --- | --- | --- | --- |
| **#1** | $E80000 | CRTC | 画面表示制御 | - |
| **#2** | $E82000 | VC | プライオリティ制御 | - |
| **#3** | $E84000 | DMAC | 高速転送（Ch0 V-SYNC同期可） | **STR 操作はバイト必須** |
| **#4** | $E86000 | - | エリアセット（メモリマップ定義） | - |
| **#5** | $E88000 | MFP | 割り込み・タイマ・シリアル | **奇数番地のみ** |
| **#6** | $E8A000 | RTC | カレンダー・アラーム | 奇数番地 |
| **#7** | $E90000 | FM音源 | YM2151 (OPM) | - |
| **#8** | $E94000 | FDC | µPD72065 フロッピーディスク | 奇数番地 |
| **#9** | $E96000 | SCSI/HDC | SASI/SCSI | 奇数番地 |
| **#10** | $E98000 | SCC | Z8530 シリアル通信 | 奇数番地 |
| **#11** | $E9A000 | ジョイスティック | i8255 入力ポート | 奇数番地 |
| **#12** | $E9C000 | ADPCM | MSM5205 音声再生 | - |
| **#13** | $EA0000 | - | ADPCM レジスタ | - |
| **#14** | $EB0000 | - | スプライトレジスタ | - |

## ポート#3: DMAC（HD63450）主要レジスタ

Ch0 基準のオフセット。各 ch は +00, +40, +80, +C0。

| アドレス | 略称 | 機能 |
| --- | --- | --- |
| $E84000 | CSR | チャンネルステータス |
| $E84001 | CER | チャンネルエラー（02:タイミング, 11:ソフトアボート） |
| $E84004 | DCR | デバイスコントロール |
| $E84005 | OCR | オペレーションコントロール |
| $E84006 | SCR | シーケンスコントロール |
| $E84007 | CCR | **チャンネルコントロール。STR(bit7) はバイトアクセス必須** |
| $E8400A | MTC | メモリトランスファカウンタ |
| $E8400C | MAR | メモリアドレス（32bit） |
| $E84010 | DAR | デバイスアドレス（32bit） |

### チャンネル割り当て

| Ch | 割り当て | 備考 |
| --- | --- | --- |
| #0 | FDD | PCL が外部ビデオ V-SYNC に接続。画面同期転送可能 |
| #1 | HDD | SASI/SCSI 転送 |
| #2 | **ユーザ解放** | VRAM 高速描画・メモリ間転送に最適 |
| #3 | ADPCM | PCL は ADPCM 転送要求に接続 |

## ポート#4: エリアセット（`$E86000`）

メモリマップ構成を動的に定義。標準 1MB 内蔵メモリ、拡張メモリ領域、I/O チップセレクト範囲を制御。物理アドレスに対する物理デバイスの応答範囲を規定する。

## ポート#5: MFP 主要レジスタ

- **GPIP**（`$E88001`）: bit7=H-SYNC, bit4=V-DISP。CRT 走査タイミングをポーリング監視。
- **IERA/IERB**（`$E88007`/`$E88009`）: 割り込み個別許可。
- **TACR/TBCR**（`$E88019`/`$E8801B`）: タイマ A/B 制御。
- **警告**: Timer B（`$E8801B`, `$E88021`）はキーボード USART クロック（4800bps）生成。変更で入力死滅。Timer C（`$E88023`）は通常カーソル点滅/FDD モータ停止タイマ。

## ポート#6: RTC

$E8A000 周辺。時刻保持＋アラーム。GPIP0（ALARM）と連動（詳細は `06_RTC.md`）。

## DMA 完了待機ロジック

```c
void wait_dma_complete(volatile struct DMAREG *dma) {
    while(!(dma->csr & 0x90)) { /* COC(bit7) or ERR(bit4) */ }
}
```

## CCR バイトアクセスの重要性

- DMAC CCR（特に STR ビット）への書き込みは**バイトアクセス**が必須。
- Human68K 標準デバッガ `db.x` の `me` コマンドはワードアクセスするため、**デバッガから DMA をスタートさせることは不可能**。
- コード生成時も `MOVE.B` 等のバイトアクセス命令を必ず使用。

## Pack Mode の理解

DMAC はオペランドサイズ 8 ビットでも、メモリアドレスが偶数なら自動的にデータを「パック」して 16 ビット転送しバス効率を高める。厳密に 1 バイトずつのアクセスを強制する場合は SIZE に `11`（No Pack）を設定。
