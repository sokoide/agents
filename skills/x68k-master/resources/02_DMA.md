# X68000 DMAコントローラ（HD 63450）

1980年代後半から90年代にかけて、パーソナルワークステーションの代名詞であったX68000。その圧倒的なグラフィックス性能の背後には、CPU（MC68000）をデータ転送というルーチンワークから解放し、システム全体のバス効率を最大化する**DMAC（HD 63450）**の存在がありました。

本書では、ハードウェアアーキテクトの視点から、このDMACの構造を解剖し、現代のエンジニアやAIがVRAM高速転送を完全に制御するための実装ロジックを提示します。

--------------------------------------------------------------------------------

## 1. X68000におけるDMA転送の戦略的意義

X68000においてDMA（Direct Memory Access）は、単なる転送手段ではなく、**「CPUと並列して動作するもう一つのプロセッサ」**として機能します。

特に毎秒60回の書き換えが要求されるVRAM制御において、CPUによる `MOVE` 命令のループは致命的なオーバーヘッドとなります。DMAC（HD 63450）を利用することで、以下の技術的利点が得られます。

- **バス占有の最適化:** CPUの命令フェッチを介さず、ハードウェア主導でバスサイクルを生成。
- **並列処理:** ADPCMのバックグラウンド再生や、FDD/HDDからのデータロードをCPU処理と完全にオーバーラップ可能。
- **高速描画:** 矩形領域の転送（スプライト的な挙動やウィンドウの移動）において、CPUのレジスタ待避を一切不要とする「アレイチェイン」による自動実行。

## 2. DMACのチャンネル割り付けとハードウェア構成

X68000に搭載されたHD 63450は4つの独立したチャンネルを持ち、以下のように物理的な接続先が規定されています。

|   |   |   |   |
|---|---|---|---|
|チャンネル|割り当てデバイス|外部信号（REQ/ACK/PCL）の接続先|備考|
|**#0**|フロッピーディスク|FDC (μPD72065)|割り込みベクタは通常 $0F|
|**#1**|ハードディスク|HDC (SASI/SCSI)|割り込みベクタは通常 $0F|
|**#2**|**ユーザー/拡張ボード**|**拡張スロットに接続**|**VRAM転送に最適**|
|**#3**|ADPCM|ADPCM制御ロジック|サウンド再生に専念|

### チャンネル#2の戦略的優位性

チャンネル#0, #1, #3はシステム予約済みであり、通常はサイクルスチールモードで動作します。一方、**チャンネル#2**は外部信号が拡張スロットに開放されているため、ユーザーによるメモリーメモリ（VRAM間）転送に制限なく使用可能です。特に「オートリクエストモード（内部要求）」を用いることで、バスの最大帯域を転送に割り当てることが可能になります。

--------------------------------------------------------------------------------

## 3. レジスタ・マップと制御パラメータの詳解

DMA制御は、`$E84000` をベースとする一連のレジスタ群を介して行われます。

### 主要レジスタ・オフセット（各チャンネル +00, +40, +80, +C0）

- **CSR ($00):** チャンネルステータスレジスタ。転送終了（COC）やエラー（ERR）を確認。
- **CER ($01):** チャンネルエラーレジスタ（Read Only）。エラー詳細。
- **DCR ($04):** デバイスコントロールレジスタ。転送元・先の属性。
- **OCR ($05):** オペレーションコントロールレジスタ。転送方向、サイズ、モード。
- **SCR ($06):** シーケンスコントロールレジスタ。アドレスの増減。
- **CCR ($07):** **重要：チャンネルコントロールレジスタ。必ず「バイトアクセス」で操作。**
- **MAR ($0C):** メモリアドレスレジスタ（32bit）。
- **DAR ($14):** デバイスアドレスレジスタ（32bit）。
- **MTC ($0A):** メモリトランスファカウンタ（16bit）。転送語数。
- **BAR ($1C):** ベースアドレスレジスタ（32bit）。チェイン時のテーブル開始位置。
- **BTC ($1A):** ベーストランスファカウンタ（16bit）。チェインするブロック数。

### 特殊レジスタ：GCR (General Control Register)

**アドレス:** `**$E840FF**` GCRはDMAC全体で共有されるレジスタであり、チャンネル#3のレジスタ空間の末尾にのみ存在します。限定速度オートリクエスト時のバス占有率を設定しますが、通常VRAM転送では最大速度（OCR=$01）を用いるため、初期設定のまま運用するのが一般的です。

--------------------------------------------------------------------------------

## 4. 転送モードの選定：デュアルアドレスと内部要求

HD 63450はシングルアドレスモードをサポートしていますが、**X68000のアーキテクチャでは基本的にデュアルアドレスモードのみを使用します。**

- **デュアルアドレスモードの挙動:**
    1. 読み込みサイクル: MAR（またはDAR）からDMAC内部バッファにデータをロード。
    2. 書き込みサイクル: 内部バッファからDAR（またはMAR）へデータをストア。
- **オートリクエスト（最大速度）:** VRAMクリアやメモリーメモリ転送では、外部REQ信号を待たずに連続してバス権を行使する「オートリクエスト最大速度モード（OCRのREQGビット = 01）」を選択します。

--------------------------------------------------------------------------------

## 5. 高度な転送制御：アレイチェインによる矩形転送

不連続なメモリ領域（VRAM上の矩形領域など）を転送する場合、**アレイチェインモード**が「最強の武器」となります。

### アレイチェインの転送情報テーブル（XFR_INF）構造

メモリ上に以下の構造体（1エントリ6バイト）を連続して配置し、その先頭アドレスをBARにセットします。

|   |   |   |   |
|---|---|---|---|
|オフセット|サイズ|名称|説明|
|+0|4 bytes|**MAR**|転送元（または先）のメモリアドレス|
|+4|2 bytes|**MTC**|そのラインで転送する語数（Word/Long単位）|

### チェイン動作の比較

|   |   |   |
|---|---|---|
|項目|アレイチェイン|リンクアレイチェイン|
|**テーブル配置**|メモリ上に連続|不連続（各エントリにリンクポインタを含む）|
|**終了条件**|**BTC**レジスタが0になった時|リンクアドレスが **$00000000** になった時|
|**VRAM用途**|矩形描画（固定ライン数）に最適|複数のパーツを動的に合成する場合|

## 6. VRAM転送の実装プロセス

### メモリアドレス計算ロジック

グラフィックVRAM（GVRAM）は `$C00000` から始まりますが、内部アドレス空間は**水平512ドット固定**として扱われます（表示モードに依存しません）。また、1ドット＝1ワード（16bit）構成です。

- **座標(X, Y)のアドレス計算:** `Address = $C00000 + (512 * Y + X) * 2` ※ `* 2` は1ドットが2バイト（1ワード）であることを示します。

### Register Presets (Quick Start)

|   |   |   |   |   |
|---|---|---|---|---|
|ユースケース|DCR|OCR|SCR|設定のポイント|
|**TVRAM高速クリア**|$08|$21|$01|MAR固定、DAR増加。MARのアドレスに0を書き込んで転送。|
|**GVRAM矩形転送**|$08|$99|$05|**アレイチェイン指定**。MAR, MTCはテーブルからロード。|
|**メモリ間高速コピー**|$00|$02|$05|16bit転送、MAR/DARともに増加設定。|

## 7. 実行制御とステータス管理

DMA転送を安定させるためには、以下の「堅牢な監視ロジック」が不可欠です。

### 転送実行アルゴリズム（疑似コード）

1. **ステータスクリア:** `CSR = $FF` を書き込み、古い `COC`, `BTC`, `NDT`, `ERR`, `ACT` フラグをすべて倒す。 **※重要：フラグが1のままでは新規転送（STR）を受け付けず「動作タイミングエラー」となります。**
2. **パラメータ設定:** `MAR`, `DAR`, `MTC` (または `BAR`, `BTC`) を設定。 **※MTCに0を設定してSTRを叩くと「カウントエラー」が発生します。**
3. **転送開始:** `**CCR = $80**` **(STRビット) を「バイトアクセス」で書き込む。**
4. **終了監視ループ:**
5. **事後処理:** 再度 `CSR = $FF` でフラグをクリアし、システムをクリーンな状態に戻す。

### 結論

HD 63450は、X68000のハードウェアポテンシャルを極限まで引き出すための「影の主役」です。特にチャンネル#2とアレイチェインの組み合わせは、現代のタイルベース描画にも通じる効率的なスプライト・ブロック転送を実現します。CCRへのバイトアクセスと、CSRの明示的クリア――この2点を厳守することが、堅牢なシステムプログラミングの第一歩となります。
