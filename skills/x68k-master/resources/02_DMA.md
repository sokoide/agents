# X68000 DMAコントローラ（HD63450）

DMAC は CPU をデータ転送から解放し、バス効率を最大化する。VRAM 高速転送、ADPCM バックグラウンド再生、FDD/HDD データロードを CPU 処理と並行実行可能。

## チャンネル割り付け

| チャンネル | 割り当て | 外部信号 | 備考 |
| --- | --- | --- | --- |
| **#0** | FDD | FDC (µPD72065) | 割り込みベクタ通常 $0F |
| **#1** | HDD | HDC (SASI/SCSI) | 割り込みベクタ通常 $0F |
| **#2** | **ユーザー/VRAM** | 拡張スロット | **VRAM転送に最適**、オートリクエスト可能 |
| **#3** | ADPCM | ADPCM制御ロジック | サウンド再生専用 |

Ch0/1/3 はシステム予約。**Ch2** は外部信号が拡張スロットに開放されており、オートリクエスト最大速度モードでバス帯域をフル活用可能。

## レジスタマップ（ベース: `$E84000`、各ch +00/+40/+80/+C0）

| レジスタ | オフセット | 機能 |
| --- | --- | --- |
| CSR | $00 | チャンネルステータス（COC/BTC/NDT/ERR/ACT） |
| CER | $01 | チャンネルエラー（Read Only） |
| DCR | $04 | デバイスコントロール（転送元・先の属性） |
| OCR | $05 | オペレーションコントロール（方向、サイズ、モード） |
| SCR | $06 | シーケンスコントロール（アドレス増減） |
| **CCR** | $07 | **チャンネルコントロール。必ずバイトアクセスで操作** |
| MAR | $0C | メモリアドレス（32bit） |
| DAR | $14 | デバイスアドレス（32bit） |
| MTC | $0A | メモリトランスファカウンタ（16bit） |
| BAR | $1C | ベースアドレス（32bit、チェイン時テーブル先頭） |
| BTC | $1A | ベーストランスファカウンタ（16bit、チェインブロック数） |

**GCR**（General Control Register）: `$E840FF`。DMAC 全体で共有。限定速度オートリクエスト時のバス占有率設定。通常 VRAM 転送では最大速度（OCR=$01）を使うため初期設定のままでよい。

## 転送モード

X68000 では**デュアルアドレスモードのみ**を使用。

- 読み込みサイクル: MAR（または DAR）→ DMAC 内部バッファ
- 書き込みサイクル: 内部バッファ → DAR（または MAR）

**オートリクエスト最大速度**: VRAM クリア等では外部 REQ を待たずに連続バス権を行使。OCR の REQG ビット = 01 で設定。

## アレイチェイン（矩形転送用）

不連続 VRAM 領域（矩形）を転送する最適なモード。メモリ上に以下のテーブル（1 エントリ 6 バイト）を連続配置し、先頭アドレスを BAR にセット。

| オフセット | サイズ | 名称 | 説明 |
| --- | --- | --- | --- |
| +0 | 4 bytes | MAR | 転送元（または先）のメモリアドレス |
| +4 | 2 bytes | MTC | そのラインの転送語数 |

| | アレイチェイン | リンクアレイチェイン |
| --- | --- | --- |
| テーブル配置 | 連続 | 不連続（リンクポインタ付き） |
| 終了条件 | **BTC** が 0 | リンクアドレスが `$00000000` |
| 用途 | 矩形描画（固定ライン数） | 複数パーツの動的合成 |

## GVRAM アドレス計算

GVRAM 内部は**水平512ドット固定**（表示モードに依存しない）。1 ドット＝1 ワード。

- 座標 (X, Y): `Address = $C00000 + (512 * Y + X) * 2`

## Register Presets

| ユースケース | DCR | OCR | SCR | ポイント |
| --- | --- | --- | --- | --- |
| TVRAM高速クリア | $08 | $21 | $01 | MAR固定、DAR増加 |
| GVRAM矩形転送 | $08 | $99 | $05 | アレイチェイン。MAR/MTCはテーブルから |
| メモリ間高速コピー | $00 | $02 | $05 | 16bit転送、MAR/DARとも増加 |

## 転送実行アルゴリズム

1. **CSR クリア**: `CSR = $FF` を書き込み、全フラグを倒す。※フラグが 1 のままでは新規転送（STR）を受け付けない。
2. **パラメータ設定**: MAR, DAR, MTC（または BAR, BTC）。※MTC=0 で STR を叩くとカウントエラー。
3. **転送開始**: `CCR = $80`（STR ビット）を**バイトアクセス**で書き込む。
4. **終了監視**: CSR の COC/ERR をポーリング。
5. **事後処理**: 再度 `CSR = $FF` でクリア。
