# X68000 メモリマップ

## 1. X68000 メモリ空間の全体像

MC68000というCPUを選択した時点で、我々設計陣が目指したのは「16MBのリニアなメモリ空間」がもたらす圧倒的な自由度であった。当時のライバル機であった80X86系CPUのような、セグメント方式による64KBの壁や、独立したI/O空間といった煩わしい制約はここには存在しない。

X68000のアーキテクチャは、すべてのレジスタや周辺デバイスをメモリ空間の一部として扱う「メモリマップドI/O」を採用している。プログラマは、メモリ転送命令（MOVE）ひとつで、メインメモリへのアクセスも、FM音源の制御も、VRAMへの描画も等価に行うことができる。この「フラット」な思想こそが、ホビー向けPCの枠を超えた「パーソナルワークステーション」たる所以である。

### 全体領域配分表

|   |   |   |   |
|---|---|---|---|
|アドレス範囲|サイズ|役割・機能|物理特性・設計意図|
|`$000000` - `$BFFFFF`|12MB|メインメモリ（RAM）|プログラム・データ用。最大12MBまで拡張可能|
|`$C00000` - `$DFFFFF`|2MB|グラフィックVRAM|物理実装512KB。論理1024x1024空間|
|`$E00000` - `$E7FFFF`|512KB|テキストVRAM|4プレーン・ビットマップ構成|
|`$E80000` - `$EBFFFF`|256KB|システムI/O|CRT, DMA, MFP, RTC等の制御LSI|
|`$EC0000` - `$ECFFFF`|64KB|ユーザI/O|拡張ボード用。**ユーザモードアクセス可**|
|`$ED0000` - `$ED3FFF`|16KB|SRAM|バッテリーバックアップ済。システム設定保持|
|`$F00000` - `$FBFFFF`|768KB|CGROM|文字パターンデータ（8x8〜24x24）|
|`$FE0000` - `$FFFFFF`|128KB|IPL-ROM|起動プログラム・基本IOCS。末尾64KBは起動鏡像|

## 2. IPLイメージとリセット直後の論理配置

MC68000は、リセット解除直後に`$000000`からSSP（スーパーバイザスタックポインタ）を、`$000004`からPC（プログラムカウンタ）を読み出す仕様となっている。しかし、X68000の物理実装において、0番地側はDRAM（メインメモリ）である。電源投入直後の内容は不定であり、そのままでは起動できない。

これを解決するのが、ハードウェアによる「IPLイメージ」の投影と、鮮やかな「ミラー解除」の仕掛けである。

### 起動時のミラーリングとハードウェア・トグル

電源投入、あるいはリセットスイッチ押下時、ハードウェアは自動的に`$000000`〜`$00FFFF`（64KB）の領域に、IPL-ROMの末尾64KB（`$FF0000`〜`$FFFFFF`）の内容を投影する。 特筆すべきは、このミラーリングを終了させるトリガーである。**CPUが一度でも本来のROM領域（**`**$FF0000**`**以降）へアクセスした瞬間、このミラーリング・ハードウェアはオフになり、0番地側は本来のDRAMへと切り替わる。** この機構により、ROM内のブートコードからスムーズにRAMベースのシステムへと制御を移すことが可能となっている。

### システムアーキテクトの視点：開発時の留意事項

このミラーリングは、物理的なリセット（電源ON/リセットスイッチ）でのみ発動する。ソフトウェア上の「RESET命令」では発動しないため、プログラムから再起動をかける場合は、直接IPLのエントリポイントへジャンプするか、IOCSを介する必要がある。また、デバッガ等で不用意に`$FFxxxx`領域を読み出すとミラーが解除されてしまうため、低レイヤー開発者には極めて慎重なメモリ操作が求められる。

## 3. メインメモリ（RAM）領域の構造と拡張性

ユーザーに開放された最大12MBのメインメモリは、当時としては極めて贅沢な「フラットな広野」であった。

### メモリ拡張の物理構造

- **標準装備（1MB/2MB）：** 0番地から1MBまでは全機種共通の標準領域。`$100000`からの2枚目の1MBは、機種によって標準搭載か内蔵増設オプション（CZ-6BE1等）かが分かれる。
- **拡張スロットとXVIの内蔵拡張：** `$200000`以降の増設は、通常は拡張スロットのメモリボードで行うが、XVI以降は「Area Set」コントローラの進化により、本体内部だけで8MBもの増設が可能となった。

### システムアーキテクトの視点：互換性と「フラット」の価値

X68000は、8bit PCのような「バンク切り替え」という醜悪な回避策を排除した。初期の1MB標準機（無印）において2MBを前提としたソフトが動かない問題はあったが、これは単なる容量不足であり、メモリさえ増設すればアーキテクチャの変更なしに動作する。この「リニアな拡張性」が、長期間にわたる高い互換性を支えたのである。

## 4. VRAM（グラフィック・テキスト）領域の定義

X68000の描画性能を支えるのは、論理と物理を巧みに使い分けたVRAMマッピングである。

### グラフィックVRAM（C00000〜DFFFFF）

論理空間は2MB（1024x1024ドット）確保されているが、物理実装は512KBである。

- **1ドット＝1ワード（16bit）設計：** 16色、256色モードであっても、1ドットに16bitを割り当てる。物理メモリが512KBしかないため、実際にはアクセス可能な領域は限られるが、この設計には明確な意図がある。
- **座標計算の最適化：** 画面の横幅を1024ドットに固定することで、Y座標の算出が`Y << 10`（10ビットのシフト演算）のみで完結する。これは、複雑な乗算器を持たない68000において、高速描画を実現するための「ハードウェアによる数学的解決」である。

### テキストVRAM（E00000〜E7FFFF）

512KBが実装されており、こちらは4プレーン構成のビットマップ方式を採用している。

- **ビットマップ方式の柔軟性：** 多くの16bit機が採用していた「キャラクタジェネレータによる文字表示」ではなく、テキスト画面自体がグラフィック画面と同様のビットマップである。これにより、CGROMのフォントをCPUやDMAが「描画」する形で表示を行い、多種多様なフォント混在を容易にしている。

## 5. システム/ユーザI/O、SRAM、およびCGROM

周辺機器との接点であるI/O領域は、バス制御において「68000 Bus Type」（DCR bit 10-8 = 00）として統合されており、拡張ボードの設計を簡略化させている。

### 主要システムI/Oの配置（E80000〜EBFFFF）

|   |   |   |
|---|---|---|
|アドレス|デバイス名称|概要|
|`$E80000`|CRTコントローラ|画面解像度、同期信号、ラスタ割り込み制御|
|`$E82000`|ビデオコントローラ|画面ON/OFF、プライオリティ設定|
|`$E84000`|DMAコントローラ|高速データ転送（HD 63450）|
|`$E88000`|MFP|MFP (MC68901) タイマ、シリアル、GPIP|
|`$E8A000`|RTC|リアルタイムクロック（時計）|
|`$E90000`|FM音源|OPM (YM2151) 8音ポリフォニック|
|`$E94000`|FDC|フロッピーディスクコントローラ|
|`$EA0000`|ADPCM|音声合成・再生（OKI MSM5205互換）|

### ユーザI/O（EC0000）とSRAM（ED0000）

- **ユーザI/Oの解放：** 64KBのこの領域は**ユーザモードでアクセス可能**である。これは極めて野心的な設計で、OSを介さずとも自作ハードウェアを直接制御できる自由を個人ユーザに与えた。この「聖域なき解放」が、独創的な同人ハードウェア文化を育んだ。
- **SRAM：** 16KBのバッテリーバックアップ領域。画面色やブートデバイス設定など、OS以前の「システムの状態」を永続化する。

### CGROM（$F00000〜）

8x8から24x24ドットまでの6種類のフォントを内蔵。テキストVRAMがビットマップであるため、これらは固定文字ではなく「高速なパターンリソース」として機能する。

## 6. IPL-ROM領域とシステム初期化の役割

メモリマップの最終端（`$FE0000`〜`$FFFFFF`）には、システムの魂とも言える128KBのIPL-ROMが鎮座する。

### IOCSの進化と柔軟な再配置

初期のシステムではROM内のIOCS（基本入出力サブルーチン）が直接利用されていた。しかし、Human68Kの進化に伴い、IPL-ROMの役割は「周辺デバイスの最小限の初期化」と「ブートローダ」へと純化されていった。 現代的な運用では、ディスクから最新の`IOCS.X`をRAM上にロードし、システム全体をアップデートされたソフトウェアベースの環境へと移行させる。この「ROMを固定しながらシステムを成長させる」設計こそが、初代機を10年経っても現役たらしめた最大の要因である。

| ----------- | --------------- | --------------------------------- |
| アドレス範囲      | 主な機能            | アーキテクトの注釈                         |
| `$000000` - | メインメモリ / IPLミラー | 起動直後はROM末尾64KBの鏡像。ROMアクセスでDRAMへ切替 |
| `$100000` - | 標準/内蔵増設RAM      | 機種により標準2MB、または1MB+内蔵オプション増設       |
| `$200000` - | 拡張RAM領域         | 最大12MB。XVI以降は内蔵バス経由で8MB増設可        |
| `$C00000` - | グラフィックVRAM      | 論理1024x1024。Y座標計算をシフト演算（ASL）で高速化  |

| `$E80000` - | システムI/O         | 68000バスに直結された制御レジスタ群。メモリ操作で完結     |
| `$EC0000` - | ユーザI/O          | ユーザモード解放領域。カスタムハードウェアの「実験場」       |
| `$ED0000` - | SRAM            | システム設定の永続化。電源OFFでも保持される16KB       |
| `$F00000` - | CGROM           | 文字リソース。ビットマップVRAMへの描画用フォント供給源     |
| `$FE0000` - | IPL-ROM         | 128KBの実体。末尾64KBが電源投入時のブート鏡像となる    |
