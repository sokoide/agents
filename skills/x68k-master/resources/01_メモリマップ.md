# X68000 メモリマップ

## 全体領域配分

| アドレス範囲 | サイズ | 役割 | 備考 |
| --- | --- | --- | --- |
| `$000000` - `$BFFFFF` | 12MB | メインメモリ（RAM） | 最大12MB拡張可能 |
| `$C00000` - `$DFFFFF` | 2MB | グラフィックVRAM | 物理実装512KB、論理1024x1024 |
| `$E00000` - `$E7FFFF` | 512KB | テキストVRAM | 4プレーン・ビットマップ構成 |
| `$E80000` - `$EBFFFF` | 256KB | システムI/O | CRTC, DMA, MFP, RTC等の制御LSI |
| `$EC0000` - `$ECFFFF` | 64KB | ユーザI/O | **ユーザモードアクセス可** |
| `$ED0000` - `$ED3FFF` | 16KB | SRAM | バッテリーバックアップ、システム設定保持 |
| `$F00000` - `$FBFFFF` | 768KB | CGROM | 文字パターンデータ（8x8〜24x24） |
| `$FE0000` - `$FFFFFF` | 128KB | IPL-ROM | 起動プログラム・基本IOCS、末尾64KBは起動鏡像 |

## IPL ミラーリング

電源投入/リセット時、`$000000`〜`$00FFFF`（64KB）に IPL-ROM 末尾 64KB（`$FF0000`〜`$FFFFFF`）が投影される。

- **ミラー解除条件**: CPU が本来の ROM 領域（`$FF0000`以降）に一度でもアクセスすると、ミラーリングがオフになり 0 番地側は DRAM に切り替わる。
- **注意**: 物理リセット（電源 ON/リセットスイッチ）でのみ発動。ソフトウェア `RESET` 命令では発動しない。デバッガで `$FFxxxx` を不用意に読むとミラーが解除される。

## メインメモリ構成

- **標準**: 0 番地から 1MB は全機種共通。`$100000` からの 2 枚目は機種により標準/増設オプション。
- **拡張**: `$200000` 以降は拡張スロットのメモリボード。XVI 以降は本体内部だけで 8MB 増設可能。
- バンク切り替えなしのリニアアクセス。メモリさえ増設すればアーキテクチャ変更なしで動作。

## グラフィックVRAM（`$C00000`〜`$DFFFFF`）

- 論理空間 2MB（1024x1024 ドット）、物理実装 512KB。
- **1ドット＝1ワード（16bit）**。16 色/256 色モードでも 16bit 割り当て。
- 座標計算: `Address = $C00000 + (Y << 10 + X) << 1`。Y 座標が 10 ビットシフトで済む設計。

## テキストVRAM（`$E00000`〜`$E7FFFF`）

- 512KB 実装、4 プレーン・ビットマップ方式。
- キャラクタジェネレータではなくビットマップのため、CGROM フォントを DMA/CPU が「描画」する形式。多様なフォント混在が容易。

## システムI/O 主要配置（`$E80000`〜`$EBFFFF`）

| アドレス | デバイス | 概要 |
| --- | --- | --- |
| `$E80000` | CRTC | 画面解像度、同期信号、ラスタ割り込み |
| `$E82000` | ビデオコントローラ | 画面ON/OFF、プライオリティ設定 |
| `$E84000` | DMAC | HD63450 高速データ転送（4ch） |
| `$E88000` | MFP | MC68901 タイマ、シリアル、GPIP |
| `$E8A000` | RTC | RP5C15 リアルタイムクロック |
| `$E90000` | FM音源 | YM2151 (OPM) 8音ポリフォニック |
| `$E94000` | FDC | µPD72065 フロッピーディスクコントローラ |
| `$EA0000` | ADPCM | MSM5205 音声合成・再生 |

## ユーザI/O・SRAM・CGROM

- **ユーザI/O**（`$EC0000`）: 64KB、ユーザモードアクセス可。自作ハードウェア直接制御用。
- **SRAM**（`$ED0000`）: 16KB、バッテリーバックアップ。画面色・ブートデバイス等のシステム設定を永続化。
- **CGROM**（`$F00000`）: 8x8〜24x24 ドットのフォント 6 種。テキスト VRAM がビットマップのため「パターンリソース」として機能。

## IPL-ROM（`$FE0000`〜`$FFFFFF`）

128KB。初期は ROM 内 IOCS を直接利用していたが、Human68K の進化に伴い「最小限の初期化＋ブートローダ」に純化。現代運用ではディスクから `IOCS.X` を RAM にロードし、ソフトウェアベースでシステム全体をアップデート可能。
