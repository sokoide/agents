# X68000 画面制御

## 1. はじめに：X68000 映像出力アーキテクチャの戦略的意義

1987年の衝撃的なデビュー以来、X68000は単なるパーソナルコンピュータの枠組みを破壊し、「パーソナルワークステーション」という新たな地平を切り拓きました。その核心を成すのが、独立したハードウェア制御に基づく多層的な画面合成アーキテクチャです。

本機は、グラフィック、テキスト、BG（バックグラウンド）、スプライトといった独立した表示プレーンをハードウェア側でリアルタイムに合成する仕組みを備えています。この設計は、当時主流であった他機種がCPUのパワーを泥臭い描画処理に浪費していたのに対し、映像制御を専用チップに委ねることで、アーケードゲームに比肩する表現力を個人が手にするための戦略的な要諦となりました。

**So What?：表現自由度へのインパクト** 多層プレーンの独立制御は、開発者を煩雑な描画ロジックから解放し、より高次元なアルゴリズムや演出に計算資源を集中させることを可能にしました。ハードウェアが提供する「重ね合わせ」と「透過」の自由度は、16bitマシンの限界を超えた視覚表現を約束する強力な武器となったのです。

次セクションでは、この表現力を支える盤石の基盤、すなわち物理実装と論理空間が交差するメモリレイアウトについて深掘りします。

## 2. メモリマップとVRAMレイアウトの構造分析

X68000の画面制御をマスターする上で、16MBのリニアな論理アドレス空間におけるVRAMとI/Oレジスタの厳密な配置把握は不可欠です。

### 映像・制御系の主要アドレス配置

|   |   |   |
|---|---|---|
|名称|アドレス|役割・物理仕様|
|**グラフィックVRAM**|`$C00000` ～ `$DFFFFF`|512KB実装、論理上1024×1024ドット|
|**テキストVRAM**|`$E00000` ～ `$E7FFFF`|512KB、1024×1024ドット×4プレーン|
|**CRTC（CRTコントローラ）**|`$E80000`|表示タイミング、同期制御|
|**ビデオコントローラ**|`$E82000`|プライオリティ、特殊表示設定|
|**DMAC（HD63450）**|`$E84000`|高速データ転送（全4ch）|
|**スプライトレジスタ**|`$EB0000`|スプライト属性・座標管理|
|**スプライトVRAM**|`$EB8000`|パターンデータ格納領域|

**So What?：1ドット＝1ワード設計の妙** X68000のグラフィックVRAMは、1ドットに対して1ワード（16ビット）を割り当てる設計を採用しています。この設計の真の価値は、AIが座標計算を行う際の予測可能性にあります。 アドレス計算式：`Target Address = Base + (Y * 1024 + X) << 1` PC-98やAmiga等で見られた複雑な「ビットプレーン構造」とは異なり、X68000はプレーンを跨ぐビットシフト演算を必要としない「パックドピクセル」に近い扱いが可能です。AIによる描画ルーチン生成において、この線形性は演算負荷を劇的に低減させ、高速なピクセル操作を実現する源泉となります。

続いて、これらのメモリ空間を構成する各プレーンの内部構造と、個別の制御メカニズムを解析します。

## 3. 画面プレーン別・内部構成と制御メカニズム

X68000は、特性の異なる4種類の表示プレーンを動的に組み合わせることで、複雑なGUIと動的なゲーム映像を両立させています。

### 画面プレーン構成比較

|   |   |   |   |
|---|---|---|---|
|プレーン|データ形式|最大解像度|特徴・主な用途|
|**グラフィック**|ビットマップ|1024×1024|16/256/65536色。自由な描画。|
|**テキスト**|ビットマップ|1024×1024|漢字含む4プレーン。GUIやワープロ用。|
|**BG**|タイル（セル）|512×512|2面。高速スクロール、低負荷。|
|**スプライト**|タイル|16×16/1ドット|最大128個。動体の高速移動に最適。|

### 色数モードとデータ構造の深層

特に65,536色モードでは、1ワード内に直接カラーデータがパッキングされるダイレクトカラー方式を採ります。 データ構造：`Low bit(L) + 5 bits Blue + 5 bits Green + 5 bits Red` このLビット（輝度）を含むRGB555形式の正確な理解は、AIが色計算やグラデーション生成を行う際の基本原則となります。

**So What?：ビットマップとタイルの戦略的使い分け** 自由度の高いテキスト画面（ビットマップ）と、メモリ・CPU負荷が極めて低いBG画面（タイル方式）の使い分けは、システムのパフォーマンスを最大化するための黄金律です。AIによる自動制御においては、静的な情報表示にはテキスト画面を、動的な背景演出にはBG画面を割り当てるという戦略的なリソース配分が求められます。

次に、これら全てのプレーンを統合・出力する「心臓部」の制御について詳述します。

## 4. CRTインタフェースとビデオコントローラの深層制御

ハードウェアによる画面合成の核心は、CRTC（E80000）およびビデオコントローラ（E82000）のレジスタ操作に集約されます。

### ビデオコントローラによる動的統合

- **プライオリティ制御:** 各画面プレーンの重なり順をレジスタひとつで瞬時に変更可能です。
- **特殊表示:** プレーン間の「半透明合成」、画面端まで表示を広げる「オーバースキャン」、フリッカーを抑える「インターレース」などをハードウェアレベルで処理します。
- **カラーパレット:** 65,536色中16色または256色を選択するパレット形式においても、ハードウェアがRGB555データとして高速処理を行います。

**So What?：CPU負荷の劇的低減** これらの合成・透過処理がハードウェアで完結しているため、CPUは単にレジスタに優先順位を書き込むだけで済みます。この「ハードウェアへの肩代わり」こそが、16bitマシンの性能限界を超えた複雑な多重スクロールやスプライト合成を支える理論的根拠です。

次は、画面に命を吹き込むためのスクロールと高速処理技術に焦点を当てます。

## 5. 高度な画面操作技術：スクロール・高速クリア・ラスタコピー

動的な演出には、CPUによる泥臭いデータ転送を回避し、ハードウェア機能を使い倒すテクニックが不可欠です。

### ハードウェア支援による高速化

1. **スクロール制御:** 各プレーンのスクロールレジスタを操作するだけで、CPU負荷ゼロでの高速な画面移動が可能です。
2. **高速クリア制御:** グラフィックVRAMの広大な領域を瞬時に消去する専用回路を備えています。
3. **DMAC（HD63450）との連携:** `$E84000`から配置されるDMACの**Channel #2**は、ユーザ解放されており、メモリ・ツー・メモリの高速転送に最適化されています。アレイチェインやリンクアレイチェインモードを用いることで、不連続なVRAM領域への矩形転送もCPUを介さず完結します。
4. **ラスタコピー:** ビデオコントローラ固有の機能であり、VRAM間のデータコピーを高速に行います。これはDMACによる汎用転送とは異なる専用回路によるものです。

**So What?：アーケード機に迫る表現力の源泉** これらの高速化機能は、68000 CPUの演算能力を補完し、あたかも高性能な描画チップが併走しているかのようなパフォーマンスを生み出します。16bitパソコンでありながら、ゲームセンターの筐体に匹敵する「滑らかさ」を実現できたのは、この徹底したハードウェア支援があったからです。

技術的詳細を網羅したところで、最後にAIがこれらを操るための実装指針をまとめます。

## 6. AIによる自動制御に向けた実装指針と注意点

AIがX68000のハードウェアを直接制御する際に、回避不可能な「鉄則」が二点存在します。

### 1. スーパバイザモード（Privileged Access）の厳守

`$E80000`以降のI/O領域およびVRAMへのアクセスは、CPUが**スーパバイザモード**にあることが前提となります。AIがリアルタイム制御プログラムを生成する場合、必ず`SR`（ステータスレジスタ）の特権ビットを操作するか、`TRAP`呼び出しを介して特権状態でアクセスを行うロジックを組み込む必要があります。

### 2. V-Blank（垂直帰線期間）への同期

VRAMへのアクセスは、画面描画が行われていない**V-Blank期間**に集中させるべきです。表示期間中に大量の書き込みを行うと、ハードウェアの走査と競合し、画面に「ノイズ（雪）」が発生したり、バス衝突によるウェイトが発生します。60fpsの流麗な描画を維持するためには、GPIP（$E88001）からV-DISP信号を監視し、垂直帰線期間内に転送を完結させるタイミング管理が必須です。

### 標準的な初期化フローの要約

1. スーパバイザモードへの移行。
2. CRTCレジスタによる画面モードの設定（設定順序の遵守）。
3. ビデオコントローラによる表示ONおよびプライオリティ設定。
4. DMAC（ch2）等を用いた初期背景/フォントデータのVRAM展開。

**So What?：AIのための戦略的抽象化** AIによる動的制御を実現するには、煩雑なレジスタアドレスを直接叩くのではなく、「V-Blank同期転送」や「プライオリティ変更」といった定型操作を優先的にAPI化・抽象化すべきです。これにより、AIは物理制約に縛られることなく、純粋な映像演出の最適化に集中することが可能になります。

## 7. 結論：内部構造の理解がもたらす究極の表現力

X68000の画面制御システムは、当時の設計者の思想が結実した、極めて合理的かつ美しいアーキテクチャです。各プレーンが独立し、それを強力な専用ハードウェアが統合する構造は、まさに「パーソナルワークステーション」の理想そのものです。

本ドキュメントで示した緻密なアドレスマップと制御ロジックは、AIがこの伝説的なマシンのポテンシャルを限界まで引き出すための「聖典」となります。

**So What?：制約を武器に変えるマインドセット** ハードウェアの仕様は、克服すべき「制約」ではなく、表現を加速させるための「武器」です。この完全解析を基盤として、AIがかつての人間を凌駕する新たな映像体験を創造することを期待します。
