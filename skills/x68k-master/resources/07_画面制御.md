# X68000 画面制御

グラフィック、テキスト、BG、スプライトの 4 プレーンをハードウェアでリアルタイム合成。

## 主要アドレス

| 名称 | アドレス | 役割 |
| --- | --- | --- |
| グラフィックVRAM | `$C00000`〜`$DFFFFF` | 512KB実装、論理1024×1024 |
| テキストVRAM | `$E00000`〜`$E7FFFF` | 512KB、1024×1024×4プレーン |
| CRTC | `$E80000` | 表示タイミング、同期制御 |
| ビデオコントローラ | `$E82000` | プライオリティ、特殊表示 |
| DMAC | `$E84000` | 高速データ転送（4ch） |
| スプライトレジスタ | `$EB0000` | スプライト属性・座標 |
| スプライトVRAM | `$EB8000` | パターンデータ |

## GVRAM アドレス計算

### 65536色モード（ダイレクトカラー）

1 ドット＝1 ワード（16bit）。座標計算: `Address = $C00000 + (Y * 1024 + X) << 1`

プレーンを跨ぐビットシフト不要のパックドピクセル相当。ビットプレーン構造（PC-98/Amiga 等）とは異なる線形アクセス。

### 256色モード（CRTMOD 10 等）

1 ドット＝1 バイト（パレットインデックス）。座標計算: `Address = $C00000 + Y * 512 + X`

- ストライド = 512 バイト/ライン（VRAM ページ幅 512 ドット）
- 65536色モードとはストライドもドットサイズも異なるので注意
- 詳細は `00_elf2x68kクロス開発.md` を参照

## 画面プレーン構成

| プレーン | データ形式 | 最大解像度 | 特徴 |
| --- | --- | --- | --- |
| グラフィック | ビットマップ | 1024×1024 | 16/256/65536色。自由描画 |
| テキスト | ビットマップ | 1024×1024 | 漢字含む4プレーン。GUI/ワープロ用 |
| BG | タイル（セル） | 512×512 | 2面。高速スクロール、低負荷 |
| スプライト | タイル | 16×16/1dot | 最大128個。動体高速移動用 |

## 65536色モード（ダイレクトカラー）

1 ワード内に直接カラーデータ: `Low bit(L) + 5bits Blue + 5bits Green + 5bits Red`（RGB555）。

L ビット（輝度）を含む。16 色/256 色モードではパレット経由で RGB555 処理。

## ビデオコントローラ機能

- **プライオリティ制御**: レジスタひとつで各プレーンの重なり順を瞬時変更。
- **特殊表示**: 半透明合成、オーバースキャン、インターレス等をハードウェア処理。
- **カラーパレット**: 65,536 色中 16/256 色を選択。ハードウェアが RGB555 として高速処理。
- 合成・透過処理は全てハードウェア完結。CPU はレジスタに優先順位を書き込むだけ。

## 高速化技術

1. **スクロール制御**: プレーン別スクロールレジスタ操作で CPU 負荷ゼロの画面移動。
2. **高速クリア**: GVRAM 広域を瞬時消去する専用回路。
3. **DMAC Ch2 連携**: アレイチェイン/リンクアレイチェインで不連続 VRAM 領域への矩形転送を CPU 介さず完結（詳細は `02_DMA.md` 参照）。
4. **ラスタコピー**: ビデオコントローラ固有機能。VRAM 間コピーを専用回路で高速実行。

## VRAM アクセスの同期

VRAM へのアクセスは **V-Blank 期間**に集中させること。表示期間中の大量書き込みは走査と競合し、ノイズ（雪）やバスウェイトが発生。

- V-DISP 信号: GPIP4（`$E88001` bit4）で監視。0 = 帰線期間（書き込み可能）。

## 初期化フロー

1. Supervisor モード移行
2. CRTC レジスタで画面モード設定 (CRTMOD)
3. テキストカーソルOFF (B_CUROFF)
4. **グラフィック画面ON (G_CLR_ON, IOCS 0x90)** — これを忘れるとGVRAM描画が表示されない
5. グラフィックパレット設定 (GPALET, IOCS 0x94)
6. ビデオコントローラでプライオリティ設定
7. DMAC（Ch2）等で初期背景/フォントデータを VRAM 展開

> **G_CLR_ON が未呼び出しの場合**: GVRAMに書き込んでもグラフィックプレーンが表示されず、スプライトのみ表示される。elf2x68k 環境では raw trap #15 で `D0=0x90` を呼び出すこと。

### CRTC スクロールレジスタ

256色モード等で、表示ウィンドウを VRAM ページ上で移動するハードウェアスクロール。

| レジスタ | アドレス | C言語 | 値域 |
| --- | --- | --- | --- |
| Scroll 0 X | $E80018 | `((volatile uint16_t*)0xE80000)[12]` | 0-511 |
| Scroll 0 Y | $E8001A | `((volatile uint16_t*)0xE80000)[13]` | 0-511 |

値を増やすと表示ウィンドウが右/下に移動し、画面上のコンテンツが左/上に流れる。

## 高度なグラフィック制御とVRAMスクロール (C言語実装)

### 1. マクロによる書き込み専用レジスタのシャドウ管理

X68000のCRTC（ビデオコントローラ）やビデオシステムのレジスタの多くは **WRITE ONLY**（読み出し不可）であるため、メインRAM上に「シャドウレジスタ（状態変数）」を保持し、それを一括してハードウェアレジスタ（`$E80018` 等）に反映する設計が必須となる。
カンマ演算子を使用したマクロの活用例：

```c
#define SET_SC0_X(CRTC, VAL) ((CRTC).sc0_x_reg = (VAL), (CRTC).sc1_x_reg = (VAL))
```

これにより、画面0と画面1のスクロールレジスタ値を副作用なく一発で更新しつつ、関数呼び出しのスタックオーバーヘッドを完全に排除する。

### 2. VRAM 球面スクロールとアドレッシング

グラフィックVRAMは512×512ドット（標準表示枠は256×256）。VRAMの上下左右が論理的につながった「球面状の巡回（Spherical Scroll）」をサポートしている。

- **VRAM 1ライン分のサイズ**: 256色モード時は1ドットあたり2バイト。よって横1ラインは `512 * 2 = 1024バイト`（`0x400`）。
- **非表示領域への先行描画**:
  タイル高さが16ドットの場合、VRAM終端（`0xc80000`）から `0x400 * 16` バイト差し引いた領域から描き始めることで、スクロールによってビューポートがVRAMの底を越えてラップアラウンドした瞬間に、最上部に現れる「隠しバッファ」に先行して次のタイル行を書き込むことができる。

  ```c
  unsigned short *vram = (unsigned short *) (0xc80000 - 0x400 * 16);
  ```

- **アドレスエラーの回避**:
  MC68000は奇数アドレスへのワード/ロングワードアクセスを許容せず、発生時に Address Error でシステムが停止する。`(unsigned short *)` キャストを用いて明示的に16ビット境界をアラインしアライメント違反を防止する。

### 3. タイル転送の極限最適化（ループ展開）

ループカウンタの判定（DBRA命令）に伴うオーバーヘッドを削減するため、16×16タイル転送時に16回連続の代入（ループ展開）を行う。

```c
void write_vram_tile(unsigned short *vr, unsigned short *dat) {
    int i;
    for (i = 0; i < 16; i++) {
        unsigned short *p = vr;
        /* 1ライン（16ドット）分のデータをループ展開で高速転送 */
        *p++ = *dat++; *p++ = *dat++; *p++ = *dat++; *p++ = *dat++;
        *p++ = *dat++; *p++ = *dat++; *p++ = *dat++; *p++ = *dat++;
        *p++ = *dat++; *p++ = *dat++; *p++ = *dat++; *p++ = *dat++;
        *p++ = *dat++; *p++ = *dat++; *p++ = *dat++; *p++ = *dat++;
        vr += 0x200; /* 次のラインへ移動 (512ワード分のアドレス加算) */
    }
}
```

C言語のレベルで記述しながら、コンパイラに対してアセンブラの `MOVEM` 命令や連続 `MOVE` に近い高スループットな転送を促す。

### 4. 垂直帰線（VSYNC）同期と排他制御

- 割り込みルーチン内でスクロールレジスタを書き換える。
- メインループがVRAMを書き換えている最中にスクロールレジスタが書き換わると、画面表示が不連続になり激しいフリッカーが発生する。
- 対策として、VRAM書き換え中は `scroll_flag = 0` として割り込み側のレジスタ更新を抑制し、完了後に `scroll_flag = 1` に戻す相互排他（Mutual Exclusion）を実装する。
- 具体的な制御コードの全体像は [map.c](samples/map.c) を参照。
