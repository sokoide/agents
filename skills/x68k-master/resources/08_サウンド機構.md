# X68000 サウンド

X68000が「パーソナルワークステーション」として1987年に産声を上げた際、その設計思想の核となったのは、ホビーPCの枠を超えた「表現力の解放」であった。マルチプロセッサ的なカスタムIC群をMC68000の広大なリニアメモリ空間に統合する中で、サウンド機構は単なるBGM演奏デバイスではなく、システムのリアルタイム性を象徴する最重要コンポーネントとして定義されている。

本ドキュメントでは、X68000のサウンドハードウェアを低レイヤから掌握し、バスサイクル、ウェイトステート、および周辺LSIとの調停を最適化するための実装リファレンスを提示する。

---

## 1. X68000におけるサウンド機構の戦略的アーキテクチャ

### システムI/O領域の配置とバス保護

X68000のメモリマップにおいて、サウンド関連ハードウェアは「システムI/O領域（E80000–EFFFFF）」に厳格に配置されている。

- **FM音源 (OPM: YM2151):** レジスタ番号 `$E90001`、データ `$E90003`
- **ADPCM (MSM6258V):** コマンド/ステータス `$E92001`、データ `$E92003`

この領域へのアクセスにおける最大の注意点は、ハードウェアレベルのプロテクションである。CPUからのファンクションコード信号（FC0-FC2）により、この領域は**スーパーバイザモード**でのアクセスのみが許可されている。ユーザーモード（Human68K上の通常のアプリケーション実行状態）でこれらのアドレスを直接叩こうとすれば、即座にバスエラー例外が発生する。これはシステム資源を不用意な競合から守るためのアーキテクチャ上の必然である。

### 主要構成要素の定義

サウンド機構を構成する要素は、以下の3大コンポーネントに集約される。

1. **FM音源 (OPM):** 8音同時発声が可能な旋律楽器の中核。
2. **ADPCM (MSM6258V):** 音声や自然音を担う、ハードウェア転送を前提としたサンプリング音源。入出力ともモノラルで、パンポット制御（左右中央の振り分け）はPPI（8255）で付加。
3. **SCC (Z8530):** MIDI通信等のシリアル制御を担い、外部音源との精密な同期を司る通信中枢。

### サウンド系の信号経路

ADPCMの出力はバッファアンプを通した後、左右に振り分けられ、FM音源ICの出力と合成される。振り分けにはPPI（8255）の出力ピンが使用され、PC0が左チャンネル、PC1が右チャンネルに対応する。出力が1（Highレベル）のとき、該当チャンネルへの出力はOFFとなる。両チャンネルともONのとき、聴感上、中央から出力されているように聞こえる。

ハードウェアリセット後は8255のI/Oピンはすべて入力となるため、ADPCMの出力は両方ともOFFに、8255のイニシャライズ直後は出力ピンはすべて"0"（Lowレベル）となるため、ADPCM出力は両チャンネルともONになる。

## 2. FM音源 (OPM: YM2151) のハードウェア制御と内部構造

### OPMの機能的役割とスロット構造

OPM (YM2151) は、サイン波の変調に基づくFM合成を行うLSIである。音色の核となるのは4つの「スロット（オペレータ）」であり、これらをアルゴリズムによって直列・並列に組み合わせることで、複雑な倍音構造を動的に生成する。システムアーキテクトは、エンベロープジェネレータ（EG）とLFOによる時間的変調を制御し、低レイヤでの音色エディットを完結させる必要がある。

OPMは8つのチャンネルを持ち、各チャンネルはM1（モジュレータ1）、M2（モジュレータ2）、C1（キャリア1）、C2（キャリア2）の4スロットで構成される。M1スロットだけは自分自身の出力を入力とするフィードバック回路を持ち、FLパラメータでフィードバック量を指定する。

### スロットの基本演算

SIN波形テーブルに矩形波（フェーズジェネレータ出力）と他のスロット出力を加算して入力すると、複雑な波形が生成される。2スロット直列接続の場合の出力は：

```
A(out) = SIN(θ + ZSIN(θ)α)
```

θは位相（時間）、Zは初段スロットの出力レベル、αはEGの出力に相当。

### レジスタ・インターフェース

CPUからOPMへのアクセスは2ポート構成。

| アドレス | アクセス | データ | 説明 |
| :--- | :--- | :--- | :--- |
| `$E90001` | W | レジスタ番号 | レジスタ番号設定ポート |
| `$E90003` | R/W | データ | データRead/Writeポート |

ライトレジスタ（設定書き込み）とリードレジスタ（ステータス確認）を介して行われる。リード時はレジスタ番号に関係なく、つねにステータスレジスタが読み出される。

### OPMのリードレジスタ（ステータス）

| bit 7 | bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| BUSYフラグ | - | - | - | - | - | タイマA | タイマB |

- **BUSYフラグ**: 1=データ書き込み中（CPUは次の書き込みを行ってはいけない）、0=通常動作
- **タイマA**: 1=タイマAオーバーフロー発生
- **タイマB**: 1=タイマBオーバーフロー発生

### OPMのライトレジスタ一覧

| レジスタ範囲 | 内容 |
| :--- | :--- |
| $00-$1F | ノイズ、タイマ、LFOなど（チップ内1つの設定） |
| $20-$3F | チャンネル単位の設定 |
| $40-$FF | スロット単位の設定 |

主要レジスタ：

| レジスタ | 説明 |
| :--- | :--- |
| $08 | KON（キーオン/オフ制御）。下位3ビットでチャンネル選択、bit3-6で各スロットのON/OFF |
| $0F | ノイズ制御。bit7=NE（イネーブル）、下位5ビット=NFRQ（周波数） |
| $10-$11 | タイマA設定（10ビット）。Ta = 64×(1024-CLKA)/4000(ms) |
| $12 | タイマB設定（8ビット）。Tb = 1024×(256-CLKB)/4000(ms) |
| $13 | タイマ制御。bit0/1=動作開始、bit2/3=IRQEN、bit4/5=F-RESET、bit7=CSM |
| $18 | LFO周波数設定 |
| $19 | PMD/AMD設定。bit7=1でPMD、0でAMD |
| $1B | 汎用出力/LFO波形。bit7=CT2（ADPCMクロック切替: 1=4MHz, 0=8MHz）、bit6=CT1（FDC READY強制） |
| $20-$27 | チャンネル構成。bit6-7=R/L出力選択、bit3-5=FL（フィードバック率）、bit0-2=CON（接続方式8種） |
| $28-$2F | KC（キーコード）。bit4-6=OCT、bit0-3=NOTE |
| $30-$37 | KF（キーフラクション）。上位6ビットで約1.6セント単位の微調整 |
| $38-$3F | PMS/AMS。bit4-6=PMS（周波数変調度）、bit0-1=AMS（出力レベル変調度） |
| $40-$5F | DT1/MUL。DT1=微調整、MUL=倍率設定（0=1/2倍、1以上=設定値倍） |
| $60-$7F | TL（トータルレベル）。0.75dB単位。L = L₀ - 0.75×TL (dB) |
| $80-$9F | KS/AR。KS=KCによるレート補正、AR=アタックレート |
| $A0-$BF | AMS-EN/D1R。AMS-EN=LFO→EG変調ON/OFF、D1R=ファーストディケイ |
| $C0-$DF | DT2/D2R。DT2=±0/600/781/950セント、D2R=セカンドディケイレート |
| $E0-$FF | D1L/RR。D1L=サスティンレベル、RR=リリースレート |

### コネクションの種類（CON=0〜7）

| CON | 接続方式 |
| :--- | :--- |
| 0 | M1→M2→C1→C2→OUT（すべて直列） |
| 1 | M1→M2, C1→C2→OUT（2系統並列） |
| 2 | M1→M2, C2→OUT |
| 3 | M1→C1, M2→C2→OUT |
| 4 | M1, M2, C1, C2→OUT（すべて並列） |
| 5 | M1→M2, M1→C1, C2→OUT |
| 6 | M1→M2, M1→C1, M1→C2→OUT |
| 7 | M1→C1, M2→C2→OUT |

### KC（キーコード）と音程指定

X68000ではOPMに4MHzクロックが与えられるが、OPMの音階テーブルは3.579545MHzで最小誤差になるように設計されている。このため、約192.27セント（約1音分）高い音が出る。OPMマニュアルでNOTE=$AがAの音のところ、X68000ではBの音になる。

出力周波数：

```
f(out) = 440 × 2^(OCT-4+(n-9)/12)
```

### 割り込み制御：MFP GPIP3 ($E88001 bit 3)

OPMは演奏テンポの管理のために内部タイマ（Timer A/B）を保持しており、これがシステムへの割り込み要求信号（FMIRQ）を生成する。

- **MFPとの結合:** FMIRQは**MFP (MC68901) のGPIP3端子（$E88001 bit 3）**に直結されている。
- **割り込み優先順位:** FMIRQ (GPIP3) は、RTC ALARM (GPIP0) や EXPON (GPIP1) よりも高い優先順位を持つが、V-DISP (GPIP4) や H-SYNC (GPIP7) といった映像同期信号、および CIRQ (GPIP6) よりも低く設定されている。これは音楽演奏が映像描画のリアルタイム性を妨げないための設計的バランスである。

### タイマA/Bによる一斉キーオン

タイマ制御レジスタ（$13）のbit7（CSM）を1に設定すると、タイマAのオーバーフロー時に全スロットが自動的にキーオンされる。複数チャンネルの位相を確実に揃えてスタートさせる場合に有効。

## 3. ADPCM再生機構と高能率データ転送の仕組み

### ADPCM制御の設計思想

ADPCM（適応差分パルス符号変調）は、4ビットのデータで16ビット相当の動的範囲を表現する高能率アルゴリズムを採用している。X68000におけるADPCM再生の要諦は、CPUをデータ転送から完全に解放し、自律的なオーディオ・ストリーミングを実現することにある。

### ADPCM LSI：MSM6258V

X68000では沖電気のMSM6258Vを使用。入出力ともモノラルのため、パンポット制御回路を付加している。

主な仕様：

- **ADPCMビット数**: 4ビット（3ビットモードもあるがX68000では4ビット固定）
- **A/Dコンバータ**: 8ビット精度
- **D/Aコンバータ**: 10ビット精度
- **サンプリング周波数**: クロックの1/512、1/768、1/1024から選択

### ADPCMのサンプリング周波数

| 基本クロック | 1/512 | 1/768 | 1/1024 |
| :--- | :--- | :--- | :--- |
| 4MHz | 7.8kHz | 5.2kHz | 3.9kHz |
| 8MHz | 15.6kHz | 10.4kHz | 7.8kHz |

1秒あたり使用メモリ量はサンプリング周波数の半分（例: 15.6kHz → 7.8KB/s）。

### ADPCM関係のレジスタ

| アドレス | アクセス | 説明 |
| :--- | :--- | :--- |
| `$E92001` | R | ADPCMステータス（bit7=PLAY/REC、bit1=ST、bit0=REC） |
| `$E92001` | W | ADPCMコマンド（bit2=REC ST、bit1=PLAY ST、bit0=SP停止） |
| `$E92003` | R/W | ADPCMデータ（上位4ビット=n+1、下位4ビット=n） |

### ADPCMコマンドレジスタ

| bit | 名前 | 説明 |
| :--- | :--- | :--- |
| 0 | SP | 1=録音/再生動作停止 |
| 1 | PLAY ST | 1=ADPCM再生開始 |
| 2 | REC ST | 1=ADPCM録音開始 |

再生動作が終了すると、最後に与えたデータを繰り返し出力するため、停止コマンドを必ず書き込む。停止時、出力レベルは最後の状態のまま保持され、次回再生開始時に1/2 VDDに戻されるため、最後の出力レベルが1/2 VDD近辺でないと「ボツッ」というノイズが出る。

### PPI（8255）によるパンポット・サンプリングレート制御

PPI（8255）のポートC（`$E9A005`）で以下を制御：

| bit | 説明 |
| :--- | :--- |
| 0-1 | PCM PAN（パンポット）。00=左右ON、01=右のみ、10=左のみ、11=左右OFF |
| 2-3 | Sampling Rate。00=1/512、01=1/768、10=1/1024、11=未使用 |
| 4 | ジョイスティック2コントロール |
| 5 | ジョイスティック1コントロール |
| 6-7 | ジョイスティックトリガ |

ビットセット/リセット機能（`$E9A007`）を使うと、ポートCの任意ビットを1つだけ変更できる。

### OPMレジスタ$1BによるADPCMクロック切替

OPMの汎用出力端子CT2（$1Bレジスタbit7）でADPCM基本クロックを選択：

- **1**: 4MHz
- **0**: 8MHz

このレジスタは書き込み専用で読み出せないため、他のビットの値に注意が必要。

### DMACによる転送

ADPCMのデータ転送は、DMAC (HD63450) の**チャンネル#3**が専有する。DMACの設定方法はDMA章を参照。

## 4. サウンド制御におけるシステム・インテグレーション

### ホールドなしサイクルスチールモードによるバス占有の調停

ADPCMの高速転送を制御する上で、DMACの設定は「**ホールドなしサイクルスチールモード**」が絶対条件である。

- **採用の理由:** もし「バーストモード」で転送を行った場合、DMACが全データ転送を完了するまでバスを占有し続け、CPUの動作が完全に停止する。これは60Hzで駆動するゲームロジックや、他の高位割り込み（V-DISP等）に壊滅的なジッタを引き起こす。
- **利点:** 1オペランド転送ごとにバスをCPUへ返却（Bus Arbitration）することで、音切れを発生させることなく、メインプログラムのリアルタイム性を担保している。

### MFPタイマAによる垂直同期演奏制御

演奏テンポの精密な同期には、MFPのタイマAを**イベントカウントモード**で使用することを推奨する。

- **V-DISP同期:** タイマAの入力端子（TAI）にはCRTCのV-DISP信号が接続されている。これを利用することで、ソフトウェアによる生の時間計測ではなく、画面リフレッシュレート（60Hz）に基づいた絶対的なタイミングでの演奏同期が可能となる。

### AIおよび自動化制御への低レイヤ提言

AIがX68000のサウンド資源を自律管理する場合、単なるレジスタ操作ではなく、以下のステータス監視による例外処理を組み込むべきである。

1. **CER ($E84001) の異常検知:**
    - `$1C` (Count Error): MTC（メモリトランスファカウンタ）が意図せず0になった場合。
    - `$02` (Operation Timing Error): チャンネル動作中に不正なレジスタ書き込みが行われた場合。
    - `$11` (Software Abort): SABビットによる強制停止の確認。
2. **MFP IPRA/IPRB の滞留監視:**
    - 特定の割り込み（FMIRQ等）がペンディング状態にある時間を監視し、バス占有率がサウンド再生に与える影響を動的に算出・調停する。

### 結びに

X68000のサウンドハードウェアは、FM音源の変調論理と、DMAによるADPCMの自律転送が、MFPという精密なタイマ管理機構によって統合された傑作である。本リファレンスが示すレジスタ配置、バスサイクル、および割り込み優先順位の理解こそが、この名機のポテンシャルを極限まで引き出し、真のパーソナルワークステーションとしての威信を現代に再現するための礎となる。
