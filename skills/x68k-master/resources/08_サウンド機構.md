# X68000 サウンド機構

FM 音源（OPM）・ADPCM・シリアル通信（SCC）の 3 コンポーネントで構成。

## 配置

| デバイス | アドレス |
| --- | --- |
| FM音源 (OPM: YM2151) | `$E90000` |
| ADPCM | `$E9C000`（※ `$EA0000` は ADPCMレジスタ別記載あり） |

## FM音源 (OPM: YM2151)

8 音同時発声の FM 合成 LSI。4 スロット（オペレータ）をアルゴリズムで直列/並列に組み合わせ、EG/LFO で時間的変調を制御。

### レジスタアクセス

- **ライトレジスタ（`$E90000`）**: アドレス指定 → データ書き込みの連続操作。
- **ウェイト必須**: OPM 内部処理時間を考慮したウェイトステート挿入が必要。書き込み完了を待たずに次のコマンドを投入するとデータ破棄の可能性。
  - アセンブラレベルのディレイ挿入、またはビジーフラグ監視が必須。

### 割り込み

- OPM 内部タイマ（Timer A/B）が FMIRQ 信号を生成。
- FMIRQ は MFP GPIP3（`$E88001` bit3）に直結。
- 優先順位: H-SYNC / CIRQ > FMIRQ > RTC ALARM / EXPON（映像同期を妨げない設計）。

## ADPCM 再生

4bit データで 16bit 相当の動的範囲を表現する高能率方式。CPU をデータ転送から解放し、自律的なオーディオストリーミングを実現。

### DMA 転送

- **DMAC Ch3** が専有。
- **デュアルアドレスモード**: 「メモリ読み出し」と「ADPCM レジスタ書き込み」の 2 つの独立バスサイクルを実行。
- **PCL3**: ADPCM 制御ロジックがデータ補充要求時にアクティブにする DMA 要求信号。

### ホールドなしサイクルスチールモード（必須）

バーストモードでは DMAC が全転送完了までバスを占有し、CPU が完全停止する。60Hz ゲームロジックや高位割り込みに壊滅的ジッタを引き起こすため、**1オペランドごとにバスを CPU に返却するサイクルスチールモードが必須**。

## MFP Timer A による演奏同期

Timer A をイベントカウントモードで使用すると、TAI に入力されている V-DISP 信号で画面リフレッシュレート（60Hz）に基づく絶対タイミング同期が可能。

## エラー監視

- **CER（`$E84001`）**: `$1C`（カウントエラー）、`$02`（動作タイミングエラー）、`$11`（ソフトウェアアボート）。
- **MFP IPR**: FMIRQ 等のペンディング滞留時間を監視し、バス占有率への影響を動的に調停。
