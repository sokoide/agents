# elf2x68k クロス開発環境

elf2x68k (macOS/Linux → X68000 Human68k クロースコンパイラ) での実践的知見。
PX68k エミュレータでの検証結果を含む。

## 1. ツールチェーン構成

- コンパイラ: `m68k-xelf-gcc` (GCC, m68k-elf ターゲット)
- リンカ: `m68k-xelf-gcc` + elf2x68k コンバータ
- ライブラリ: `libx68kiocs.a`, `libx68kdos.a` (IOCS/DOS スタブ)
- 共通フラグ: `-O2 -m68000 -Wall`

## 2. IOCS ライブラリ関数は信頼できない（重要）

elf2x68k 付属の `libx68kiocs.a` に含まれる IOCS ライブラリ関数（`_iocs_sp_init`, `_iocs_sp_on`, `_iocs_gpalet` 等）は、**スプライト・グラフィック制御において正常動作しない**。

### 影響が確認された関数

- `_iocs_sp_init()` → スプライト初期化失敗
- `_iocs_sp_cgclr()` → PCG クリア失敗
- `_iocs_sp_on()` → スプライト表示オン失敗
- `_iocs_gpalet()` → 未検証だが同様の懸念あり
- `_iocs_crtmod()` → 画面モード変更失敗（黒画面）

### 影響がない（動作する）関数

- `_iocs_b_super()` → Supervisor モード移行は正常
- `_iocs_b_keysns()` → キー入力判定は正常
- `_iocs_b_keyinp()` → キー入力取得は正常

### 回避策: raw trap #15 インラインアセンブラ

全ての IOCS 呼び出しを `trap #15` の raw インラインアセンブラで直接実行する。

```c
/* 引数なし IOCS 呼び出し */
{
    register int trapNo asm("d0") = 0xC0; /* SP_INIT */
    asm("trap #15" :: "d"(trapNo));
}

/* 引数1個 IOCS 呼び出し */
{
    register int trapNo asm("d0") = 0x10; /* CRTMOD */
    register int mode   asm("d1") = 10;
    asm("trap #15" :: "d"(trapNo), "d"(mode));
}

/* 引数2個 IOCS 呼び出し */
{
    register int trapNo asm("d0") = 0x94; /* GPALET */
    register int pal    asm("d1") = 1;
    register int col    asm("d2") = 0xFBE1; /* white */
    asm("trap #15" :: "d"(trapNo), "d"(pal), "d"(col));
}

/* 引数4個 IOCS 呼び出し (SP_PALET) */
{
    register int trapNo   asm("d0") = 0xCF;
    register int pal_code asm("d1") = 0x80000000 | 0; /* bit31=set */
    register int block    asm("d2") = 1;
    register int col_code asm("d3") = 21140;
    asm("trap #15" :: "d"(trapNo), "d"(pal_code), "d"(block), "d"(col_code));
}
```

> **[注意]** `"+r"` 出力制約や clobber リストを追加すると `asm specifier conflicts with asm clobber list` エラーになる。上記の入力のみ制約 `"d"()` のパターンを使用すること。

## 3. IOCS 関数番号一覧

`libx68kiocs.a` を逆アセンブルして確認した実際の IOCS 関数番号 (D0 レジスタ値)。

### 画面制御系

| 関数名 | IOCS番号 (D0) | 引数 | 備考 |
| --- | --- | --- | --- |
| CRTMOD | 0x10 | D1=モード番号 | 画面モード設定 |
| B_CUROFF | 0x1F | なし | テキストカーソルOFF |
| B_CURON | 0x1E | なし | テキストカーソルON |
| G_CLR_ON | **0x90** | なし | **グラフィック画面表示ON（必須）** |
| GPALET | **0x94** | D1=パレット番号, D2=色 | **グラフィックパレット設定** |

> **[よくある間違い]** GPALET の IOCS 番号を 0x11 とする情報があるが、正しくは **0x94**。0x11 は CRTMOD 近傍の別関数。

### スプライト制御系

| 関数名 | IOCS番号 (D0) | 引数 | 備考 |
| --- | --- | --- | --- |
| SP_INIT | 0xC0 | なし | スプライト初期化 |
| SP_ON | 0xC1 | なし | スプライト表示ON |
| SP_OFF | 0xC2 | なし | スプライト表示OFF |
| SP_CGCLR | 0xC3 | D1=PCG番号 | PCGパターンクリア |
| SP_PALET | 0xCF | D1=0x80000000\|番号, D2=block, D3=色 | bit31=設定モード |

### システム系

| 関数名 | IOCS番号 (D0) | 引数 | 備考 |
| --- | --- | --- | --- |
| B_SUPER | 0x20 | D1=0 (supervisor移行) | IOCSライブラリ経由なら動作 |
| B_KEYSNS | 0x24 | なし | IOCSライブラリ経由なら動作 |
| B_KEYINP | 0x25 | なし | IOCSライブラリ経由なら動作 |

## 4. G_CLR_ON の必要性

GVRAM に描画した内容を画面に表示するには、**G_CLR_ON (IOCS 0x90) の呼び出しが必須**。

- G_CLR_ON なし: GVRAM に書き込んでも何も表示されない（スプライトのみ表示）
- G_CLR_ON あり: GVRAM の内容がグラフィックプレーンとして表示される

呼び出しタイミングは CRTMOD 後、メインループ前でよい。

## 5. CRTMOD モード一覧 (主要)

| モード番号 | 解像度 | 色数 | 水平周波数 | 備考 |
| --- | --- | --- | --- | --- |
| 9 | 768×512 | 16 | 31kHz | 高解像度 |
| 10 | 256×256 | 256 | 31kHz | **ゲーム向け。PX68k で動作確認済み** |
| 16 | 512×512 | 16 | 15kHz | デフォルト |

CRTMOD(10) は 256×256 ドット表示だが、VRAM ページは 512×512。
CRTC スクロールレジスタで 256×256 の表示ウィンドウを移動可能。

## 6. 256色モードの GVRAM フォーマット

CRTMOD(10) (256色モード) における GVRAM アクセス:

- **1ドット = 1バイト** (パレットインデックス 0-255)
- **ストライド = 512バイト/ライン** (VRAMページ幅 512ドット)
- **GVRAM先頭 = $C00000**
- **総サイズ = 512×512 = 262,144バイト** (1ページ分)

```c
/* ピクセル (x, y) へのアクセス */
volatile uint8_t *p = (volatile uint8_t *)0xC00000;
p[y * 512 + x] = palette_index; /* 0-255 */
```

> 65536色モードの「1ドット=1ワード」「ストライド=1024バイト」とは異なるので注意。
> `07_画面制御.md` の記述は主に65536色モード基準。

### パレット色フォーマット

グラフィックパレット (GPALET) の色データ (D2) は RGB555 + Valid ビット:

```c
#define PAL(r,g,b) ((uint16_t)((((g)&0x1F)<<11)|(((r)&0x1F)<<6)|(((b)&0x1F)<<1)|1))
/* 例: 白 = PAL(31,31,31) = 0xFBE1 */
```

## 7. CRTC スクロールレジスタ

256色モードでのハードウェアスクロール。表示ウィンドウ (256×256) を VRAMページ (512×512) 上で移動する。

| レジスタ | アドレス | C言語アクセス | 機能 |
| --- | --- | --- | --- |
| Scroll 0 X | $E80018 | `CRTC[12]` | 水平スクロールオフセット (0-511) |
| Scroll 0 Y | $E8001A | `CRTC[13]` | 垂直スクロールオフセット (0-511) |

```c
#define CRTC ((volatile uint16_t *)0xE80000)

/* 水平スクロール（左にスクロール＝値を増加） */
scroll_x = (scroll_x + 1) & 511;
CRTC[12] = scroll_x;
```

値を増やすと表示ウィンドウが右に移動し、画面上のコンテンツが左に流れる。

## 8. VSync 待ち（MFP GPIP4 ポーリング）

最も確実で低オーバーヘッドな VSync 待ち手法。

```c
volatile char *mpf = (volatile char *)0xe88001;
while (!((*mpf) & 0x10)) ;  /* 表示期間開始を待つ */
while  ((*mpf) & 0x10) ;    /* 垂直帰線期間（ブランク）を待つ */
```

- `$E88001` は MFP の GPIP レジスタ
- Bit 4 = V-DISP 信号 (1=表示期間, 0=帰線期間)
- この順序で両エッジを待つことで1フレームを正確に同期

## 9. ビルド・実行手順

```bash
export PATH="/opt/homebrew/Cellar/elf2x68k/20260331/bin:$PATH"
make clean && make && make install
```

- `make install` は `cp g1.x ~/tmp` で PX68k の HDD イメージに配置
- PX68k の Human68k 上で `g1.x` を実行
