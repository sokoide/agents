# X68000 フロッピーディスクドライブ（FDD）

## 1. X68000 FDDシステムのアーキテクチャ概要

X68000が「パーソナルワークステーション」を冠する所以は、その先進的な周辺デバイス制御にあります。中でもフロッピーディスクドライブ（FDD）システムは、単なるストレージを超えたインテリジェントなサブシステムとして設計されました。標準装備されたオートイジェクト・オートクランプ機構や、I/Oコントローラによる各種オプション機能の統合は、当時のユーザーエクスペリエンスを劇的に向上させた象徴的機能です。

### FDDインタフェースの構成

システムアーキテクチャの視点から見ると、FDD制御は以下の3要素の高度な同期によって成立しています。

- **MPU (68000):** システム全体を統括し、MMIO（メモリマップドI/O）を介して各コントローラへコマンドを発行します。
- **FDC (PD72065):** 日本電気製のフロッピーディスクコントローラです。物理層のビットストリーム、シーク動作、セクタ読み書きの論理制御を司ります。
- **VFO (SED9420AC):** ディスクから読み出された波形からデータとクロックを分離するIC（データセレクタ）。FDCと合わせてディスクインタフェースを構成します。
- **DMAC (HD63450):** 高速データ転送を実現する要です。FDCとメインメモリ間のデータバイパスを担います。

### ハードウェア主要仕様

- **ドライブ形式:** 5インチ・フロッピーディスクドライブ。
- **標準装備数:** 2基（最大4基まで拡張可能）。
- **記憶容量:** 1667KB（アンフォーマット）、1065KB（フォーマット済み / IBM準拠・高密度モード・256バイト/セクタ・26セクタ/トラック）。
- **データ転送速度:** 500kbit/s。
- **トラック仕様:** 77トラック/SIDE、96 TPI、2ヘッド、MFM調方式。
- **独自機能:** ソフトウェア制御によるオートイジェクト、オートクランプ。内蔵FDDは2HDのみサポート（2DD/2Dは不可）。
- **制御インタフェース:** I/Oコントローラ（シャープ製LSI）およびOPM（YM2151）のCT2端子を介した特殊信号制御。

### 技術的意義：DMACチャンネル#0の専有

X68000において、DMAC（HD63450）の全4チャンネルのうち、**チャンネル#0はFDD専用**として設計段階から固定されています。これは、ディスク入出力というクリティカルな処理に対し、バス帯域を確実に保証するためのアーキテクチャ的決断です。後述する「サイクルスチールモード」での運用により、MPUの処理を長時間停止させることなく、バックグラウンドでの安定したデータ転送を可能にしています。

物理的なエンベロープ信号の制御方法を理解した上で、ソフトウェアがハードウェアを支配するための窓口であるI/Oポートの詳細へ進みます。

## 2. FDD関連ポートとI/Oアドレッシング

メモリマップドI/Oを採用するX68000において、FDD制御は特定のアドレス空間に配置されたレジスタ群を介して行われます。これらは物理的なドライブの選択やモータ制御といった「外枠」の信号を司ります。

### FDCポート（$E94001, $E94003）

FDCへのアクセスは、コマンド・データとも`$E94003`で受け渡しを行い、ステータスを`$E94001`で読み出します。`$E94001`への書き込みは、FDCの初期化などの非常時に使用されるコマンドに限られます。

| デバイス | アドレス | R/W | 備考 |
| :--- | :--- | :--- | :--- |
| FDC (PD72065) | `$E94001` | R | FDCステータスレジスタ (MSR) |
| | `$E94001` | W | FDCコマンドレジスタ（SET STANDBY, RESET STANDBY, SOFTWARE RESET のみ使用可） |
| | `$E94003` | R/W | FDCデータレジスタ（コマンド/パラメータ/リザルトの受け渡し） |

> **注意:** FDCのPD72065はUS0, US1信号で4台まで選択できるが、X68000ではこの信号は使用せず、I/Oコントローラから出力する。ディスクアクセス時にはFDCにコマンドを書き込む前にI/Oコントローラでアクセスするドライブを選択しておく必要がある。

### I/Oコントローラ：FDD関連ポート（$E94005, $E94007）

X68000のI/Oコントローラ（シャープ製LSI）は、割り込みやオートイジェクト、LEDなどのオプション機能の制御を担います。

| デバイス | アドレス | R/W | bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| I/Oコントローラ | `$E94005` | R | DISKIN | ERROR | - | - | - | - | - | - | ドライブステータス |
| | `$E94005` | W | LED | EJECT | EJECT | - | DRIVE | DRIVE | DRIVE | DRIVE | ドライブ制御 |
| | `$E94007` | W | MOTOR | - | 2HD | - | - | ACCESS | DRIVE | DRIVE | アクセスドライブセレクト |

**`$E94005` Read: ドライブステータスレジスタ**

- **bit7 DISKIN:** ディスク挿入状態（1:挿入、0:未挿入）
- **bit6 ERROR:** ディスク誤挿入（1:誤挿入、0:それ以外）。誤挿入時は自動的に排出される。

**`$E94005` Write: ドライブコントロールレジスタ**

- **bit7 LED CTRL:** 1→"0"の変化時、ビット5〜7で選択したオプション機能が動作。
- **bit6 EJECT MASK:** イジェクトボタンの無効化（1:無効、0:有効）。
- **bit5 EJECT ON/OFF:** メディアのイジェクト（1:排出、0:しない）。
- **bit3〜0 DRIVE #3〜#0:** オプション機能を働かせるドライブ番号。

**`$E94007` Write: アクセスドライブセレクトレジスタ**

- **bit7 MOTOR ON:** 1でドライブセレクト有効＆モータON（アクセスランプ赤色）、0でモータOFF（ランプ緑色）。
- **bit4 2HD/2DD:** 1で2DD/2D、0で2HD（内蔵ドライブでは2HDのみ使用可）。
- **bit1,bit0 ACCESS DRIVE:** アクセスするドライブ番号（11:Dr3, 10:Dr2, 01:Dr1, 00:Dr0）。

### 割り込み関連ポート（$E9C001, $E9C003）

I/OコントローラはSASI（HDD）、フロッピーディスク、プリンタなどの割り込みを管理しています。

| デバイス | アドレス | R/W | bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 割り込み | `$E9C001` | R | FDC | FDD | PRT | HDD | HDDI | FDCI | FDDI | PRTI | 割り込み信号ステータス |
| | `$E9C001` | W | - | - | - | - | HDDI | FDCI | FDDI | PRTI | 割り込み信号マスク |
| | `$E9C003` | W | - | Vect | - | - | - | - | DEVICE | DEVICE | 割り込みベクタ番号 |

**`$E9C001` Read: 割り込みステータス**（上位4ビット=要求状態、下位4ビット=マスク状態）

- **bit7 FDC INT:** FDC割り込み発生中
- **bit6 FDD INT:** FDD割り込み（メディア挿抜）発生中
- **bit5 PRT INT:** プリンタBUSY
- **bit4 HDD INT:** SASIディスク割り込み発生中

**`$E9C001` Write: 割り込みマスク**（1:許可、0:禁止）

**`$E9C003` Write: 割り込みベクタ**（上位6ビット有効、下位2ビットは DEVICE で自動変更）

- **DEVICE (bit1,bit0):** 11=プリンタ、10=HDD、01=FDD、00=FDC

### OPM（YM2151）のFDD関連ポート（$E90003）

OPMのCT2端子の制御を行うレジスタ$1Bです。`$E90001`に`$1B`を書き込んでから`$E90003`にアクセスします。

- **bit6 CT2:** FDCのREADY端子を強制的にレディ状態にする（1:強制レディ、0:通常動作）。
  - ディスクの接続状態チェックに使用。このビットを1にしてRECALIBRATEを発行すると、ディスク未接続時はトラック0検出不能で異常終了する。
  - **このビットはドライブ接続チェック以外では常に0にする。**
- **bit4 CT1:** ADPCM基本クロック選択（1:4MHz、0:8MHz）。

### 「So What?」：ポート操作のリスクとタイミング

低レイヤー制御において、ポート操作の順序ミスは致命的なエラーを招きます。特に「モータON」から「FDCコマンド発行」の間には、ドライブの回転が安定するまで**約500msの待機時間**を設けるのがアーキテクトとしての推奨です。これを確認せずにコマンドを発行すると「Drive Not Ready」によるタイムアウトや、FDCがハングアップするリスクが生じます。確実なステート遷移の把握こそが、堅牢なドライバ実装の鍵となります。

ポートによる物理層の準備が整えば、次はFDC内部の状態管理プロトコルへ移行します。

## 3. フロッピーディスクコントローラ（FDC）の内部構造と状態遷移

FDC（µPD72065）との通信は、主ステータスレジスタ（MSR）を介した厳格なフェーズ管理の下で行われます。FDCは非同期動作を行うため、MPU側でのポーリング同期が不可欠です。

### 主ステータスレジスタ（MSR: $E94000）の解析

コマンドを発行する前、あるいは結果を取得する前には、必ずMSRの以下のビットをチェックする必要があります。

- **RQM (bit 7: Request for Master):** 1であればFDCがデータ転送の準備を完了しています。
- **DIO (bit 6: Data I/O):** データの方向を示します（1: FDC→MPU, 0: MPU→FDC）。
- **NDM (bit 5: Non-DMA Mode):** Non-DMAモードでのデータ転送中。（C-PHASE, R-PHASEでは常に'0'）
- **CB (bit 4: FDC Busy):** 1であればFDCがコマンドを実行中です（シーク/リキャリブレートのE-PHASEではセットされない）。
- **D3B-D0B (bit 3-0):** 各ドライブのBusy状態。1でシーク中/シーク完了割り込み保留中。

### フェーズ遷移プロトコルとポーリング・アルゴリズム

FDCとの対話は「コマンド」「エグゼキューション」「リザルト」の3フェーズを巡回します。マルチバイト命令を扱う際、**1バイト送信/受信ごとに以下のループを実行する**のが鉄則です。

1. **RQMが1になるのを待機。**
2. **DIOの状態を確認。**
    - 書き込み時：`DIO=0`であることを確認してコマンドバイトを送信。
    - 読み出し時：`DIO=1`であることを確認して結果バイトを受信。
3. この手順を逸脱すると、FDCは「プロトコル違反」を起こし、内部状態が不定となります。

### リザルトステータス（ST0 - ST3）

命令完了後、FDCは最大7バイトのステータスを返します。

- **ST0:** 終了状態。異常終了時はビット6-7で原因を特定。
- **ST1/ST2:** CRCエラーやセクタ未検出等のデータ読み書きトラブルを報告。
- **ST3:** 書き込みプロテクトやヘッド位置（トラック0）等のドライブ状態を保持。

状態管理のプロトコルを確立したところで、具体的な命令セットであるコマンド体系の解説に移ります。

## 4. FDCコマンドセットとパラメータ詳細

FDC µPD72065は、ディスク上の物理セクタを支配するための強力なコマンドセットを保持しています。

### 主要コマンド一覧

- **データ転送系:** `READ DATA`, `WRITE DATA`, `READ DELETED DATA`
- **制御・診断系:**
  - **SEEK:** 指定トラックへ移動。実行後、結果を得るには必ず`SENSE INTERRUPT STATUS`を発行する必要があります。
  - **RECALIBRATE:** トラック0へ復帰。シーク不良からの回復に必須。
  - **READ ID:** 現在位置のC(Cylinder), H(Head), R(Record), N(Number/Size)を取得。
- **設定系:**
  - **SPECIFY:** FDCのタイミングパラメータを設定。
    - **SRT (Step Rate Time):** ヘッドの移動速度。
    - **HUT (Head Unload Time):** アンロードまでの時間。
    - **HLT (Head Load Time):** ロード後の安定待ち時間。

### トラックフォーマットの重要性

ディスク上の1トラックは、INDEXパルスの後にGap 4a、SYNC、IAM (Index Address Mark)、Gap 1が続き、各セクタの情報が並びます。各セクタはSYNC、IDAM (ID Address Mark)に続いてC(シリンダ)、H(ヘッド)、R(セクタ)、N(セクタ長)のID情報が格納されます。

| 項目 | FMフォーマット | MFMフォーマット |
| :--- | :--- | :--- |
| Gap 4a | $FF x 40 | $4E x 80 |
| SYNC | $00 x 6 | $00 x 12 |
| IAM | $FC x 1 | $C2 x 3 + $FC x 1 |
| Gap 1 | $FF x 26 | $4E x 50 |

特にGap 3の長さ設定は、2HD（標準27バイト）と2DD（標準22バイト）で異なり、この設定を誤るとフォーマットや書き込み時に隣接セクタを破壊する恐れがあります。

### データ転送時間

1バイト分の転送時間は記録方式により異なり、X68000で通常使用する2HD MFMでは**16μs以内**にデータの引き取り（リード時）や書き込み（ライト時）を完了させなくてはなりません。

| 記録方式 | FM | MFM |
| :--- | :--- | :--- |
| 2HD | 32μs | 16μs |
| 2DD/2D | 64μs | 32μs |

### 「So What?」：SEEKと割り込み待機

`SEEK`コマンドは、ヘッドの物理移動を伴うため即座には完了しません。移動完了時にFDCから割り込みが発生しますが、その後に必ず`SENSE INTERRUPT STATUS`を発行してシーク成功を確認しなければなりません。これを怠ると、不正確な位置で読み書きを開始する致命的なバグに繋がります。

## 5. DMA転送と割り込み処理の同期設計

高速なFDDデータ転送において、MPUの負荷を軽減するためにDMAC（HD63450）の活用は必須です。

### DMAC設定（チャンネル#0）

Human68K等の標準環境における推奨設定値は以下の通りです。

- **DCR (Device Control):** `**$60**` (68000バスタイプ、外部要求、サイクルスチール)
- **OCR (Operation Control):** `**$B2**` (8bitポート、デバイスからメモリ、サイクルスチール)
- **SCR (Sequence Control):** `**$04**` (メモリ側インクリメント、デバイス側固定)
- **CCR (Channel Control):** `**$80**` (転送開始)

### サイクルスチールモードの挙動

サイクルスチールモードでは、FDCからのデータ要求（REQ）が発生するたびに、DMACがMPUから**1バスサイクルだけを「盗んで」**転送を実行します。これにより、MPUは処理が完全に停止することなく、僅かな速度低下のみでバックグラウンド転送を維持できます。

### 割り込みベクタの管理

- **DMAC割り込み:** 初期値は`$0F`（非初期化ベクタ）。Human68Kでは通常`$68`（チャンネル#0）が割り当てられます。
- **FDC割り込み:** I/Oコントローラ経由で発生。ベクタは`$E9C003`で設定した値の下位2ビットがデバイス（00=FDC, 01=FDD, 10=HDD, 11=PRT）で自動的に変更されます。

「ポーリングのみ」の実装は簡便ですが、実用的なシステムではDMAC完了割り込みを待機するイベントドリブン設計を採用すべきです。

## 6. 実装ガイド：FDD読み込み処理のシーケンス

これまで解説した要素を統合し、堅牢なデータ読み出しフローを構築します。

### 1. 標準読み込みシーケンス

1. **スーパバイザモード移行:** 特権レジスタアクセスの許可。
2. **モータ起動:** `$E94007`へドライブ選択とモータONを書き込み、500ms待機。
3. **DMAC初期化:** MAR（メモリ先頭アドレス）、MTC（語数）を設定し、CCRに`$80`を書き込み待機状態にする。
4. **FDCコマンド発行:** MSRをチェックしつつ、`READ DATA`およびパラメータ（C, H, R, N等）を順次書き込む。
5. **完了待機:** MSRの`CB`ビットおよびDMACの`CSR`を監視。
6. **リザルトフェーズ:** FDCから結果ステータス（ST0-2等）を読み出し、正常終了を検証。

### 2. エラー処理とリトライ戦略

FDDは物理接触メディアであるため、一度の失敗で処理を投げ出してはなりません。以下の「3-Recal-3」戦略を実装することを推奨します。

- **ステップ1:** 同一箇所を3回リトライ。
- **ステップ2:** 失敗した場合、`RECALIBRATE`を発行してヘッドをトラック0へ戻す。
- **ステップ3:** 再度`SEEK`で目標トラックへ移動し、さらに3回リトライ。 これでも失敗する場合にのみ、メディア異常として上位レイヤーへ報告します。

### 3. 実装上の注意点：C言語における`volatile`

ハードウェア制御レジスタを構造体で定義する場合、必ず`volatile`キーワードを付加してください。

```c
/* FDCおよびDMACのレジスタ定義例 */
volatile struct DMAC_REGS *dmac = (struct DMAC_REGS *)0xE84000;
volatile unsigned char *fdc_msr = (unsigned char *)0xE94000;
```

コンパイラの最適化により、ステータス待ちのループ（ポーリング）が「無意味な重複アクセス」と見なされて削除されることを防ぐためです。これは低レイヤープログラミングにおける鉄則です。

本ガイドラインに従い、物理層から論理層までを正確に制御することで、X68000のポテンシャルを最大限に引き出したFDD制御が可能となります。
