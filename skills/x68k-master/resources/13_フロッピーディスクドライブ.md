# X68000 フロッピーディスクドライブ

5 インチ FDD 2 基標準装備（最大 4 基拡張）。オートイジェクト、2HD/2DD 自動判別機能付き。DMAC Ch0 専有。

## 構成要素

- **FDC**: µPD72065（NEC 765A 互換）。物理層ビットストリーム、シーク、セクタ読み書き制御。
- **DMAC**: HD63450 Ch0 が FDD 専用。サイクルスチールモードでバックグラウンド転送。
- **ドライブ**: 5 インチ。2HD（1.2MB）/ 2DD（720KB）。

## I/O ポート（`$E94000`〜）

| アドレス | 名称 | ビット構成 |
| --- | --- | --- |
| `$E94001` | ドライブ選択/モータ制御 | bit7: モータON(1)/OFF(0), bit0-1: ドライブ番号(00:Dr0, 01:Dr1, 10:Dr2, 11:Dr3) |
| `$E94003` | イジェクト制御 | bit0-3: ドライブ0〜3のイジェクト指示 |
| `$E94005` | 密度切替/状態 | bit0: 記録密度(1:2HD/0:2DD), bit7: ディスク挿入状態(1:挿入済) |

### OPM 経由の補助制御（`$E90001`）

bit0-3: インユース LED 制御およびイジェクト補助信号。

**重要**: モータ ON 後、回転安定まで**約 500ms の待機**が必要。確認せずに FDC コマンドを発行すると Drive Not Ready タイムアウトや FDC ハングアップ。

## FDC 主ステータスレジスタ（MSR: `$E94000`）

| ビット | 名称 | 意味 |
| --- | --- | --- |
| bit 7 | RQM | 1: FDC データ転送準備完了 |
| bit 6 | DIO | 1: FDC→MPU, 0: MPU→FDC |
| bit 5 | EXM | エグゼキューションフェーズ |
| bit 4 | CB | 1: FDC コマンド実行中 |

## フェーズ遷移プロトコル

FDC との通信は「コマンド」「エグゼキューション」「リザルト」の 3 フェーズを巡回。**1バイト送受信ごとに**:

1. RQM が 1 になるまで待機
2. DIO を確認:
   - 書き込み時: `DIO=0` を確認 → コマンドバイト送信
   - 読み出し時: `DIO=1` を確認 → 結果バイト受信
3. 逸脱すると FDC 内部状態が不定になる。

## 主要コマンド

### データ転送系

- `READ DATA`, `WRITE DATA`, `READ DELETED DATA`

### 制御・診断系

- **SEEK**: 指定トラックへ移動。**実行後は必ず `SENSE INTERRUPT STATUS` を発行**して成功確認。
- **RECALIBRATE**: トラック 0 へ復帰。シーク不良からの回復に必須。
- **READ ID**: 現在位置の C/H/R/N を取得。

### 設定系

- **SPECIFY**: FDC タイミングパラメータ。
  - SRT（Step Rate Time）: ヘッド移動速度
  - HUT（Head Unload Time）: アンロードまでの時間
  - HLT（Head Load Time）: ロード後の安定待ち

### トラックフォーマット

Gap 3 の長さは 2HD（27 バイト）と 2DD（22 バイト）で異なる。設定ミスは隣接セクタ破壊の原因。

## リザルトステータス（ST0〜ST3）

- **ST0**: 終了状態。異常時は bit6-7 で原因特定。
- **ST1/ST2**: CRC エラー、セクタ未検出等。
- **ST3**: 書き込みプロテクト、ヘッド位置（トラック 0）等のドライブ状態。

## DMA 設定（Ch0 推奨値）

| レジスタ | 値 | 意味 |
| --- | --- | --- |
| DCR | $80 | 68000バスタイプ、外部要求、サイクルスチール |
| OCR | $B2 | 8bitポート、デバイス→メモリ、サイクルスチール |
| SCR | $04 | メモリ側インクリメント、デバイス側固定 |
| CCR | $80 | 転送開始 |

### サイクルスチール動作

FDC からのデータ要求（Req）ごとに、DMAC が MPU から **1 バスサイクルだけ「盗んで」**転送。MPU は完全停止せず、僅かな速度低下でバックグラウンド転送を維持。

## 読み込みシーケンス

1. Supervisor モード移行
2. `$E94001` にドライブ選択＋モータ ON 書き込み → 500ms 待機
3. DMAC 初期化: MAR, MTC 設定 → CCR = $80 で待機状態
4. MSR をチェックしつつ `READ DATA` コマンド＋パラメータ（C, H, R, N 等）を順次書き込み
5. MSR の EXM と DMAC CSR を監視
6. リザルトフェーズ: ST0-2 等を読み出し、正常終了を検証

## エラー処理（3-Recal-3 戦略）

1. 同一箇所を 3 回リトライ
2. 失敗 → `RECALIBRATE` でトラック 0 へ復帰
3. 再度 `SEEK` で目標トラックへ → さらに 3 回リトライ
4. これでも失敗 → メディア異常として上位レイヤーへ報告

## 割り込みベクタ

- **DMAC**: 初期値 $0F（非初期化）。Human68K では通常 $68（Ch0）。
- **FDC**: I/O コントローラ経由。ベクタは通常 `$60-$63` の範囲。
