# X68000 キーボード・マウス

## 1. キーボード・マウス制御の全体像とシステムアーキテクチャ

X68000という「パーソナルワークステーション」の本質は、ユーザーの思考を遅滞なく画面に反映させる、極めて高いリアルタイム応答性能にある。この応答性をAIによる自動制御、あるいは論理的な入力インジェクションによって再現・超越するためには、ハードウェア・レイヤにおける入力デバイスの統合管理構造を理解しなければならない。

システムのアーキテクチャにおいて、入力制御の双璧をなすのは、キーボード通信を司るMFP（MC68901）と、マウス操作を制御するSCC（Z8530）である。これらは独立したシリアル通信系統を持ちながらも、CPU（MC68000）の割り込み優先順位（レベル5およびレベル6）によって階層化されている。AIがシステムに介入を試みる際、最も重要な接点は、これら周辺デバイスとバスを繋ぐメモリマップドI/O（MMIO）領域、および割り込みベクタの制御権である。物理的なキー押下やマウス移動を電気信号として再現するのではなく、システム側のレジスタ状態を論理的に書き換える「ロジック・インジェクション」こそが、AIによる完全制御の最短経路となる。

次セクションでは、AIがバスを掌握するために直接アクセスすべき、具体的なレジスタアドレスとその戦略的利点について解析する。

## 2. 制御の要：システムポートとメモリマッピングの解析

X68000はI/O空間を持たないMC68000を採用しているため、全デバイス制御はメモリマップドI/O（MMIO）を介して行われる。この設計は、AIにとって「メモリ操作命令（MOVE等）のみでハードウェアを直接叩ける」という極めて高い自由度を示唆している。

### システムI/O領域の優先順位

AIが介入すべきMMIOアドレスは `$E80000` 以降に集約されている。特に、デバイスの接続状態やボタンの状態を直接反映する「システムポート」の解析が、制御の第一歩となる。

|   |   |   |   |
|---|---|---|---|
|ポート名|アドレス|優先度|主要なビット機能とAI制御の役割|
|**システムポート#2**|`$E8E003`|**最優先**|Bit 1: Mouse L-Button / Bit 0: Mouse R-Buttonの状態を反映。デバイスのコネクト状況監視の要。|
|**システムポート#4**|`$E8E007`|中|キーボードへのコントロール信号（LED、リピート設定、ディスプレイ制御）の補助。|
|**システムポート#5**|`$E8E009`|低|キーボード・マウスに関連する特殊信号線。AIによる擬似パルスの生成に使用。|

AIにとって、システムポート#2（`$E8E003`）は「ハードウェアの現在の接続状況」と「物理入力の有無」を判定するための最優先監視対象である。ここを起点に、物理入力とAIによる生成入力をシミュレート・排他制御するための戦略を構築する。

アドレス空間の特定が完了したところで、次は各デバイスが実際にデータをやり取りする通信プロトコルの核心部、MFPによるキーボード制御へと踏み込む。

## 3. キーボード通信：MFPを介したデータ送受信プロトコル

キーボード制御の心臓部は、マルチファンクション・ペリフェラル（MC68901 / MFP）である。このチップはタイマ、GPIP（汎用I/O）、USARTを統合しており、キーボードとの2400bps非同期通信を司る。

### MFP USARTの設定とデータ監視

AIがキーボード入力を完全に掌握するには、MFP内のUSART関連レジスタ、特にステータスフラグの挙動を監視する必要がある。

1. **RSR (レシーバステータスレジスタ)** `**$E8802B**`**:**
    - **Bit 7 (BF: Buffer Full):** 受信バッファにデータが準備されると '1' になる。AIはこのビットをポーリング、あるいは割り込みのトリガーとして、物理キーの打鍵を検知する。
2. **UDR (USARTデータレジスタ)** `**$E8802F**`**:**
    - キーボードからのスキャンコードが格納される。AIが擬似的なキー入力を発生させる場合、物理入力をマスクした上で、このレジスタにターゲットのスキャンコードをインジェクションする。

### 同期タイミングの確保

AIが入力の精度を高めるためには、MFPのGPIPレジスタ（`$E88001`）を介した同期が不可欠である。

- **GPIP 7 (H-SYNC):** 水平同期信号。
- **GPIP 4 (V-DISP):** 垂直表示期間信号。 AIはこれらビットを読み取ることで、画面更新タイミングに同期した入力送出を可能にする。特にV-DISP（Bit 4）の監視は、フレーム単位での入力精度が要求されるアクションゲームやGUI操作の自動化において決定的な役割を果たす。

キーボードの文字入力を論理的に掌握した後は、ポインティングデバイスであるマウスの、より動的な座標制御へと移行する。

## 4. マウス制御：SCCとトラックマウスの統合インターフェース

X68000のトラックマウスは、SCC（シリアルコミュニケーションコントローラ / Z8530）のチャンネルBを介してシステムと通信する。マウス制御は相対座標パケットの連続であり、キーボードよりも頻繁なデータ処理が要求される。

### SCCを介したデータ流し込み

マウスのデータ処理は、割り込みレベル5で実行される。AIがマウスカーソルを自在に操るためには、SCCの内部レジスタを通じて、物理的なマウスからの相対移動量（X, Y方向の変位）をシミュレートしなければならない。

- AIは物理マウスからの割り込みを抑制（後述するIMR操作）し、SCCのバッファに対して、AIが生成した座標変化パケットを周期的に供給する。これにより、システム側は「物理的なマウスが動いている」と誤認し、マウスカーソルの座標更新が行われる。

このシミュレーションを遅延なく実行するためには、システムの割り込み機構そのものをAIの管理下に置く最適化戦略が求められる。

## 5. 割り込み駆動型制御の最適化

X68000のハードウェア割り込みは、レベル6（MFP）とレベル5（SCC）に割り振られている。AIによるデバイス操作を完遂するには、この「レベル割り付け」に伴う優先順位の競合を戦略的に解決しなければならない。

### 優先順位の競合回避とマスキング

- **レベル6 (MFP):** キーボード、タイマ。
- **レベル5 (SCC):** マウス、RS-232C。 ここで発生する課題は、AIがレベル6（キーボード）の処理に時間を取られすぎると、レベル5（マウス）のパケットをドロップする可能性がある点である。AIによる制御ロジックは、これら両レベルを統合監視するスーパーバイザーとして機能する必要がある。

### ハードウェア信号の抑制戦略

物理入力とAI入力を完全に分離するため、AIはMFPの以下のレジスタを操作し、物理デバイスからの割り込みを一時的にマスク（隠蔽）する。

- **IERA/IERB (割り込みイネーブルレジスタ)** `**$E88007/$E88009**`**:** 割り込み発生そのものを許可/禁止する。
- **IMRA/IMRB (インタラプトマスクレジスタ)** `**$E88013/$E88015**`**:** 発生した割り込みをCPUに通知するか否かを制御する。

AIは `IMRA/IMRB` を操作して物理的なデバイス入力を「透明化」し、代わりに自ら生成したイベントを例外ベクタテーブル（MFP: `$40-$4F`, SCC: `$50-$5F`）に書き込んだ独自のハンドラ経由でシステムへ送り出す。これにより、OSやアプリケーションに一切の変更を加えることなく、完全な自動操作が可能となる。

## 6. AIによる自動制御実装への指針

本リファレンスで解析した情報を基に、AIがシステムを統合制御するための論理的フローを以下に示す。これは単なるプログラム順序ではなく、ハードウェア信号と同期したAIの意志決定プロセスである。

### 論理的制御フロー図（テキスト形式）

1. **[INITIALIZATION]**
    - MFPタイマBを確認し、USARTを2400bps非同期モードへ設定維持。
    - $E8E003（システムポート#2）をスキャンし、デバイス接続状況を内部モデルに同期。
2. **[SYNC & MONITOR LOOP]**
    - **IF** `$E88001` Bit 4 (V-DISP) == 1 (垂直表示期間開始) **THEN**:
        - **GOTO** [INJECTION PHASE]
    - **ELSE**:
        - MFP RSR ($E8802B) Bit 7 (BF) をポーリングし、物理キー入力を記録。
3. **[INJECTION PHASE]**
    - **IF** AI_Buffer (Keyboard_Data) が空ではない **THEN**:
        - `IMRA/IMRB` を操作して物理割り込みをマスク。
        - `UDR ($E8802F)` へAI生成スキャンコードをMOVE。
    - **IF** AI_Buffer (Mouse_Movement) が空ではない **THEN**:
        - SCCチャンネルBへAI生成相対座標をシリアル流し込み。
4. **[FEEDBACK & STATE RECOVERY]**
    - システムポート#4 ($E8E007) を監視し、システム側からのLED変更要求をAI内部状態に反映。
    - 制御完了後、`IMRA/IMRB` を復帰させ、物理デバイスとの共存を再開。

以上のハードウェア仕様に基づく制御フローを実装することで、X68000はAIという新たな知性を得た「パーソナルワークステーション」として再構築される。物理的なデバイスの制限を論理的な介入によって突破することこそ、システムアーキテクトが目指すべきAI制御の極致である。
