# X68000 SASI

SASI（Shugart Associates System Interface）は初期型 X68000 の HDD インタフェース。SCSI の前身。全信号は**負論理（Lアクティブ）**、オープンコレクタ構成。

## バス信号

| 信号 | 役割 |
| --- | --- |
| **BSY** | バス使用中。全フェーズで保持される最重要信号 |
| **SEL** | ターゲット選択 |
| **C/D** | L=コマンド/ステータス、H=データ |
| **I/O** | L=IN（ターゲット→本体）、H=OUT |
| **REQ** | ターゲートによるデータ転送要求（ハンドシェイク起点） |
| **ACK** | REQ に対するイニシエータ応答 |
| **RST** | 全デバイス強制初期化 |
| **MSG** | バス状態情報伝達 |

- C/D, I/O, MSG, REQ はターゲット側が制御権を握る。イニシエータ（本体）は提示されたフェーズに従う。
- オープンコレクタ＋終端抵抗により、複数デバイスが同時に信号をドライブしても短絡しない。

## フェーズ遷移

1. **BUS FREE**: BSY/SEL が H（バス開放状態）
2. **SELECTION**: データバスにターゲット ID 出力 → SEL を L にして選択
3. **COMMAND**: ターゲットが C/D=L, I/O=H → 本体から 6 バイト CDB 受け取り
4. **DATA**: I/O 極性に基づきデータ IN/OUT
5. **STATUS**: ターゲットが実行結果（1 バイト）を送出
6. **MESSAGE**: 完了通知（$00）送信 → バス解放

### REQ/ACK ハンドシェイク

非同期ハンドシェイク。ターゲットが REQ を立て、イニシエータが ACK で応える。共通クロック不要。ドライブ側の速度差を吸収。

### セレクション時の ID マッピング

データバス 8 ビットがそのまま ID にマッピング（ID#0 → Bit0 を L、ID#1 → Bit1 を L）。最大 8 台まで識別可能。

## I/O ポート（`$E96001`〜`$E9600F`、奇数番地のみ）

| アドレス | 読み出し | 書き込み |
| --- | --- | --- |
| `$E96001` | データレジスタ | データレジスタ |
| `$E96003` | ステータス（BSY, REQ, I/O, C/D, MSG） | バスリセット（RST）発行 |
| `$E96005` | セレクトコード | セレクション動作トリガー |
| `$E96009` | BSY（bit7）, REQ（bit6） | - |
| `$E9600B` | I/O（bit7）, C/D（bit6）, MSG（bit5） | - |
| `$E9600D` | 一般ステータス | - |
| `$E9600F` | - | 割り込み制御 |

※ `$E96003` は読み出し（Status）と書き込み（Reset）で全く異なる意味を持つ。Read-Modify-Write 操作に注意。

## コマンドフォーマット（6バイト CDB）

| バイト | 内容 |
| --- | --- |
| 第0 | オペレーションコード（READ=$08, WRITE=$0A 等） |
| 第1 | LUN + LBA 上位5ビット |
| 第2-3 | LBA 残り16ビット |
| 第4 | 転送ブロック数（0=256ブロックの場合あり） |
| 第5 | コントロール（通常 $00） |

### 主要コマンド

| コマンド | Opcode | 機能 |
| --- | --- | --- |
| TEST UNIT READY | $00 | ドライブ稼働準備確認 |
| REQUEST SENSE | $03 | エラー時の詳細ステータス取得 |
| READ / WRITE | $08 / $0A | LBA 指定データ転送 |
| SEEK | $0B | ヘッド目標ブロックへの事前移動 |

### 21ビット LBA の壁

6 バイト CDB の LBA フィールドは実質 21 ビット。512B/ブロック換算で最大 1GB。この制約が SCSI-2 の 10 バイト CDB への進化の要因。

## 読み出しアルゴリズム

1. Supervisor 移行
2. データレジスタに ID ビットをセット → セレクションポートで選択
3. C/D, I/O 信号をポーリング監視 → コマンドフェーズ移行確認 → 6 バイト CDB 順次書き込み
4. REQ 立ち上がり確認 → ACK 返しながらデータレジスタから 1 バイトずつ取得
5. ステータス＋メッセージ受信 → BSY 解除でバス解放

## 実装上の注意

- I/O レジスタポインタには `volatile` 必須。コンパイラ最適化でポーリングループが削除されるのを防ぐ。
- ポーリング方式は、SASI のシビアなタイミングにおいて不確定な割り込みレイテンシを排除し、決定論的動作を保証する合理的な選択。

### トラブルシューティング

エラー時は `$E96009`/`$E9600B` で「どの信号が、どのフェーズで固まったか」を確認:

- BSY が落ちない → ターゲット側ハング
- REQ が来ない → フェーズ不整合
