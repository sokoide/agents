# X68000 入力処理

ゲーム向けのキーボード・ジョイスティック・マウス入力処理パターン。

## キーボード — BITSNS (IOCS $04)

キーの押し下げ状態をグループ単位で取得。d1 にグループ番号(0-15)を渡し、戻り値 d0 の各ビット(0-7)がキー押下状態(1=押下)。

```c
/* C言語例: グループ7のカーソルキーを確認 */
int state = _iocs_bitsns(7);
if (state & 0x40) { /* カーソル↓ */ }
if (state & 0x20) { /* カーソル→ */ }
if (state & 0x10) { /* カーソル↑ */ }
if (state & 0x08) { /* カーソル← */ }
```

```asm
; アセンブラ例
    moveq    #7,d1
    IOCS    _BITSNS
    btst    #6,d0           ; カーソル↓
    beq    @f
        ; 押されている
    @@:
```

### 直接アクセス

BITSNS が参照しているワーク領域 `$000800` (16 バイト) を Supervisor モードで直接読み出し可能。IOCS 呼び出しのオーバーヘッドを省ける。

## ジョイスティック — JOYGET (IOCS $3B)

d1 にポート番号 (0=1P, 1=2P) を渡す。戻り値 d0 は**負論理** (ON=0)。

```asm
    moveq    #0,d1           ; 1P
    IOCS    _JOYGET($3B)
    not    d0               ; 正論理化必須

    btst    #0,d0           ; 上
    btst    #1,d0           ; 下
    btst    #2,d0           ; 左
    btst    #3,d0           ; 右
    btst    #5,d0           ; トリガA
    btst    #6,d0           ; トリガB
```

直叩き: `$E9A001` (1P) / `$E9A003` (2P) バイト読み出し。NOT 反転は同じ。

### エッジ検出 (押した瞬間 / 離した瞬間)

ボタンの「押した瞬間」「離した瞬間」だけを検出するパターン。前回値 (old_input) を保持し、XOR で変化を、AND で方向を判定。

```asm
    moveq    #0,d1
    IOCS    _JOYGET
    not    d0               ; d0 = 今回の入力(正論理)

    move    d0,d1
    eor    old_input(a6),d1  ; d1 = 変化したビット
    move    d1,d2
    and    d0,d1            ; d1 = 押した瞬間のビット
    and    old_input(a6),d2  ; d2 = 離した瞬間のビット
    move    d0,old_input(a6)  ; 今回値を保存
```

C 言語での実装:

```c
static unsigned char old_input;
unsigned char new_input = ~(*(volatile unsigned char *)0xE9A001); /* 正論理 */
unsigned char pressed  = (new_input ^ old_input) & new_input;   /* 押した瞬間 */
unsigned char released = (new_input ^ old_input) & old_input;   /* 離した瞬間 */
old_input = new_input;
```

### 8方向ジャンプテーブル

レバー入力を 4 ビット (bit0=右, bit1=左, bit2=下, bit3=上) のインデックスに変換し、16 エントリのジャンプテーブルで処理を分岐。上下同時押しを Start、左右同時押しを Select として処理する設計もある。

```asm
    moveq    #0,d1
    IOCS    _JOYGET
    not    d0
    andi    #15,d0            ; 下位4ビット抽出
    add    d0,d0              ; ワード境界に揃える
    move    joytable(pc,d0),d0
    jmp    joytable(pc,d0)

joytable:
    .dc.w    neutral-joytable          ; 0000 無入力
    .dc.w    right-joytable            ; 0001 右
    .dc.w    left-joytable             ; 0010 左
    .dc.w    select-joytable           ; 0011 左右同時=Select
    .dc.w    down-joytable             ; 0100 下
    .dc.w    right_down-joytable       ; 0101 右下
    .dc.w    left_down-joytable        ; 0110 左下
    .dc.w    select_left-joytable      ; 0111
    .dc.w    up-joytable               ; 1000 上
    .dc.w    right_up-joytable         ; 1001 右上
    .dc.w    left_up-joytable          ; 1010 左上
    .dc.w    select_right-joytable     ; 1011
    .dc.w    start-joytable            ; 1100 上下同時=Start
    .dc.w    start_up-joytable         ; 1101
    .dc.w    start_down-joytable       ; 1110
    .dc.w    start_select-joytable     ; 1111
```

## マウス — MS_GETDT (IOCS $74)

マウスの移動量(Δx, Δy)とボタン状態をパックされた 32 ビット値で取得。引数なし。

### 戻り値 d0.l の構造

| ビット範囲 | 内容 | 値 |
| --- | --- | --- |
| bit 31-24 | 横移動量 (Δx) | 符号付き8ビット |
| bit 23-16 | 縦移動量 (Δy) | 符号付き8ビット |
| bit 15-8 | 左ボタン | $00=離, $FF=押 |
| bit 7-0 | 右ボタン | $00=離, $FF=押 |

### 展開コード

```asm
    IOCS    _MS_GETDT($74)
    move    d0,d1
    swap    d1             ; 上位ワード(Δx)を下位に移動
    move    d1,d2

    lsr    #8,d1
    ext.w    d1            ; d1 = 横移動量(符号付き16ビット)

    ext.w    d2            ; d2 = 縦移動量(符号付き16ビット)

    btst    #8,d0
    bne    msLeftButton    ; 左ボタン押下

    btst    #0,d0
    bne    msRightButton   ; 右ボタン押下
```
