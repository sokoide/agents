# X68000 メモリマップ

Inside X68000 のメモリマップ図に合わせた要約。
既存ファイルのように未確認の細かい周辺アドレスを断定せず、図で確認できる範囲を優先する。

## 全体

| 範囲 | 内容 | メモ |
| :--- | :--- | :--- |
| `$000000-$BFFFFF` | メインメモリ | 最大 12MB。リセット直後は先頭 64KB が IPL イメージに切り替わる |
| `$C00000-$DFFFFF` | グラフィック VRAM | 2MB の空間を持つ。実使用量は表示モード依存 |
| `$E00000-$E7FFFF` | テキスト VRAM | 512KB |
| `$E80000-$EBFFFF` | システム I/O | 直叩きは Supervisor 前提 |
| `$EC0000-$ECFFFF` | ユーザ I/O | 64KB。拡張用 |
| `$ED0000-$ED3FFF` | SRAM | 16KB。バッテリバックアップ |
| `$ED4000-$EDFFFF` | 予約 | 48KB |
| `$F00000-$FBFFFF` | CGROM | 768KB |
| `$FE0000-$FFFFFF` | IPL-ROM | 128KB |

## システム I/O に含まれるもの

メモリマップ図では、`$E80000-$EBFFFF` に以下のブロックが配置されている。

- CRTC
- ビデオコントローラ
- DMAC
- MFP
- RTC
- プリンタ
- FM 音源
- ADPCM
- FDC
- HDC
- SCC
- i8255
- I/O コントローラ
- スプライトレジスタ
- スプライト VRAM

個別のベースアドレスや注意点は、それぞれの reference を参照。

## 実務上の注意

- GVRAM/TVRAM はシステム I/O ではない。I/O 領域と同列に扱わない。
- 画面表示系と音源/ストレージ制御はどちらもメモリマップドだが、要求される権限と副作用が違う。
- 先頭 64KB の IPL イメージ切り替えを考慮し、リセット直後と通常時の見え方を混同しない。
