# Input / SCC / RTC

Inside X68000 の USART、ジョイスティック、RTC まわりの章を踏まえた整理。

## 重要な訂正

- キーボードを SCC 扱いしない。
- SCC は RS-232C / マウス側の文脈で参照する。
- ジョイスティックは i8255 / I/O コントローラ系として扱う。

## キーボード

- キーボードは MFP 内蔵 USART 系として扱う前提で考える。
- したがって、キーボード入力処理と SCC 初期化を同一視しない。
- 実装では IOCS を優先し、直叩きは割り込み・受信処理の設計が必要。

## SCC

- SCC はシリアル通信系の章と対応づける。
- 実務上は RS-232C やマウス関連の扱いで参照する。
- 「SCC Channel A = キーボード」という決め打ちは誤り。

## ジョイスティック

- ジョイスティック章は独立して存在する。
- 参照先は i8255 と I/O コントローラ側。
- 単純な 1 本のポート読み出しだけで完結する前提にしない。

## RTC

- RTC は独立ブロックとしてメモリマップに存在する。
- 時刻保持だけでなく、実装ではバッテリバックアップ前提の周辺設計も意識する。

## 実務ルール

- 入力系は IOCS 優先。直叩きは割り込み・FIFO・状態遷移まで扱えるときだけ。
- キーボード、SCC、i8255、RTC を混同しない。
- MFP を触るコードは、キーボード入力と割り込み制御の両方に影響し得る。
