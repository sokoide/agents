# DMAC (HD63450) Reference

## チャンネル割り当て

- **CH#0**: フロッピーディスク (FDC)
- **CH#1**: ハードディスク (HDC)
- **CH#2**: 拡張スロット (メモリ転送等に使用可能)
- **CH#3**: ADPCM

## レジスタ (Ch 2: $E84080〜)

| オフセット | レジスタ | 説明 |
| :--- | :--- | :--- |
| $00 | CSR | チャネルステータスレジスタ |
| $01 | CER | チャネルエラーレジスタ |
| $04 | DCR | デバイス制御レジスタ |
| $05 | OCR | オペレーション制御レジスタ |
| $06 | SCR | シーケンス制御レジスタ |
| $07 | CCR | チャネルコマンドレジスタ |
| $0A-0B | MTC | メモリ転送カウンタ (Word) |
| $0C-0F | MAR | メモリアドレスレジスタ (Long) |
| $14-17 | DAR | デバイスアドレスレジスタ (Long) |
| $1A-1B | BTC | ベース転送カウンタ (Word) |
| $1C-1F | BAR | ベースアドレスレジスタ (Long) |
| $29 | MFC | メモリファンクションコード |
| $2D | CPR | チャネルパラメータレジスタ |
| $31 | DFC | デバイスファンクションコード |
| $39 | BFC | ベースファンクションコード |

### OCR (Operation Control Register) — $05

| Bit | フィールド | 説明 |
| :--- | :--- | :--- |
| 5-4 | SIZE | 転送サイズ: 00=Byte, 01=16bit(ワード), 10=32bit(ロング) |
| 3-2 | CHAIN | アレイチェイニング: 10=アレイチェイニング有効 |
| 1-0 | REQ | リクエスト生成: 00=オート限定速度, 01=オート最大速度, 10=外部要求, 11=最初オート以降外部 |

**OCR値の例:**
- `$10` = 16bit転送, チェーンなし (SIZE=01, CHAIN=00, REQ=00)
- `$14` = 16bit転送, チェーンなし (SIZE=01, CHAIN=01)
- `$19` = 16bit転送, アレイチェイニング, オート最大 (SIZE=01, CHAIN=10, REQ=01)
- `$20` = 32bit転送, チェーンなし (SIZE=10, CHAIN=00, REQ=00)
- `$29` = 32bit転送, アレイチェイニング, オート最大 (SIZE=10, CHAIN=10, REQ=01)

### DCR (Device Control Register) — $04

| Bit | フィールド | 説明 |
| :--- | :--- | :--- |
| 3 | DPS | デバイスポートサイズ: 0=8bit, 1=16bit |

**DCR値の例:**
- `$00` = 8bitポート
- `$08` = 16bitポート (bit 3)

### SCR (Sequence Control Register) — $06

| Bit | フィールド | 説明 |
| :--- | :--- | :--- |
| 2 | MAR_INC | MARインクリメント有効 |
| 0 | DAR_INC | DARインクリメント有効 |

**SCR値の例:**
- `$05` = MAR inc + DAR inc (bit 2 + bit 0)

### CCR (Channel Command Register) — $07

| Bit | フィールド | 説明 |
| :--- | :--- | :--- |
| 7 | START | 転送開始 |

- CCR=`$80`: 転送開始
- CCR=`$00`: 転送停止（チャネルリセット、内部アドレスカウンタもリセット）

### CSR (Channel Status Register) — $00

| Bit | フィールド | 説明 |
| :--- | :--- | :--- |
| 7 | COC | 転送完了フラグ |
| 4 | ACT | 動作中フラグ |

- 転送完了待ち: `while (!(ch[CSR] & 0x90))`
- 転送開始前に `CSR = 0xFF` で前回のフラグをクリアするのが作法

## DARの重要仕様

- **転送動作中**にDARレジスタに値を書き込むと動作タイミングエラーになる
- **連続転送**（複数ブロック転送）では、DARは自動的にリロードされない
- 前回の転送終了時の内部状態がそのまま引き継がれる
- **対処**: 各転送前に CCR=`$00` でチャネルを停止し、内部カウンタをリセットしてからDARを書き込む

## 転送モード

### ノン・チェーンモード (OCR bit 2-3 = 00)

CCR=`$80` で開始後、MTCワード転送が完了したら停止。
複数回転送するには毎回 CCR=`$00`→レジスタ設定→CCR=`$80` が必要。

```
Setup: DCR, OCR=$10, SCR
Loop:
  CCR=$00  // チャネル停止（内部カウンタリセット）
  CSR=$FF  // フラグクリア
  MTC=w, MAR=src, DAR=dst
  CCR=$80  // 転送開始
  wait CSR & 0x90
```

### アレイチェイニングモード (OCR bit 2-3 = 10, OCR=$19)

BTCエントリ数分の転送を1回のCCR=`$80`で自動実行。
BARにチェーンテーブルの先頭アドレスを、BTCにエントリ数を設定する。

**X68000 HD63450 チェーンエントリフォーマット (6 bytes/entry):**

```
[MAR: 4 bytes (big-endian)] [MTC: 2 bytes (big-endian)]
```

※標準HD63450とは異なり、X68000ではMARが先、MTCが後。

**チェーンテーブル構造体:**
```c
typedef struct {
    uint32_t mar;   // 転送元アドレス (4 bytes)
    uint16_t mtc;   // 転送ワード数 (2 bytes, 0=65536)
} dma_chain6_t;  // 6 bytes total
```

**制約事項:**
- DARはエントリごとに設定できない（自動進行のみ）
- 連続転送（src/dstストライド == 転送幅w）の場合にのみ使用可能
- MTC=0 は 65536 を意味する
- BTCは16-bit、最大 65535 エントリ

```
Setup: DCR=$08, OCR=$19, SCR=$05
BAR = chain_table
BTC = num_entries
DAR = dst_base
MAR is loaded from each chain entry
CCR=$80  // 全エントリを自動処理
wait CSR & 0x90
```

## 1バイト飛ばし転送テクニック

16bitデバイスポート (DCR bit3=1) + Byte転送 (OCR SIZE=00) の組み合わせで、
DMAが1バイトずつ16bitデバイスに転送し、DARが自動的に+2インクリメントされる。

- **用途**: 16色VRAMの特定プレーンのみ更新したい場合等
- **65536色には不適**: 全ワード転送には OCR SIZE=01 + DCR bit3=1 を使用すること

## Human68k初期設定値例

- **OCR**: `$B2` (CH#0,1), `$32` (CH#3) — 外部要求、サイクルスチール
- **SCR**: `$04` (CH#0,1,3) — メモリアドレス増加、デバイスアドレス固定
