# DMAC (HD63450) 実装リファレンス

> 目的: 転送設定ミス（サイズ/チェーン/再始動）を防ぐ。

## チャンネル割り当て

- **CH0**: FDC
- **CH1**: HDC
- **CH2**: 汎用/拡張用途
- **CH3**: ADPCM

## よく使うレジスタ（チャネル基底 + offset）

| Offset | レジスタ | 役割 |
| :--- | :--- | :--- |
| `$00` | CSR | ステータス（完了/動作中） |
| `$01` | CER | エラー種別 |
| `$04` | DCR | デバイス側ポート条件 |
| `$05` | OCR | 転送サイズ/要求方式/チェーン方式 |
| `$06` | SCR | MAR/DAR の増減制御 |
| `$07` | CCR | 開始/停止コマンド |
| `$0A-$0B` | MTC | 転送カウント |
| `$0C-$0F` | MAR | メモリアドレス |
| `$14-$17` | DAR | デバイスアドレス |
| `$1A-$1B` | BTC | チェーン要素数 |
| `$1C-$1F` | BAR | チェーンテーブル先頭 |

## 実戦ルール

- 再転送前は **CCR 停止 → CSR クリア → 各レジスタ再設定 → CCR 開始** を徹底する。
- 動作中の DAR/MAR 直接書き換えは避ける（不定動作の原因）。
- 大きな連続転送はチェーンを使い、CPU ループを避ける。

## 推奨シーケンス（非チェーン）

```text
CCR = STOP
CSR = CLEAR
DCR/OCR/SCR 設定
MTC/MAR/DAR 設定
CCR = START
CSR 完了ビット待ち
```

## アレイチェーンの要点

- BAR: チェーンテーブル先頭
- BTC: エントリ数
- 1 エントリは「転送元 + 転送量」を持つ形式で扱う
- DAR は連続先として運用し、エントリごとの個別 DAR を前提にしない

## 安全チェック

- 転送サイズ（Byte/Word/Long）と DCR のポート幅設定が一致しているか。
- VRAM 先が 16bit 境界を満たすか。
- 完了待ちループが `volatile` 読み取りになっているか。
