# Security Review Checkpoints

## 1. Input Validation
- [ ] 外部からの入力（HTTPリクエスト、CLI引数、環境変数）はすべてバリデーションされているか？
- [ ] SQL インジェクションの脆弱性はないか？（Prepared Statement / Placeholder を使用しているか）
- [ ] XSS 対策はなされているか？（エスケープなど）

## 2. Authentication & Authorization
- [ ] 認証が必要なエンドポイントは保護されているか？
- [ ] 認可（権限チェック）は UseCase 層または適切なインターセプターで実行されているか？
- [ ] 機密情報（パスワード、APIキー）がログに出力されていないか？

## 3. Data Protection
- [ ] 通信は TLS で保護されているか？
- [ ] 機密データは適切にハッシュ化または暗号化されて保存されているか？
- [ ] 不要なエラー詳細が外部（レスポンス）に露出していないか？

## 4. Resource Management
- [ ] タイムアウトが適切に設定されているか？
- [ ] リソース（ファイルポインタ、DB接続、Goroutine）が適切にクローズ/解放されているか？
- [ ] DoS 対策（レートリミット、ボディサイズ制限など）が検討されているか？
